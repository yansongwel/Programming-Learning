# 日志分析

> 一句话说明：用 Shell 工具链对各类日志进行实战分析，快速定位问题和生成报表。

---

## 1. Nginx/Apache 访问日志分析

```bash
# 日志格式示例
# 192.168.1.100 - - [15/Jan/2024:14:30:45 +0800] "GET /api/users HTTP/1.1" 200 1234 "-" "curl/7.68"

# Top 10 访问 IP
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# Top 10 请求 URL
awk '{print $7}' access.log | sort | uniq -c | sort -rn | head -10

# 各状态码统计
awk '{print $9}' access.log | sort | uniq -c | sort -rn

# 每小时请求量
awk '{print substr($4,14,2)":00"}' access.log | sort | uniq -c

# 查看 5xx 错误
awk '$9 >= 500' access.log

# 慢请求（假设最后一列是响应时间，单位ms）
awk '$NF > 1000 {printf "%s %s %s %sms\n", $1, $7, $9, $NF}' access.log | sort -t'=' -k2 -rn | head -20

# 带宽统计（按 IP）
awk '{bytes[$1]+=$10} END {for(ip in bytes) printf "%15s  %.2f MB\n", ip, bytes[ip]/1024/1024}' access.log | sort -k2 -rn | head -10

# 按时间段分析错误（14:00~15:00）
awk '/15\/Jan\/2024:1[4-4]/' access.log | awk '$9 >= 400' | awk '{print $9}' | sort | uniq -c
```

## 2. 应用日志分析

```bash
# 日志格式示例
# 2024-01-15 14:30:45 [ERROR] [thread-1] com.app.Service - Connection timeout

# 按级别统计
grep -cE '\[(ERROR|WARN|INFO|DEBUG)\]' app.log
awk -F'[][]' '{print $2}' app.log | sort | uniq -c | sort -rn

# 每分钟错误数趋势
grep "ERROR" app.log | awk '{print substr($1,1,10)" "$2}' | cut -d: -f1,2 | sort | uniq -c

# 错误类型分布
grep "ERROR" app.log | grep -oE '[A-Za-z]+Exception' | sort | uniq -c | sort -rn

# 查看某个错误的上下文
grep -B 5 -A 10 "OutOfMemoryError" app.log

# 统计各线程的错误数
grep "ERROR" app.log | grep -oE '\[thread-[0-9]+\]' | sort | uniq -c | sort -rn
```

## 3. 系统日志分析

```bash
# syslog / journalctl
# 查看最近的错误
journalctl -p err --since "1 hour ago" --no-pager

# 统计各服务的日志量
journalctl --since today --no-pager | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn | head -20

# 登录失败统计
grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10

# OOM 检查
dmesg | grep -i "oom\|out of memory"
grep -i "oom" /var/log/syslog
```

## 4. 完整分析脚本

```bash
#!/usr/bin/env bash
# log_analyzer.sh — Nginx 日志分析报告

set -euo pipefail

LOG_FILE="${1:?用法: $0 <access.log>}"
[[ -f "$LOG_FILE" ]] || { echo "文件不存在: $LOG_FILE"; exit 1; }

TOTAL=$(wc -l < "$LOG_FILE")

echo "========================================="
echo "  Nginx 日志分析报告"
echo "  文件: $LOG_FILE"
echo "  总请求数: $TOTAL"
echo "========================================="

echo ""
echo "--- 状态码分布 ---"
awk '{print $9}' "$LOG_FILE" | sort | uniq -c | sort -rn | \
    awk '{printf "  %-6s %8d (%5.1f%%)\n", $2, $1, $1*100/'"$TOTAL"'}'

echo ""
echo "--- Top 10 访问 IP ---"
awk '{print $1}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -10 | \
    awk '{printf "  %-18s %8d 次\n", $2, $1}'

echo ""
echo "--- Top 10 请求路径 ---"
awk '{print $7}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -10 | \
    awk '{printf "  %-40s %8d 次\n", $2, $1}'

echo ""
echo "--- 每小时请求量 ---"
awk '{print substr($4,14,2)}' "$LOG_FILE" | sort | uniq -c | \
    awk '{printf "  %s:00  %8d 次  ", $2, $1; bar=""; for(i=0;i<$1/100;i++) bar=bar"█"; print bar}'

echo ""
echo "--- 5xx 错误 (最近 10 条) ---"
awk '$9 >= 500 {print $1, $4, $7, $9}' "$LOG_FILE" | tail -10 | \
    awk '{printf "  %s %s %s [%s]\n", $1, $2, $3, $4}'

echo ""
echo "========================================="
```

## 5. 小结

| 分析类型 | 核心命令 |
|----------|----------|
| IP 统计 | `awk '{print $1}' \| sort \| uniq -c \| sort -rn` |
| 状态码 | `awk '{print $9}' \| sort \| uniq -c` |
| 时间趋势 | `awk + substr` 截取时间 + `uniq -c` |
| 错误上下文 | `grep -B 5 -A 10 "ERROR"` |
| 实时监控 | `tail -f \| grep --line-buffered` |
