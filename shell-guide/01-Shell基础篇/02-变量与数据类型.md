# Shell 变量与数据类型

> 一句话说明：掌握变量的定义、赋值、引用和作用域，理解 Shell 中"一切皆字符串"的核心思想。

---

## 1. 变量基础

### 1.1 变量定义与赋值

```bash
# 定义变量（等号两边不能有空格！）
NAME="张三"
AGE=25
HOST="192.168.1.100"

# ❌ 常见错误
NAME = "张三"    # 错误！bash 会把 NAME 当命令执行
NAME= "张三"     # 错误！
NAME ="张三"     # 错误！
```

### 1.2 变量引用

```bash
NAME="World"

# 基本引用
echo $NAME           # World
echo ${NAME}         # World（推荐，边界更清晰）
echo "Hello, ${NAME}!"  # Hello, World!

# 为什么推荐 ${} 花括号写法
FILE="report"
echo "$FILEname"     # 空！bash 以为变量名是 FILEname
echo "${FILE}name"   # reportname ✓
```

### 1.3 只读变量

```bash
readonly PI=3.14159
PI=3.14    # 报错: -bash: PI: readonly variable

# 另一种方式
declare -r DB_HOST="localhost"
```

### 1.4 删除变量

```bash
NAME="张三"
echo $NAME    # 张三

unset NAME
echo $NAME    # （空）

# 注意：readonly 变量不能 unset
```

## 2. 引号的区别

这是 Shell 编程中最重要的概念之一。

```bash
NAME="World"

# 双引号：允许变量替换和命令替换
echo "Hello, $NAME"       # Hello, World
echo "Today is $(date)"   # Today is Thu Jan 01 ...

# 单引号：所有内容原样输出，不做任何替换
echo 'Hello, $NAME'       # Hello, $NAME
echo 'Today is $(date)'   # Today is $(date)

# 反引号/命令替换：执行命令并返回结果
echo "Files: `ls | wc -l`"       # 旧式写法（不推荐）
echo "Files: $(ls | wc -l)"      # 新式写法（推荐，可嵌套）
```

**引号对比表：**

| 引号类型 | 变量替换 | 命令替换 | 转义字符 | 使用场景 |
|----------|---------|---------|---------|---------|
| 双引号 `"` | ✅ | ✅ | ✅ | 大多数场景（推荐默认使用） |
| 单引号 `'` | ❌ | ❌ | ❌ | 需要原样保留内容时 |
| 无引号 | ✅ | ✅ | ✅ | 简单场景（但容易踩坑） |

```bash
# 不加引号的隐患
FILE="my report.txt"
rm $FILE         # 实际执行: rm my report.txt（删除两个文件！）
rm "$FILE"       # 实际执行: rm "my report.txt"（正确）

# 经验法则：变量引用几乎总是应该用双引号包裹
```

## 3. 环境变量与局部变量

### 3.1 环境变量

```bash
# 查看所有环境变量
env
printenv

# 常用环境变量
echo $HOME        # 用户主目录: /home/username
echo $USER        # 当前用户名
echo $PATH        # 命令搜索路径
echo $PWD         # 当前工作目录
echo $SHELL       # 当前 Shell
echo $HOSTNAME    # 主机名
echo $LANG        # 语言设置
echo $TERM        # 终端类型
echo $RANDOM      # 随机数 (0-32767)
echo $$           # 当前进程 PID
echo $?           # 上一条命令的退出码
echo $!           # 最近后台进程的 PID
```

### 3.2 export 导出变量

```bash
# 普通变量只在当前 Shell 可见
MY_VAR="hello"
bash -c 'echo $MY_VAR'   # （空）子进程看不到

# export 后子进程可以继承
export MY_VAR="hello"
bash -c 'echo $MY_VAR'   # hello

# 定义时同时 export
export DB_HOST="localhost"
export DB_PORT=3306
```

### 3.3 变量作用域

```bash
#!/bin/bash

GLOBAL_VAR="全局变量"

my_func() {
    local LOCAL_VAR="局部变量"     # local 声明局部变量
    GLOBAL_VAR="在函数中修改了"     # 修改全局变量
    echo "函数内: LOCAL_VAR=${LOCAL_VAR}"
    echo "函数内: GLOBAL_VAR=${GLOBAL_VAR}"
}

my_func
echo "函数外: LOCAL_VAR=${LOCAL_VAR:-未定义}"    # 未定义
echo "函数外: GLOBAL_VAR=${GLOBAL_VAR}"          # 在函数中修改了
```

## 4. 特殊变量

```bash
#!/bin/bash
# special_vars.sh — 演示特殊变量

echo "脚本名称: $0"
echo "第一个参数: $1"
echo "第二个参数: $2"
echo "所有参数(整体): $*"
echo "所有参数(独立): $@"
echo "参数个数: $#"
echo "当前进程PID: $$"
echo "上条命令退出码: $?"

# $* 与 $@ 的区别（在双引号内才体现）
set -- "hello world" "foo bar"

echo '--- "$*" ---'
for arg in "$*"; do
    echo "  arg: $arg"    # 只循环一次: hello world foo bar
done

echo '--- "$@" ---'
for arg in "$@"; do
    echo "  arg: $arg"    # 循环两次: hello world / foo bar
done

# 结论：遍历参数时，永远使用 "$@"
```

## 5. 变量替换（参数展开）

这是 Shell 中非常强大的特性，可以减少大量 if/else 判断。

### 5.1 默认值操作

```bash
# ${var:-default}  如果 var 未定义或为空，返回 default（不修改 var）
echo ${NAME:-"匿名用户"}       # 如果 NAME 为空则输出"匿名用户"

# ${var:=default}  如果 var 未定义或为空，赋值为 default 并返回
echo ${NAME:="匿名用户"}       # 如果 NAME 为空则赋值并输出

# ${var:+value}    如果 var 已定义且非空，返回 value
echo ${NAME:+"NAME已设置"}     # NAME 有值时输出"NAME已设置"

# ${var:?message}  如果 var 未定义或为空，打印错误信息并退出
echo ${DB_HOST:?"DB_HOST 未设置，请检查配置"}
```

### 5.2 字符串操作

```bash
FILE="/home/user/documents/report.tar.gz"

# 字符串长度
echo ${#FILE}                # 37

# 子串截取
echo ${FILE:0:5}             # /home
echo ${FILE:6}               # user/documents/report.tar.gz

# 删除匹配（从左边开始）
echo ${FILE#*/}              # home/user/documents/report.tar.gz  (最短匹配)
echo ${FILE##*/}             # report.tar.gz  (最长匹配 = 取文件名)

# 删除匹配（从右边开始）
echo ${FILE%.*}              # /home/user/documents/report.tar  (最短匹配)
echo ${FILE%%.*}             # /home/user/documents/report  (最长匹配)
echo ${FILE%/*}              # /home/user/documents  (取目录路径)

# 替换
echo ${FILE/report/summary}  # 替换第一个匹配
echo ${FILE//o/O}            # 替换所有匹配

# 大小写转换（Bash 4.0+）
STR="Hello World"
echo ${STR^^}                # HELLO WORLD  (全部大写)
echo ${STR,,}                # hello world  (全部小写)
echo ${STR^}                 # Hello World  (首字母大写)
```

### 5.3 实用示例

```bash
# 批量重命名文件扩展名
for file in *.txt; do
    mv "$file" "${file%.txt}.md"
done

# 从路径中提取信息
FILEPATH="/var/log/nginx/access.log"
echo "目录: ${FILEPATH%/*}"         # /var/log/nginx
echo "文件名: ${FILEPATH##*/}"       # access.log
echo "扩展名: ${FILEPATH##*.}"       # log
echo "无扩展名: ${FILEPATH%.*}"      # /var/log/nginx/access

# 安全的默认值写法
TIMEOUT="${TIMEOUT:-30}"
RETRIES="${RETRIES:-3}"
LOG_LEVEL="${LOG_LEVEL:-INFO}"
```

## 6. 数值运算

Shell 中变量默认都是字符串，做数学运算需要特殊语法。

```bash
# 方式一：$(( )) 算术展开（推荐）
A=10
B=3
echo $((A + B))      # 13
echo $((A - B))      # 7
echo $((A * B))      # 30
echo $((A / B))      # 3（整数除法）
echo $((A % B))      # 1（取余）
echo $((A ** 2))     # 100（幂运算）

# 支持自增自减
COUNT=0
((COUNT++))
echo $COUNT    # 1
((COUNT+=5))
echo $COUNT    # 6

# 方式二：let 命令
let "RESULT = A + B"
echo $RESULT   # 13

# 方式三：expr 命令（旧式，不推荐）
RESULT=$(expr $A + $B)    # 注意运算符两边必须有空格

# 浮点运算需要用 bc
echo "scale=2; 10 / 3" | bc      # 3.33
echo "scale=4; sqrt(2)" | bc -l  # 1.4142

# 或用 awk
awk "BEGIN{printf \"%.2f\n\", 10/3}"   # 3.33
```

## 7. declare 声明变量类型

```bash
# 声明整数类型
declare -i NUM=10
NUM=NUM+5        # 自动做算术运算
echo $NUM        # 15
NUM="hello"      # 非法赋值
echo $NUM        # 0

# 声明数组
declare -a ARRAY=("a" "b" "c")

# 声明关联数组（Bash 4.0+）
declare -A MAP
MAP[name]="张三"
MAP[age]=25

# 声明只读
declare -r CONST="不可修改"

# 声明大写/小写（Bash 4.0+）
declare -u UPPER="hello"    # 自动转大写
echo $UPPER                 # HELLO

declare -l LOWER="HELLO"    # 自动转小写
echo $LOWER                 # hello

# 查看变量属性
declare -p NUM
# declare -i NUM="15"
```

## 8. 实战练习

### 练习 1：配置文件解析器

```bash
#!/usr/bin/env bash
# config_parser.sh — 简易配置文件解析

set -euo pipefail

# 模拟一个配置文件
CONFIG_FILE="/tmp/app.conf"
cat > "$CONFIG_FILE" << 'EOF'
# 应用配置
APP_NAME=MyApp
APP_PORT=8080
DB_HOST=localhost
DB_PORT=3306
DB_NAME=mydb
LOG_LEVEL=INFO
EOF

# 读取配置（跳过注释和空行）
while IFS='=' read -r key value; do
    # 跳过注释和空行
    [[ -z "$key" || "$key" =~ ^# ]] && continue
    # 去除首尾空格
    key=$(echo "$key" | xargs)
    value=$(echo "$value" | xargs)
    # 导出为环境变量
    export "$key=$value"
    echo "已加载: ${key}=${value}"
done < "$CONFIG_FILE"

echo ""
echo "应用名称: ${APP_NAME}"
echo "监听端口: ${APP_PORT}"
echo "数据库地址: ${DB_HOST}:${DB_PORT}/${DB_NAME}"
```

### 练习 2：简易计算器

```bash
#!/usr/bin/env bash
# calculator.sh — 命令行计算器

set -euo pipefail

usage() {
    echo "Usage: $0 <num1> <operator> <num2>"
    echo "Operators: + - x / %"
    echo "Example: $0 10 + 5"
    exit 1
}

[[ $# -ne 3 ]] && usage

NUM1=$1
OP=$2
NUM2=$3

case "$OP" in
    +) RESULT=$((NUM1 + NUM2)) ;;
    -) RESULT=$((NUM1 - NUM2)) ;;
    x|X) RESULT=$((NUM1 * NUM2)) ;;
    /)
        [[ "$NUM2" -eq 0 ]] && { echo "错误：除数不能为 0"; exit 1; }
        RESULT=$(echo "scale=2; $NUM1 / $NUM2" | bc)
        ;;
    %) RESULT=$((NUM1 % NUM2)) ;;
    *) echo "不支持的运算符: $OP"; usage ;;
esac

echo "$NUM1 $OP $NUM2 = $RESULT"
```

## 9. 常见错误与排查

```bash
# 错误 1：赋值等号有空格
NAME = "value"   # ❌ bash: NAME: command not found
NAME="value"     # ✓

# 错误 2：引用变量忘记引号
FILE="my file.txt"
cat $FILE         # ❌ 试图读取 "my" 和 "file.txt" 两个文件
cat "$FILE"       # ✓

# 错误 3：混淆单引号和双引号
NAME="World"
echo 'Hello $NAME'   # 输出: Hello $NAME（不是你想要的）
echo "Hello $NAME"    # 输出: Hello World

# 错误 4：在 $(( )) 中用 $ 引用变量
A=5
echo $(($A + 1))     # 可以，但多余
echo $((A + 1))      # 更简洁，(( )) 内不需要 $
```

## 10. 小结

| 知识点 | 要点 |
|--------|------|
| 变量赋值 | 等号两边无空格，`KEY="value"` |
| 变量引用 | 推荐 `"${VAR}"` 形式 |
| 引号区别 | 双引号展开变量，单引号原样输出 |
| 默认值 | `${VAR:-default}` 简化判断 |
| 字符串操作 | `#` `##` `%` `%%` 删除匹配 |
| 数值运算 | `$(( ))` 整数运算，`bc` 浮点运算 |
| 特殊变量 | `$?` `$#` `$@` `$$` `$0` 是常考点 |
