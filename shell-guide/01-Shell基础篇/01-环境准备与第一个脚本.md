# Shell 环境准备与第一个脚本

> 一句话说明：了解 Shell 是什么，配置好开发环境，写出并运行你的第一个 Shell 脚本。

---

## 1. 什么是 Shell

Shell 是操作系统与用户之间的"翻译官"——你输入命令，Shell 解释并交给内核执行。

```
用户输入命令 → Shell 解释 → 内核执行 → 返回结果 → Shell 显示
```

### 常见的 Shell 类型

| Shell | 路径 | 特点 |
|-------|------|------|
| bash | /bin/bash | Linux 默认，功能最全面，兼容性最好 |
| sh | /bin/sh | POSIX 标准，功能最少，兼容性最广 |
| zsh | /bin/zsh | macOS 默认，扩展功能丰富 |
| dash | /bin/dash | Debian/Ubuntu 的 /bin/sh，启动极快 |
| fish | /usr/bin/fish | 现代交互式 Shell，语法与 bash 不兼容 |

```bash
# 查看当前使用的 Shell
echo $SHELL

# 查看系统安装了哪些 Shell
cat /etc/shells

# 查看 bash 版本
bash --version
```

### 为什么学 Bash

- **普及率最高**：几乎所有 Linux 发行版默认安装
- **运维必备**：90% 的运维脚本用 bash 编写
- **CI/CD 基础**：GitHub Actions、Jenkins Pipeline 底层都是 Shell
- **面试加分**：SRE/DevOps 岗位必考

## 2. 环境准备

### 2.1 Linux 环境（推荐）

```bash
# CentOS / RHEL
sudo yum install -y bash bash-completion vim

# Ubuntu / Debian
sudo apt install -y bash bash-completion vim

# 确认版本
bash --version
# GNU bash, version 5.x.x
```

### 2.2 macOS 环境

```bash
# macOS 自带的 bash 版本很旧（3.2），建议升级
brew install bash

# 查看新版 bash 路径
which -a bash
# /opt/homebrew/bin/bash  (新版)
# /bin/bash               (系统自带旧版)

# 将新版 bash 添加到允许列表
sudo sh -c 'echo /opt/homebrew/bin/bash >> /etc/shells'

# 切换默认 Shell（可选）
chsh -s /opt/homebrew/bin/bash
```

### 2.3 Windows 环境

```bash
# 方式一：WSL2（推荐）
wsl --install -d Ubuntu

# 方式二：Git Bash（轻量）
# 安装 Git for Windows 即可获得

# 方式三：虚拟机
# 安装 VirtualBox + CentOS/Ubuntu
```

### 2.4 编辑器配置

```bash
# 推荐 VS Code + ShellCheck 插件
# ShellCheck 是 Shell 脚本的"代码质量检查器"

# CentOS
sudo yum install -y ShellCheck

# Ubuntu
sudo apt install -y shellcheck

# macOS
brew install shellcheck

# 使用方式
shellcheck your_script.sh
```

## 3. 第一个 Shell 脚本

### 3.1 Hello World

```bash
#!/bin/bash
# 文件名：hello.sh
# 说明：我的第一个 Shell 脚本

echo "Hello, Shell World!"
echo "当前用户: $(whoami)"
echo "当前时间: $(date '+%Y-%m-%d %H:%M:%S')"
echo "当前目录: $(pwd)"
```

### 3.2 运行脚本的三种方式

```bash
# 方式一：赋予执行权限后直接运行（推荐）
chmod +x hello.sh
./hello.sh

# 方式二：用 bash 解释器运行
bash hello.sh

# 方式三：用 source 在当前 Shell 中运行
source hello.sh
# 或
. hello.sh
```

**三种方式的区别：**

| 方式 | 是否新建子进程 | 是否需要执行权限 | 变量是否影响当前 Shell |
|------|---------------|-----------------|----------------------|
| `./script.sh` | 是 | 是 | 否 |
| `bash script.sh` | 是 | 否 | 否 |
| `source script.sh` | 否 | 否 | 是 |

```bash
# 验证 source 的区别
# test_source.sh
#!/bin/bash
MY_VAR="hello_from_script"

# 用 bash 运行
bash test_source.sh
echo $MY_VAR    # 输出为空，变量不会传到当前 Shell

# 用 source 运行
source test_source.sh
echo $MY_VAR    # 输出: hello_from_script
```

### 3.3 Shebang 行详解

```bash
#!/bin/bash
# ↑ 这一行叫 Shebang（也叫 Hashbang）

# 作用：告诉系统用哪个解释器来执行这个脚本
# #!  → 魔数（magic number），系统识别的标记
# /bin/bash → 解释器的绝对路径
```

常见的 Shebang：

```bash
#!/bin/bash          # 使用 bash（最常用）
#!/bin/sh            # 使用 POSIX sh（更兼容）
#!/usr/bin/env bash  # 通过 env 查找 bash（更灵活，推荐）
#!/usr/bin/env python3  # Python 脚本也用 Shebang
```

## 4. 脚本编写规范

### 4.1 基本模板

```bash
#!/usr/bin/env bash
#
# 脚本名称: script_name.sh
# 功能描述: 简要说明脚本的用途
# 作者: your_name
# 创建日期: 2024-01-01
# 使用方法: bash script_name.sh [参数]
#

# 严格模式（推荐始终开启）
set -euo pipefail

# ========================
# 全局变量
# ========================
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
SCRIPT_NAME="$(basename "$0")"
LOG_FILE="/tmp/${SCRIPT_NAME%.sh}.log"

# ========================
# 函数定义
# ========================
usage() {
    echo "Usage: ${SCRIPT_NAME} [options]"
    echo "Options:"
    echo "  -h, --help    显示帮助信息"
    exit 0
}

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "${LOG_FILE}"
}

# ========================
# 主逻辑
# ========================
main() {
    log "脚本开始执行..."
    # 你的代码写在这里
    log "脚本执行完成"
}

# 入口
main "$@"
```

### 4.2 set -euo pipefail 详解

```bash
set -e          # 命令出错时立即退出（Exit on error）
set -u          # 使用未定义变量时报错（Unset variables as error）
set -o pipefail # 管道中任一命令失败则整个管道失败

# 合并写法
set -euo pipefail

# 示例：没有 set -e 时
rm /nonexistent/file    # 报错但继续执行
echo "这行仍会执行"     # 会输出

# 有 set -e 时
set -e
rm /nonexistent/file    # 报错
echo "这行不会执行"     # 不会输出，脚本已退出
```

## 5. 实战练习

### 练习 1：系统信息报告

```bash
#!/usr/bin/env bash
# system_info.sh — 输出当前系统基本信息

set -euo pipefail

echo "========== 系统信息报告 =========="
echo "主机名:     $(hostname)"
echo "操作系统:   $(cat /etc/os-release 2>/dev/null | grep PRETTY_NAME | cut -d'"' -f2 || echo '未知')"
echo "内核版本:   $(uname -r)"
echo "CPU 架构:   $(uname -m)"
echo "当前用户:   $(whoami)"
echo "当前时间:   $(date '+%Y-%m-%d %H:%M:%S')"
echo "已运行时间: $(uptime -p 2>/dev/null || uptime)"
echo "内存使用:   $(free -h 2>/dev/null | awk '/Mem:/{print $3"/"$2}' || echo '未知')"
echo "磁盘使用:   $(df -h / | awk 'NR==2{print $3"/"$2" ("$5" used)"}')"
echo "=================================="
```

### 练习 2：带参数的问候脚本

```bash
#!/usr/bin/env bash
# greet.sh — 根据时间段问候用户

set -euo pipefail

NAME="${1:-$(whoami)}"  # 第一个参数，默认为当前用户名
HOUR=$(date +%H)

if [ "$HOUR" -lt 12 ]; then
    GREETING="早上好"
elif [ "$HOUR" -lt 18 ]; then
    GREETING="下午好"
else
    GREETING="晚上好"
fi

echo "${GREETING}, ${NAME}! 现在是 $(date '+%H:%M')。"
```

## 6. 常见错误与排查

### 错误 1：权限不足

```bash
$ ./hello.sh
-bash: ./hello.sh: Permission denied

# 解决：添加执行权限
chmod +x hello.sh
```

### 错误 2：Shebang 路径错误

```bash
$ ./hello.sh
-bash: ./hello.sh: /bin/bash^M: bad interpreter

# 原因：Windows 下编辑的文件有 \r\n 换行符
# 解决：
dos2unix hello.sh
# 或
sed -i 's/\r$//' hello.sh
```

### 错误 3：找不到命令

```bash
$ ./hello.sh
./hello.sh: line 5: some_command: command not found

# 排查步骤
which some_command      # 查看命令是否安装
type some_command       # 查看命令类型
echo $PATH              # 查看 PATH 是否包含命令所在目录
```
