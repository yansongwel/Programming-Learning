# Shell 输入输出与重定向

> 一句话说明：理解文件描述符、重定向和管道，掌握 Shell 数据流控制的核心机制。

---

## 1. 文件描述符（FD）

Linux 中一切皆文件，每个进程打开的文件用数字编号标识：

| 文件描述符 | 名称 | 默认指向 | 简写 |
|-----------|------|---------|------|
| 0 | 标准输入 (stdin) | 键盘 | `/dev/stdin` |
| 1 | 标准输出 (stdout) | 屏幕 | `/dev/stdout` |
| 2 | 标准错误 (stderr) | 屏幕 | `/dev/stderr` |

```bash
# 查看当前进程的文件描述符
ls -la /proc/$$/fd
# 0 -> /dev/pts/0  (终端输入)
# 1 -> /dev/pts/0  (终端输出)
# 2 -> /dev/pts/0  (终端错误)
```

## 2. 输出重定向

### 2.1 基本重定向

```bash
# > 覆盖写入（stdout → 文件）
echo "Hello" > output.txt       # 创建或覆盖文件
ls /tmp > filelist.txt

# >> 追加写入
echo "World" >> output.txt      # 追加到文件末尾

# 2> 错误重定向（stderr → 文件）
ls /nonexistent 2> error.log

# 2>> 错误追加
ls /nonexistent 2>> error.log

# &> 或 >& 同时重定向 stdout 和 stderr
command &> all_output.log          # Bash 推荐写法
command > all_output.log 2>&1      # POSIX 兼容写法

# 分别重定向
command > stdout.log 2> stderr.log
```

### 2.2 /dev/null — 黑洞设备

```bash
# 丢弃标准输出
command > /dev/null

# 丢弃错误输出
command 2> /dev/null

# 丢弃所有输出
command &> /dev/null
# 或
command > /dev/null 2>&1

# 常用场景：只关心命令是否成功
if command &> /dev/null; then
    echo "命令执行成功"
fi

# 判断命令是否存在
if command -v docker &> /dev/null; then
    echo "Docker 已安装"
else
    echo "Docker 未安装"
fi
```

### 2.3 重定向的执行顺序

```bash
# ⚠️ 顺序很重要！
command > file 2>&1     # ✓ 先将 stdout 指向 file，再将 stderr 指向 stdout（即 file）
command 2>&1 > file     # ✗ 先将 stderr 指向 stdout（屏幕），再将 stdout 指向 file
                        #   结果：stdout → file，stderr → 屏幕
```

## 3. 输入重定向

```bash
# < 从文件读取输入
wc -l < /etc/passwd           # 统计行数
sort < unsorted.txt           # 排序文件内容

# << Here Document（多行输入）
cat << EOF
这是第一行
变量替换: $USER
命令替换: $(date)
这是最后一行
EOF

# << 'EOF' 单引号禁止替换
cat << 'EOF'
这里 $USER 不会被替换
$(date) 也不会执行
EOF

# <<- 忽略行首 Tab（方便缩进）
if true; then
    cat <<- EOF
	这行前面是 Tab，会被忽略
	整齐的输出
	EOF
fi

# <<< Here String（单行输入）
grep "root" <<< "root:x:0:0:root"
read -r FIRST SECOND <<< "Hello World"
echo "$FIRST"    # Hello
echo "$SECOND"   # World

# 实际应用：生成配置文件
cat > /tmp/nginx.conf << 'EOF'
server {
    listen 80;
    server_name example.com;
    root /var/www/html;
}
EOF
```

## 4. 管道 (Pipe)

管道将一个命令的 stdout 连接到下一个命令的 stdin。

```bash
# 基本管道
ls -la | grep ".sh"
cat /etc/passwd | sort | head -5

# 多级管道
ps aux | grep nginx | grep -v grep | awk '{print $2}'

# 常用管道组合
# 统计日志中各 IP 的访问次数，取前 10
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -10

# 查看磁盘使用 top 10 目录
du -sh /* 2>/dev/null | sort -rh | head -10

# 查找最大的 10 个文件
find / -type f -exec du -h {} + 2>/dev/null | sort -rh | head -10
```

### 管道与重定向的组合

```bash
# 管道只传递 stdout，不传递 stderr
ls /tmp /nonexistent 2>/dev/null | wc -l    # 只统计成功输出的行数

# 如果需要 stderr 也进入管道
ls /tmp /nonexistent 2>&1 | grep "No such"

# 管道到 tee：同时输出到屏幕和文件
echo "重要信息" | tee output.log             # 覆盖
echo "更多信息" | tee -a output.log          # 追加

# tee 到多个文件
echo "数据" | tee file1.txt file2.txt file3.txt

# 管道 + tee 实用场景
./deploy.sh 2>&1 | tee deploy.log    # 部署日志同时显示和保存
```

## 5. 高级重定向

### 5.1 自定义文件描述符

```bash
# 打开文件描述符 3 用于写入
exec 3> output.txt
echo "写入到 FD 3" >&3
echo "继续写入" >&3
exec 3>&-                    # 关闭文件描述符 3

# 打开文件描述符 4 用于读取
exec 4< /etc/passwd
while read -r line <&4; do
    echo "读取: $line"
done
exec 4<&-                    # 关闭

# 同时打开读写
exec 5<> tmpfile.txt
echo "Hello" >&5
cat <&5
exec 5>&-
```

### 5.2 交换 stdout 和 stderr

```bash
# 交换 stdout 和 stderr
command 3>&1 1>&2 2>&3 3>&-

# 实用场景：让 stderr 进入管道，stdout 保持输出到屏幕
command 2>&1 1>/dev/tty | grep "ERROR"
```

### 5.3 进程替换

```bash
# <(command) 将命令输出当作文件来读取
diff <(ls dir1) <(ls dir2)           # 比较两个目录的文件列表
diff <(sort file1) <(sort file2)     # 比较两个排序后的文件

# >(command) 将文件写入当作命令输入
# 同时将日志输出到两个不同的处理程序
tee >(grep ERROR > errors.log) >(grep WARN > warnings.log) < app.log > /dev/null

# 对比：不用进程替换时需要临时文件
sort file1 > /tmp/sorted1
sort file2 > /tmp/sorted2
diff /tmp/sorted1 /tmp/sorted2
rm /tmp/sorted1 /tmp/sorted2
```

## 6. read 命令 — 交互式输入

```bash
# 基本读取
echo -n "请输入你的名字: "
read -r NAME
echo "你好, $NAME"

# -p 提示符
read -rp "请输入用户名: " USERNAME

# -s 静默模式（输入密码）
read -rsp "请输入密码: " PASSWORD
echo ""    # 换行

# -t 超时
read -rp "5秒内输入 (y/n): " -t 5 ANSWER || true
echo "${ANSWER:-超时未输入}"

# -n 指定读取字符数
read -rn 1 -p "按任意键继续..." KEY
echo ""

# 读取到数组
read -ra WORDS <<< "one two three"
echo "${WORDS[0]}"    # one
echo "${WORDS[1]}"    # two

# 按行读取文件
while IFS= read -r line; do
    echo "行内容: $line"
done < /etc/hostname

# 以特定分隔符读取
while IFS=: read -r user _ uid gid _ home shell; do
    echo "用户: $user, UID: $uid, 主目录: $home"
done < /etc/passwd
```

## 7. 实战练习

### 练习 1：日志分割器

```bash
#!/usr/bin/env bash
# log_splitter.sh — 将混合日志按级别分割到不同文件

set -euo pipefail

INPUT="${1:?用法: $0 <日志文件>}"
OUTPUT_DIR="/tmp/split_logs"
mkdir -p "$OUTPUT_DIR"

# 使用 tee + 进程替换同时分流
cat "$INPUT" | tee \
    >(grep "ERROR" > "${OUTPUT_DIR}/error.log") \
    >(grep "WARN"  > "${OUTPUT_DIR}/warn.log") \
    >(grep "INFO"  > "${OUTPUT_DIR}/info.log") \
    > /dev/null

echo "日志分割完成:"
for f in "${OUTPUT_DIR}"/*.log; do
    echo "  $(basename "$f"): $(wc -l < "$f") 行"
done
```

### 练习 2：多行配置生成器

```bash
#!/usr/bin/env bash
# gen_config.sh — 根据输入生成 Nginx 虚拟主机配置

set -euo pipefail

read -rp "域名: " DOMAIN
read -rp "端口 [80]: " PORT
PORT="${PORT:-80}"
read -rp "根目录 [/var/www/${DOMAIN}]: " ROOT
ROOT="${ROOT:-/var/www/${DOMAIN}}"

CONFIG_FILE="/tmp/${DOMAIN}.conf"

cat > "$CONFIG_FILE" << EOF
server {
    listen ${PORT};
    server_name ${DOMAIN} www.${DOMAIN};

    root ${ROOT};
    index index.html index.htm;

    access_log /var/log/nginx/${DOMAIN}_access.log;
    error_log  /var/log/nginx/${DOMAIN}_error.log;

    location / {
        try_files \$uri \$uri/ =404;
    }
}
EOF

echo "配置已生成: $CONFIG_FILE"
cat "$CONFIG_FILE"
```

## 8. 小结

| 知识点 | 要点 |
|--------|------|
| 文件描述符 | 0=stdin, 1=stdout, 2=stderr |
| 输出重定向 | `>` 覆盖, `>>` 追加, `2>` 错误, `&>` 全部 |
| 输入重定向 | `<` 文件, `<<` Here Document, `<<<` Here String |
| 管道 | `cmd1 \| cmd2` stdout → stdin |
| tee | 同时输出到屏幕和文件 |
| /dev/null | 丢弃不需要的输出 |
| 进程替换 | `<(cmd)` `>(cmd)` 避免临时文件 |
