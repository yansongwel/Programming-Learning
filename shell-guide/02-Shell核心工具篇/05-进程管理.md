# 进程管理

> 一句话说明：掌握进程的查看、监控、控制和管理，是系统运维的基本功。

---

## 1. 查看进程

```bash
# ps — 进程快照
ps aux                           # BSD 风格，显示所有进程
ps -ef                           # System V 风格
ps aux --sort=-%mem | head -10   # 按内存排序 Top 10
ps aux --sort=-%cpu | head -10   # 按 CPU 排序 Top 10
ps -eo pid,user,%cpu,%mem,cmd --sort=-%cpu | head -20

# 查看进程树
pstree
pstree -p                       # 显示 PID
pstree -u username               # 指定用户

# top — 实时监控
top
# 快捷键：M(按内存排序) P(按CPU排序) k(杀进程) q(退出) 1(显示各CPU)

# htop — 增强版 top（推荐安装）
htop
```

## 2. 进程控制

### 2.1 前后台管理

```bash
# 后台运行
command &                         # 在后台运行
nohup command &                   # 后台运行 + 免疫 HUP 信号（关闭终端不受影响）
nohup command > output.log 2>&1 & # 标准用法

# 查看后台任务
jobs                              # 查看当前 Shell 的后台任务
jobs -l                           # 显示 PID

# 前后台切换
Ctrl+Z                            # 挂起当前前台进程（SIGTSTP）
bg %1                             # 将任务 1 放到后台继续运行
fg %1                             # 将任务 1 调到前台

# disown — 将后台任务与 Shell 脱离
command &
disown %1                        # 关闭 Shell 后进程不会被杀
```

### 2.2 信号发送

```bash
# kill — 发送信号
kill PID                          # 默认发送 SIGTERM (15)
kill -9 PID                       # 强制杀死 SIGKILL
kill -HUP PID                     # 重载配置 SIGHUP
kill -STOP PID                    # 暂停进程
kill -CONT PID                    # 继续进程

# 常用信号
# SIGTERM (15) — 优雅终止（推荐首选）
# SIGKILL (9)  — 强制杀死（最后手段）
# SIGHUP  (1)  — 重载配置
# SIGSTOP (19) — 暂停进程
# SIGCONT (18) — 继续进程

# killall — 按名称杀进程
killall nginx

# pkill — 按模式杀进程
pkill -f "python app.py"
pkill -u username                 # 杀掉某用户所有进程

# 安全停止进程的流程
kill PID                          # 1. 先发 SIGTERM
sleep 5                           # 2. 等待进程清理
kill -0 PID 2>/dev/null && kill -9 PID   # 3. 还活着就强杀
```

## 3. 进程信息

```bash
# /proc 文件系统
cat /proc/PID/status              # 进程状态
cat /proc/PID/cmdline | tr '\0' ' '  # 完整命令行
ls -la /proc/PID/fd               # 打开的文件描述符
cat /proc/PID/environ | tr '\0' '\n' # 环境变量

# lsof — 查看文件和端口占用
lsof -i :80                      # 谁在用 80 端口
lsof -u username                  # 用户打开的文件
lsof -p PID                      # 进程打开的文件
lsof +D /var/log                  # 谁在读写这个目录

# /proc 全局信息
cat /proc/loadavg                 # 负载
cat /proc/meminfo                 # 内存
cat /proc/cpuinfo                 # CPU
uptime                            # 运行时间和负载
```

## 4. 资源限制

```bash
# ulimit — 查看和设置资源限制
ulimit -a                         # 查看所有限制
ulimit -n                         # 最大打开文件数
ulimit -u                         # 最大进程数
ulimit -n 65535                   # 设置最大打开文件数（当前 Shell）

# 永久设置 (/etc/security/limits.conf)
# nginx  soft  nofile  65535
# nginx  hard  nofile  65535

# cgroups — 资源限制（现代方式）
# systemctl set-property myservice MemoryMax=512M
# systemctl set-property myservice CPUQuota=50%
```

## 5. 实战：进程监控脚本

```bash
#!/usr/bin/env bash
# proc_monitor.sh — 监控指定进程

set -euo pipefail

PROCESS="${1:?用法: $0 <进程名>}"

echo "监控进程: $PROCESS"
echo "时间                CPU%  MEM%  RSS(MB)  PID"
echo "----------------------------------------------"

while true; do
    LINE=$(ps aux | grep "[${PROCESS:0:1}]${PROCESS:1}" | head -1)
    if [[ -n "$LINE" ]]; then
        PID=$(echo "$LINE" | awk '{print $2}')
        CPU=$(echo "$LINE" | awk '{print $3}')
        MEM=$(echo "$LINE" | awk '{print $4}')
        RSS=$(echo "$LINE" | awk '{printf "%.1f", $6/1024}')
        printf "%s  %5s  %5s  %7s  %s\n" "$(date '+%H:%M:%S')" "$CPU" "$MEM" "$RSS" "$PID"
    else
        echo "$(date '+%H:%M:%S')  进程未运行!"
    fi
    sleep 5
done
```

## 6. 小结

| 操作 | 命令 |
|------|------|
| 查看进程 | `ps aux`, `top`, `htop` |
| 后台运行 | `nohup cmd &`, `disown` |
| 杀进程 | `kill`, `killall`, `pkill` |
| 端口/文件占用 | `lsof -i :PORT`, `ss -tlnp` |
| 进程详情 | `/proc/PID/`, `lsof -p PID` |
| 资源限制 | `ulimit`, cgroups |
