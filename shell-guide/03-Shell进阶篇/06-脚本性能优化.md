# 脚本性能优化

> 一句话说明：掌握 Shell 脚本的性能瓶颈分析和优化技巧，让脚本处理大数据量时也能高效运行。

---

## 1. 性能分析

```bash
# time 命令计时
time ./script.sh
# real    0m5.123s    # 实际耗时
# user    0m3.456s    # 用户态 CPU 时间
# sys     0m0.789s    # 内核态 CPU 时间

# 逐行分析
PS4='+ $(date +%s.%N) ${BASH_SOURCE}:${LINENO}: '
set -x
# 每行命令前都会打印时间戳

# 分段计时
start=$(date +%s%N)
# ... 操作 ...
end=$(date +%s%N)
echo "耗时: $(( (end - start) / 1000000 ))ms"
```

## 2. 核心优化原则

### 2.1 减少进程创建

```bash
# ❌ 每次循环创建多个进程
while read -r line; do
    result=$(echo "$line" | grep "pattern" | awk '{print $1}')
done < file

# ✅ 用 Bash 内置功能替代外部命令
while read -r line; do
    if [[ "$line" == *pattern* ]]; then
        result="${line%% *}"
    fi
done < file

# ❌ 不必要的 cat
cat file | grep "pattern"

# ✅ 直接输入重定向
grep "pattern" file
grep "pattern" < file
```

### 2.2 用内置替代外部命令

```bash
# 字符串操作
# ❌ 外部命令
basename="/path/to/file.txt"
dir=$(dirname "$basename")
name=$(basename "$basename")
ext=$(echo "$basename" | sed 's/.*\.//')

# ✅ 参数展开
dir="${basename%/*}"
name="${basename##*/}"
ext="${basename##*.}"

# 数学运算
# ❌
result=$(expr $a + $b)
# ✅
result=$((a + b))

# 字符串替换
# ❌
echo "$str" | tr 'a-z' 'A-Z'
echo "$str" | sed 's/old/new/g'
# ✅
echo "${str^^}"
echo "${str//old/new}"
```

### 2.3 一次性处理 vs 逐行循环

```bash
# ❌ 逐行处理（每行创建进程）
while read -r line; do
    echo "$line" | grep -c "error"
done < large_file.log

# ✅ 一次性处理
grep -c "error" large_file.log

# ❌ 循环中调用 awk
while read -r line; do
    ip=$(echo "$line" | awk '{print $1}')
    code=$(echo "$line" | awk '{print $9}')
done < access.log

# ✅ 一个 awk 完成所有工作
awk '{print $1, $9}' access.log | while read -r ip code; do
    # 处理
    :
done
```

## 3. 并行处理

```bash
# xargs 并行
find . -name "*.gz" | xargs -P 4 -I {} gunzip {}

# 后台进程并行
MAX_JOBS=4

for task in "${TASKS[@]}"; do
    while (( $(jobs -rp | wc -l) >= MAX_JOBS )); do
        wait -n    # 等待任意一个完成（Bash 4.3+）
    done
    process "$task" &
done
wait

# GNU parallel（推荐）
cat servers.txt | parallel -j 4 "ssh {} 'uptime'"
find . -name "*.log" | parallel -j 8 gzip
```

## 4. I/O 优化

```bash
# 减少文件写入次数
# ❌
for i in {1..1000}; do
    echo "line $i" >> output.txt    # 1000 次 open/write/close
done

# ✅ 重定向整个循环
{
    for i in {1..1000}; do
        echo "line $i"
    done
} > output.txt                       # 1 次 open，1000 次 write，1 次 close

# ✅ 或用 printf 一次生成
printf "line %d\n" {1..1000} > output.txt

# 利用 /dev/shm（内存文件系统）
TMPFILE="/dev/shm/tmp_$$"
# 比 /tmp 快得多（如果在磁盘上）
```

## 5. 实用对比

```bash
# 统计文件行数的方式对比
time wc -l < largefile           # 最快
time awk 'END{print NR}' largefile
time grep -c '' largefile
time cat largefile | wc -l       # 多余的 cat

# 大文件搜索
time grep "pattern" largefile          # grep 最快
time awk '/pattern/' largefile         # awk 稍慢
time sed -n '/pattern/p' largefile     # sed 稍慢
time while read -r l; do [[ "$l" == *pattern* ]] && echo "$l"; done < largefile  # 最慢
```

## 6. 小结

| 优化方向 | 方法 |
|----------|------|
| 减少进程 | 用内置功能替代外部命令 |
| 批量处理 | 用 grep/awk/sed 一次处理整个文件 |
| 并行化 | `xargs -P`、后台进程、`parallel` |
| I/O 优化 | 重定向整个循环、用内存文件系统 |
| 选择工具 | 简单搜索用 grep，列处理用 awk |
