# Python 日志体系详解 — 构建企业级 SRE 日志方案

## 1. 是什么 & 为什么用它

Python 的 `logging` 模块是标准库内置的日志框架，提供了灵活强大的日志系统。
对 SRE 来说，良好的日志体系是可观测性（Observability）的三大支柱之一：

- **标准库 logging**：四大组件（Logger/Handler/Formatter/Filter），企业级日志的基础
- **loguru**：第三方库，API 更简洁，开箱即用
- **structlog**：结构化日志，适合 ELK/日志分析场景
- **日志轮转**：防止日志撑满磁盘的关键策略

为什么不用 `print()`：
- 无级别控制（不能按 DEBUG/INFO/WARNING 过滤）
- 无输出目标管理（不能同时输出到文件和控制台）
- 无格式化（时间戳、模块名等）
- 无轮转能力

## 3. 核心概念与原理

### 3.1 logging 四大组件

```
Logger(记录器)
  ├── 负责产生日志记录
  ├── 有层级关系（parent/child）
  └── 名称用 '.' 分隔层级：'app.db.query'

Handler(处理器)
  ├── 负责将日志发送到目标
  ├── StreamHandler → 控制台
  ├── FileHandler → 文件
  ├── RotatingFileHandler → 按大小轮转
  ├── TimedRotatingFileHandler → 按时间轮转
  ├── SocketHandler → 网络
  └── SysLogHandler → syslog

Formatter(格式器)
  ├── 定义日志的输出格式
  └── 时间、级别、模块、消息、异常等

Filter(过滤器)
  ├── 精细控制哪些日志被处理
  └── 可基于级别、模块、内容等过滤
```

### 3.2 日志级别

| 级别 | 数值 | 用途 |
|------|------|------|
| DEBUG | 10 | 调试信息，生产环境关闭 |
| INFO | 20 | 常规信息，确认运行正常 |
| WARNING | 30 | 警告，可能出问题 |
| ERROR | 40 | 错误，功能受损 |
| CRITICAL | 50 | 严重错误，系统可能崩溃 |

### 3.3 日志流转过程

```
logger.info("消息")
  → Logger 级别检查（够级别？）
    → Logger Filter 检查
      → 传递给 Handler
        → Handler 级别检查
          → Handler Filter 检查
            → Formatter 格式化
              → 输出到目标
  → 传播给 parent Logger（propagate=True 时）
```

### 4.1 logging 基础

```python
#!/usr/bin/env python3
"""logging 模块基础用法"""

import logging

# === 最简单的用法（不推荐用于生产） ===
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s [%(levelname)s] %(name)s: %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S'
)

# 获取 logger
logger = logging.getLogger('sre.demo')

# 不同级别的日志
logger.debug("这条不会显示，因为级别是 INFO")

logger.warning("连接池使用率达到 80%")
logger.error("数据库连接失败: Connection refused")
logger.critical("磁盘空间不足，系统即将停止")

# 带格式化参数（推荐用 % 风格，延迟求值）
logger.info("请求处理完成: method=%s path=%s status=%d latency=%.2fms",
            "GET", "/api/health", 200, 1.23)

# 记录异常信息
try:
    result = 1 / 0
except ZeroDivisionError:
    logger.exception("计算错误")  # 自动附加堆栈信息
```

### 4.2 完整的 Logger 配置

```python
#!/usr/bin/env python3
"""完整的 logging 配置示例"""

import logging
import logging.handlers
from pathlib import Path
import sys

def setup_logging(
    log_dir: str = "/tmp/sre-logs",
    log_level: str = "INFO",
    app_name: str = "sre-app"
):
    """配置企业级日志系统"""
    # 创建日志目录
    Path(log_dir).mkdir(parents=True, exist_ok=True)

    # 获取根 logger
    root_logger = logging.getLogger()
    root_logger.setLevel(getattr(logging, log_level.upper()))

    # === 格式器 ===
    # 详细格式（文件用）
    detailed_formatter = logging.Formatter(
        fmt='%(asctime)s [%(levelname)-8s] %(name)s:%(funcName)s:%(lineno)d - %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S'
    )
    # 简洁格式（控制台用）
    simple_formatter = logging.Formatter(
        fmt='%(asctime)s [%(levelname)s] %(message)s',
        datefmt='%H:%M:%S'
    )

    # === Handler 1: 控制台 ===
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setLevel(logging.INFO)
    console_handler.setFormatter(simple_formatter)

    # === Handler 2: 按大小轮转的文件 ===
    # 每个文件最大 10MB，保留 5 个备份
    file_handler = logging.handlers.RotatingFileHandler(
        filename=f"{log_dir}/{app_name}.log",
        maxBytes=10 * 1024 * 1024,  # 10MB
        backupCount=5,
        encoding='utf-8'
    )
    file_handler.setLevel(logging.DEBUG)
    file_handler.setFormatter(detailed_formatter)

    # === Handler 3: 按时间轮转的错误日志 ===
    # 每天轮转，保留 30 天
    error_handler = logging.handlers.TimedRotatingFileHandler(
        filename=f"{log_dir}/{app_name}.error.log",
        when='midnight',    # 每天午夜轮转
        interval=1,
        backupCount=30,     # 保留 30 天
        encoding='utf-8'
    )
    error_handler.setLevel(logging.ERROR)
    error_handler.setFormatter(detailed_formatter)

    # === 自定义 Filter ===
    class SensitiveFilter(logging.Filter):
        """过滤敏感信息（密码、Token 等）"""
        SENSITIVE_PATTERNS = ['password', 'token', 'secret', 'key']

        def filter(self, record):
            msg = record.getMessage().lower()
            for pattern in self.SENSITIVE_PATTERNS:
                if pattern in msg:
                    record.msg = record.msg + " [包含敏感信息，已标记]"
            return True  # True 表示保留该日志

    # 添加过滤器
    file_handler.addFilter(SensitiveFilter())

    # 清除已有 handler（避免重复添加）
    root_logger.handlers.clear()

    # 添加所有 handler
    root_logger.addHandler(console_handler)
    root_logger.addHandler(file_handler)
    root_logger.addHandler(error_handler)

    return root_logger

# === 使用示例 ===
setup_logging(log_level="DEBUG")

# 不同模块的 logger
app_logger = logging.getLogger('sre.app')
db_logger = logging.getLogger('sre.db')
api_logger = logging.getLogger('sre.api')

app_logger.info("应用启动")
db_logger.warning("数据库慢查询: 耗时 2.5s")
api_logger.error("API 调用失败: HTTP 503")
app_logger.debug("调试信息: password=xxx")  # 会被 Filter 标记
```

### 5.1 loguru — 更简洁的日志库

```python
#!/usr/bin/env python3
"""loguru — 开箱即用的现代日志库"""

from loguru import logger
import sys

# 移除默认 handler
logger.remove()

# 添加控制台输出（带颜色）
logger.add(
    sys.stderr,
    level="INFO",
    format="<green>{time:HH:mm:ss}</green> | <level>{level: <8}</level> | <cyan>{name}</cyan>:<cyan>{function}</cyan>:<cyan>{line}</cyan> - <level>{message}</level>",
    colorize=True,
)

# 添加文件输出（自动轮转 + 压缩）
logger.add(
    "/tmp/sre-logs/app_{time:YYYY-MM-DD}.log",
    rotation="100 MB",     # 100MB 轮转
    retention="30 days",   # 保留 30 天
    compression="gz",      # 旧日志压缩
    encoding="utf-8",
    level="DEBUG",
    format="{time:YYYY-MM-DD HH:mm:ss} | {level: <8} | {name}:{function}:{line} - {message}",
)

# 使用非常简单
logger.info("服务启动")
logger.warning("内存使用率: {percent}%", percent=85.5)
logger.error("连接超时: {host}:{port}", host="10.0.0.1", port=5432)

# 异常自动捕获

    return 1 / 0

try:
    risky_function()
except:
    pass

    logger.info("处理请求")
    logger.info("请求完成")
```

### 5.2 structlog — 结构化日志

```python
#!/usr/bin/env python3
"""structlog — 结构化日志，适合 ELK 场景"""

import structlog
import logging

# 配置 structlog
structlog.configure(
    processors=[
        structlog.contextvars.merge_contextvars,           # 合并上下文变量
        structlog.processors.add_log_level,                 # 添加日志级别
        structlog.processors.TimeStamper(fmt="iso"),       # ISO 时间戳
        structlog.processors.StackInfoRenderer(),           # 堆栈信息
        structlog.processors.format_exc_info,              # 异常信息
        structlog.processors.JSONRenderer(ensure_ascii=False),  # JSON 输出
    ],
    context_class=dict,
    logger_factory=structlog.PrintLoggerFactory(),
    wrapper_class=structlog.make_filtering_bound_logger(logging.INFO),
    cache_logger_on_first_use=True,
)

# 获取结构化 logger
log = structlog.get_logger()

# 基础使用 — 自动生成 JSON
log.info("服务启动", host="0.0.0.0", port=8080, workers=4)
# {"host": "0.0.0.0", "port": 8080, "workers": 4, "event": "服务启动", "log_level": "info", "timestamp": "2024-01-01T00:00:00Z"}

# 绑定上下文
request_log = log.bind(request_id="req-001", client_ip="192.168.1.100")
request_log.info("收到请求", method="GET", path="/api/health")
request_log.info("请求完成", status=200, latency_ms=12.5)
# 每条日志都会自动带上 request_id 和 client_ip

# 异常记录
try:
    raise ConnectionError("数据库连接失败")
except Exception:
    log.exception("处理异常", db_host="10.0.0.1")
```

## 6. SRE 实战案例：企业级日志方案设计

```python
#!/usr/bin/env python3
"""
SRE 实战：企业级日志方案
功能：
1. 分层日志（应用日志/审计日志/性能日志）
2. 结构化 JSON 格式（ELK 友好）
3. 自动轮转与清理
4. 敏感信息脱敏
5. 请求链路追踪
6. 日志指标统计
"""

import logging
import logging.handlers
import json
import re
import time
import uuid
import threading
from pathlib import Path
from datetime import datetime, timezone
from contextvars import ContextVar
from typing import Optional
from collections import defaultdict

# ============================================================
# 请求上下文（链路追踪）
# ============================================================
_request_id: ContextVar[str] = ContextVar('request_id', default='')
_trace_id: ContextVar[str] = ContextVar('trace_id', default='')

def set_request_context(request_id: str = None, trace_id: str = None):
    """设置请求上下文"""
    _request_id.set(request_id or str(uuid.uuid4())[:8])
    _trace_id.set(trace_id or str(uuid.uuid4())[:16])

def get_request_context() -> dict:
    """获取请求上下文"""
    return {
        'request_id': _request_id.get(''),
        'trace_id': _trace_id.get(''),
    }

# ============================================================
# 自定义 Formatter：JSON 结构化输出
# ============================================================
class JSONFormatter(logging.Formatter):
    """JSON 格式化器，输出结构化日志"""

    SENSITIVE_PATTERNS = [
        (re.compile(r'(password|passwd|pwd)\s*[=:]\s*\S+', re.I), r'\1=***'),
        (re.compile(r'(token|secret|key)\s*[=:]\s*\S+', re.I), r'\1=***'),
        (re.compile(r'\b\d{16,19}\b'), '****CARD****'),  # 银行卡号
    ]

    def __init__(self, service_name: str = "sre-app", env: str = "production"):
        super().__init__()
        self.service_name = service_name
        self.env = env

    def _sanitize(self, message: str) -> str:
        """脱敏处理"""
        for pattern, replacement in self.SENSITIVE_PATTERNS:
            message = pattern.sub(replacement, message)
        return message

    def format(self, record: logging.LogRecord) -> str:
        """格式化为 JSON"""
        ctx = get_request_context()
        log_data = {
            "@timestamp": datetime.now(timezone.utc).isoformat(),
            "level": record.levelname,
            "logger": record.name,
            "message": self._sanitize(record.getMessage()),
            "service": self.service_name,
            "env": self.env,
            "module": record.module,
            "function": record.funcName,
            "line": record.lineno,
            "thread": record.thread,
            "process": record.process,
        }

        # 添加请求上下文
        if ctx['request_id']:
            log_data['request_id'] = ctx['request_id']
        if ctx['trace_id']:
            log_data['trace_id'] = ctx['trace_id']

        # 添加额外字段
        if hasattr(record, 'extra_data'):
            log_data['data'] = record.extra_data

        # 添加异常信息
        if record.exc_info and record.exc_info[0]:
            log_data['exception'] = {
                'type': record.exc_info[0].__name__,
                'message': str(record.exc_info[1]),
                'stacktrace': self.formatException(record.exc_info),
            }

        return json.dumps(log_data, ensure_ascii=False)

# ============================================================
# 日志统计 Handler
# ============================================================
class MetricsHandler(logging.Handler):
    """统计日志数量的 Handler，可对接 Prometheus"""

    def __init__(self):
        super().__init__()
        self._lock = threading.Lock()
        self.counters = defaultdict(int)

    def emit(self, record):
        with self._lock:
            self.counters[record.levelname] += 1
            self.counters[f"{record.name}.{record.levelname}"] += 1

    def get_metrics(self) -> dict:
        """获取日志指标"""
        with self._lock:
            return dict(self.counters)

    def reset(self):
        """重置计数器"""
        with self._lock:
            self.counters.clear()

# ============================================================
# 日志系统初始化
# ============================================================
class SRELogSystem:
    """企业级日志系统"""

    def __init__(self, config: dict):
        self.config = config
        self.metrics_handler = MetricsHandler()
        self._setup()

    def _setup(self):
        """初始化日志系统"""
        log_dir = Path(self.config.get('log_dir', '/tmp/sre-logs'))
        log_dir.mkdir(parents=True, exist_ok=True)

        service_name = self.config.get('service_name', 'sre-app')
        env = self.config.get('env', 'production')

        # JSON 格式器
        json_formatter = JSONFormatter(service_name=service_name, env=env)

        # 简洁控制台格式器
        console_formatter = logging.Formatter(
            '%(asctime)s [%(levelname)-8s] %(name)s - %(message)s',
            datefmt='%H:%M:%S'
        )

        root = logging.getLogger()
        root.setLevel(logging.DEBUG)
        root.handlers.clear()

        # Handler 1: 控制台
        console = logging.StreamHandler()
        console.setLevel(logging.INFO)
        console.setFormatter(console_formatter)
        root.addHandler(console)

        # Handler 2: 应用日志（JSON，按大小轮转）
        app_handler = logging.handlers.RotatingFileHandler(
            filename=str(log_dir / f"{service_name}.json.log"),
            maxBytes=50 * 1024 * 1024,  # 50MB
            backupCount=10,
            encoding='utf-8'
        )
        app_handler.setLevel(logging.DEBUG)
        app_handler.setFormatter(json_formatter)
        root.addHandler(app_handler)

        # Handler 3: 错误日志（按时间轮转）
        error_handler = logging.handlers.TimedRotatingFileHandler(
            filename=str(log_dir / f"{service_name}.error.log"),
            when='midnight',
            backupCount=90,  # 保留 90 天
            encoding='utf-8'
        )
        error_handler.setLevel(logging.ERROR)
        error_handler.setFormatter(json_formatter)
        root.addHandler(error_handler)

        # Handler 4: 指标统计
        root.addHandler(self.metrics_handler)

        # 设置第三方库日志级别
        logging.getLogger('urllib3').setLevel(logging.WARNING)
        logging.getLogger('requests').setLevel(logging.WARNING)

    def get_metrics(self) -> dict:
        """获取日志指标"""
        return self.metrics_handler.get_metrics()

# ============================================================
# 辅助日志函数
# ============================================================
class SRELogger:
    """SRE 增强 Logger"""

    def __init__(self, name: str):
        self.logger = logging.getLogger(name)

    def log_with_data(self, level: int, message: str, **kwargs):
        """带额外数据的日志"""
        record = self.logger.makeRecord(
            self.logger.name, level, "(unknown)", 0, message, (), None
        )
        record.extra_data = kwargs
        self.logger.handle(record)

    def audit(self, action: str, user: str, resource: str, result: str, **kwargs):
        """审计日志"""
        self.log_with_data(
            logging.INFO,
            f"审计: {user} {action} {resource} -> {result}",
            audit_action=action,
            audit_user=user,
            audit_resource=resource,
            audit_result=result,
            **kwargs
        )

    def perf(self, operation: str, latency_ms: float, **kwargs):
        """性能日志"""
        level = logging.WARNING if latency_ms > 1000 else logging.INFO
        self.log_with_data(
            level,
            f"性能: {operation} 耗时 {latency_ms:.2f}ms",
            perf_operation=operation,
            perf_latency_ms=latency_ms,
            **kwargs
        )

# ============================================================
# 演示
# ============================================================
def demo():
    """演示企业级日志系统"""
    # 初始化
    log_system = SRELogSystem({
        'service_name': 'sre-platform',
        'env': 'production',
        'log_dir': '/tmp/sre-logs',
    })

    sre_log = SRELogger('sre.api')
    logger = logging.getLogger('sre.api')

    # 模拟请求处理
    set_request_context()
    logger.info("收到 HTTP 请求: GET /api/servers")

    # 普通日志
    logger.info("查询服务器列表: 共 42 台")
    logger.warning("服务器 web-03 CPU 使用率 95%")

    # 敏感信息自动脱敏
    logger.info("用户登录: user=admin password=secret123")

    # 审计日志
    sre_log.audit("DELETE", "admin", "server/web-05", "success")

    # 性能日志
    sre_log.perf("db_query", 1523.5, query="SELECT * FROM servers")
    sre_log.perf("api_call", 45.2, endpoint="/api/health")

    # 异常日志
    try:
        raise ConnectionError("Redis 连接超时: 10.0.0.5:6379")
    except Exception:
        logger.exception("缓存服务不可用")

    # 打印日志指标
    metrics = log_system.get_metrics()
    print("\n=== 日志指标统计 ===")
    for key, count in sorted(metrics.items()):
        print(f"  {key}: {count}")

if __name__ == '__main__':
    demo()
```

## 7. 常见坑点与最佳实践

### 坑点

1. **重复日志**：`propagate=True`（默认）会导致日志向上传播，父子 Logger 都输出
2. **basicConfig 只能调一次**：第二次调用无效，用 `dictConfig` 代替
3. **多进程日志**：`RotatingFileHandler` 在多进程下不安全，需用 `QueueHandler`
4. **性能陷阱**：`logger.debug(f"data={expensive()}")` 即使不输出也会执行表达式
5. **日志丢失**：程序崩溃时缓冲区未刷新，使用 `flush()` 或配置 `flushLevel`

### 最佳实践

## 8. 速查表

```python
# === 基础 ===
import logging
logger = logging.getLogger(__name__)
logger.setLevel(logging.DEBUG)
logger.info("消息: %s", var)
logger.exception("异常")           # 自动附加堆栈

# === Handler ===
logging.StreamHandler()                                    # 控制台
logging.FileHandler('app.log')                            # 文件
logging.handlers.RotatingFileHandler('app.log', maxBytes=10*1024*1024, backupCount=5)
logging.handlers.TimedRotatingFileHandler('app.log', when='midnight', backupCount=30)
logging.handlers.SysLogHandler(address='/dev/log')        # syslog
logging.handlers.SocketHandler('localhost', 9999)         # TCP
logging.handlers.QueueHandler(queue)                      # 队列（多进程安全）

# === Formatter ===
fmt = '%(asctime)s [%(levelname)s] %(name)s:%(lineno)d - %(message)s'
logging.Formatter(fmt, datefmt='%Y-%m-%d %H:%M:%S')

# === loguru ===
from loguru import logger
logger.add("file.log", rotation="100 MB", retention="30 days", compression="gz")
logger.info("消息: {key}", key="value")
logger.bind(request_id="123").info("带上下文")

# === structlog ===
import structlog
log = structlog.get_logger()
log.info("事件", key1="val1", key2="val2")  # → JSON
```
