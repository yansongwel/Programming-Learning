## 1. 是什么 & 为什么用它

### Shell 运维的痛点

```bash
# Shell 脚本通常直接依赖系统环境
#!/bin/bash
# 不同机器上 bash 版本不同，行为可能不一致
# 依赖的命令（jq, curl, awk）版本各异
# 没有统一的包管理，全靠 yum/apt 装系统包
```

### Python 的优势

- **版本管理**：pyenv 可以在同一台机器上安装多个 Python 版本，互不干扰
- **虚拟环境**：venv 让每个项目有独立的依赖，不会污染系统环境
- **包管理**：pip + requirements.txt 实现依赖锁定，跨机器可复现
- **跨平台**：同一份代码在 CentOS、Ubuntu、macOS 上均可运行

| 对比维度       | Shell 方式              | Python 方式                  |
|----------------|-------------------------|------------------------------|
| 版本管理       | 系统自带，难以切换      | pyenv 多版本并存             |
| 依赖隔离       | 无                      | venv 虚拟环境                |
| 包管理         | yum/apt（系统级）       | pip（项目级）                |
| 依赖锁定       | 无                      | requirements.txt / poetry    |
| 环境变量       | export PATH=...         | activate 脚本自动设置        |

## 2. Python3 安装

### 2.1 CentOS 7/8 安装

```bash
# ==========================================
# CentOS 7 安装 Python3（系统默认是 Python2）
# ==========================================

# 方法一：通过 EPEL 仓库安装（推荐快速上手）
sudo yum install -y epel-release
sudo yum install -y python3 python3-pip python3-devel

# 验证安装
python3 --version   # 输出类似 Python 3.6.8
pip3 --version      # 输出类似 pip 9.0.3

# 方法二：编译安装（需要特定版本时使用）
# 安装编译依赖
sudo yum groupinstall -y "Development Tools"
sudo yum install -y openssl-devel bzip2-devel libffi-devel \
    readline-devel sqlite-devel zlib-devel xz-devel

# 下载并编译 Python 3.11
cd /tmp
wget https://www.python.org/ftp/python/3.11.7/Python-3.11.7.tgz
tar xzf Python-3.11.7.tgz
cd Python-3.11.7
./configure --enable-optimizations --prefix=/usr/local
make -j$(nproc)
sudo make altinstall  # 注意：用 altinstall 而不是 install，避免覆盖系统 Python

# 验证
/usr/local/bin/python3.11 --version
```

## 3. pyenv 多版本管理

### 3.1 为什么需要 pyenv

```bash
# Shell 对比：想在不同项目中使用不同版本的工具
# Shell 中你可能这样做：
export PATH="/opt/python3.9/bin:$PATH"   # 手动切换，容易出错

# Python 中使用 pyenv：
pyenv install 3.9.18    # 安装 3.9
pyenv install 3.11.7    # 安装 3.11
pyenv local 3.11.7      # 当前目录使用 3.11
pyenv global 3.9.18     # 全局默认 3.9
```

### 3.2 安装 pyenv

```bash
# ==========================================
# 安装 pyenv（Linux / macOS 通用）
# ==========================================

# 方法一：使用官方安装脚本（推荐）
curl https://pyenv.run | bash

# 方法二：手动克隆（网络不好时使用）
git clone https://github.com/pyenv/pyenv.git ~/.pyenv

# 配置环境变量（添加到 ~/.bashrc 或 ~/.zshrc）
cat >> ~/.bashrc << 'EOF'
# pyenv 配置
export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init --path)"
eval "$(pyenv init -)"
EOF

# 使配置生效
source ~/.bashrc

# 验证安装
pyenv --version
```

### 3.3 pyenv 常用命令

```bash
# 查看可安装的 Python 版本
pyenv install --list | grep "^  3\."

# 安装指定版本
pyenv install 3.11.7

# 查看已安装的版本
pyenv versions

# 设置全局默认版本
pyenv global 3.11.7

# 设置当前目录的 Python 版本（会创建 .python-version 文件）
cd /home/devops/my-project
pyenv local 3.11.7

# 查看当前使用的版本
pyenv version

# 卸载某个版本
pyenv uninstall 3.9.18
```

### 3.4 pyenv 工作原理

```bash
# pyenv 通过修改 PATH 实现版本切换
# 它在 PATH 最前面插入 shims 目录
which python3
# 输出: /home/devops/.pyenv/shims/python3

# shims 是一个薄包装器，根据以下优先级选择版本：
# 1. PYENV_VERSION 环境变量
# 2. 当前目录的 .python-version 文件（pyenv local）
# 3. 上级目录的 .python-version 文件（递归查找）
# 4. ~/.pyenv/version 文件（pyenv global）
```

## 4. venv 虚拟环境

### 4.1 为什么需要虚拟环境

```bash
# Shell 对比：
# 在 Shell 中，所有脚本共享系统的命令和库
# 如果脚本 A 依赖 jq 1.5，脚本 B 依赖 jq 1.6，就会冲突

# Python 中：
# 项目 A 依赖 requests==2.28.0
# 项目 B 依赖 requests==2.31.0
# 使用 venv 可以让它们各自拥有独立的 requests 版本
```

### 4.2 创建和使用虚拟环境

```bash
# ==========================================
# 创建虚拟环境（最小可运行示例）
# ==========================================

# 创建项目目录
mkdir -p /home/devops/my-sre-tool
cd /home/devops/my-sre-tool

# 创建虚拟环境（在项目目录下创建 .venv 目录）
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 激活后，终端提示符会变化：
# (.venv) devops@hostname:~/my-sre-tool$

# 此时 python 和 pip 指向虚拟环境中的版本
which python    # /home/devops/my-sre-tool/.venv/bin/python
which pip       # /home/devops/my-sre-tool/.venv/bin/pip

# 安装包（只影响当前虚拟环境）
pip install requests psutil

# 退出虚拟环境
deactivate
```

### 4.3 venv 对比 Shell PATH

```bash
# Shell 中手动管理 PATH：
export PATH="/opt/my-tools/bin:$PATH"
# 问题：容易忘记恢复，影响全局

# venv 的 activate 脚本做了什么：
# 1. 保存原来的 PATH 到 _OLD_VIRTUAL_PATH
# 2. 把 .venv/bin 加到 PATH 最前面
# 3. 设置 VIRTUAL_ENV 环境变量
# 4. 修改 PS1 提示符

# deactivate 时自动恢复所有变量
```

## 5. pip 包管理

### 5.1 pip 对比 yum/apt

| 操作             | yum/apt                     | pip                            |
|------------------|-----------------------------|--------------------------------|
| 安装包           | yum install nginx           | pip install requests           |
| 卸载包           | yum remove nginx            | pip uninstall requests         |
| 查看已装         | yum list installed          | pip list                       |
| 搜索包           | yum search nginx            | pip search requests (已弃用)   |
| 查看包信息       | yum info nginx              | pip show requests              |
| 更新包           | yum update nginx            | pip install --upgrade requests |
| 锁定依赖         | 无原生支持                  | pip freeze > requirements.txt  |
| 从文件安装       | 无                          | pip install -r requirements.txt|

### 5.2 pip 常用命令

```bash
# ==========================================
# pip 基础操作
# ==========================================

# 安装包
pip install requests                    # 安装最新版
pip install requests==2.31.0            # 安装指定版本
pip install "requests>=2.28,<2.32"      # 安装版本范围

# 查看已安装的包
pip list                                # 列表形式
pip freeze                              # requirements.txt 格式

# 查看包的详细信息
pip show requests

# 升级包
pip install --upgrade requests

# 卸载包
pip uninstall requests -y

# 导出依赖到 requirements.txt
pip freeze > requirements.txt

# 从 requirements.txt 安装所有依赖
pip install -r requirements.txt
```

### 5.3 国内镜像源配置

```bash
# ==========================================
# 配置国内镜像源（解决下载慢的问题）
# ==========================================

# 方法一：临时使用
pip install requests -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方法二：永久配置（推荐）
mkdir -p ~/.pip
cat > ~/.pip/pip.conf << 'EOF'
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
trusted-host = pypi.tuna.tsinghua.edu.cn
timeout = 120

[install]
trusted-host = pypi.tuna.tsinghua.edu.cn
EOF

# 常用镜像源：
# 清华大学：https://pypi.tuna.tsinghua.edu.cn/simple
# 阿里云：  https://mirrors.aliyun.com/pypi/simple
# 豆瓣：    https://pypi.douban.com/simple
# 中科大：  https://pypi.mirrors.ustc.edu.cn/simple

# 验证配置
pip config list
```

### 5.4 requirements.txt 详解

```bash
# ==========================================
# requirements.txt 文件格式
# ==========================================

cat > requirements.txt << 'EOF'
# 精确版本（推荐用于生产环境）
requests==2.31.0
psutil==5.9.6
paramiko==3.4.0

# 版本范围
flask>=2.3,<3.0

# 最低版本
pyyaml>=6.0

# 从 git 仓库安装
# git+https://github.com/user/repo.git@main#egg=package_name

# 引用其他 requirements 文件
# -r base-requirements.txt
EOF

# 安装
pip install -r requirements.txt

# 最佳实践：生成精确版本锁定文件
pip freeze > requirements-lock.txt
```

## 7. SRE 实战案例：环境初始化脚本

```python
#!/usr/bin/env python3
"""
SRE 环境初始化脚本
功能：检查 Python 环境、创建虚拟环境、安装依赖、验证安装
用法：python3 setup_env.py
"""

import sys
import os
import subprocess
import shutil
from pathlib import Path


def 检查python版本():
    """检查当前 Python 版本是否满足要求"""
    print("=" * 50)
    print("【步骤1】检查 Python 版本")
    print("=" * 50)

    # 获取版本信息
    版本 = sys.version_info
    print(f"当前 Python 版本: {sys.version}")
    print(f"主版本: {版本.major}, 次版本: {版本.minor}, 微版本: {版本.micro}")
    print(f"Python 可执行文件: {sys.executable}")

    # 检查最低版本要求
    if 版本.major < 3 or (版本.major == 3 and 版本.minor < 8):
        print("[错误] 需要 Python 3.8 或更高版本！")
        print("请安装新版本: pyenv install 3.11.7")
        sys.exit(1)

    print("[通过] Python 版本满足要求\n")
    return True


def 检查pip():
    """检查 pip 是否可用"""
    print("=" * 50)
    print("【步骤2】检查 pip")
    print("=" * 50)

    try:
        import pip
        print(f"pip 版本: {pip.__version__}")
        print("[通过] pip 可用\n")
        return True
    except ImportError:
        print("[错误] pip 未安装！")
        print("修复方法: python3 -m ensurepip --upgrade")
        return False


def 创建虚拟环境(项目路径):
    """在指定路径创建虚拟环境"""
    print("=" * 50)
    print("【步骤3】创建虚拟环境")
    print("=" * 50)

    venv_路径 = Path(项目路径) / ".venv"

    if venv_路径.exists():
        print(f"虚拟环境已存在: {venv_路径}")
        print("[跳过] 使用已有的虚拟环境\n")
        return str(venv_路径)

    print(f"创建虚拟环境: {venv_路径}")
    subprocess.run(
        [sys.executable, "-m", "venv", str(venv_路径)],
        check=True
    )
    print("[完成] 虚拟环境创建成功\n")
    return str(venv_路径)


def 安装依赖(venv_路径, 需求文件="requirements.txt"):
    """在虚拟环境中安装依赖"""
    print("=" * 50)
    print("【步骤4】安装依赖")
    print("=" * 50)

    # 确定 pip 路径
    pip_路径 = os.path.join(venv_路径, "bin", "pip")

    # 先升级 pip 自身
    print("升级 pip...")
    subprocess.run(
        [pip_路径, "install", "--upgrade", "pip"],
        capture_output=True
    )

    # 检查 requirements.txt 是否存在
    if not os.path.exists(需求文件):
        print(f"[警告] {需求文件} 不存在，跳过依赖安装")
        # 创建示例 requirements.txt
        with open(需求文件, "w") as f:
            f.write("# SRE 常用 Python 包\n")
            f.write("requests>=2.28.0\n")
            f.write("psutil>=5.9.0\n")
            f.write("pyyaml>=6.0\n")
            f.write("paramiko>=3.0.0\n")
            f.write("click>=8.0.0\n")
        print(f"已创建示例 {需求文件}\n")

    # 安装依赖
    print(f"从 {需求文件} 安装依赖...")
    结果 = subprocess.run(
        [pip_路径, "install", "-r", 需求文件],
        capture_output=True,
        text=True
    )

    if 结果.returncode == 0:
        print("[完成] 依赖安装成功\n")
    else:
        print(f"[错误] 安装失败:\n{结果.stderr}")
        return False

    return True


def 验证安装(venv_路径):
    """验证关键包是否安装成功"""
    print("=" * 50)
    print("【步骤5】验证安装")
    print("=" * 50)

    python_路径 = os.path.join(venv_路径, "bin", "python")

    # 列出已安装的包
    结果 = subprocess.run(
        [python_路径, "-m", "pip", "list", "--format=columns"],
        capture_output=True,
        text=True
    )
    print("已安装的包：")
    print(结果.stdout)

    print("[完成] 环境搭建成功！")
    print(f"\n使用方法：")
    print(f"  source {venv_路径}/bin/activate")
    print(f"  python your_script.py")
    print(f"  deactivate")


def main():
    """主函数：执行完整的环境初始化流程"""
    项目路径 = os.getcwd()
    print(f"项目路径: {项目路径}\n")

    # 1. 检查 Python 版本
    检查python版本()

    # 2. 检查 pip
    检查pip()

    # 3. 创建虚拟环境
    venv_路径 = 创建虚拟环境(项目路径)

    # 4. 安装依赖
    安装依赖(venv_路径)

    # 5. 验证安装
    验证安装(venv_路径)


if __name__ == "__main__":
    main()
```

## 8. 常见坑点与最佳实践

### 坑点 1：不要动系统 Python

```bash
# 大坑！千万不要这样做：
sudo pip install xxx              # 会污染系统 Python
sudo pip install --upgrade pip    # 可能导致系统工具（如 yum）崩溃

# 正确做法：始终使用虚拟环境
python3 -m venv .venv
source .venv/bin/activate
pip install xxx                   # 只影响虚拟环境
```

### 坑点 2：python 还是 python3

```bash
# 在 CentOS 7 上：
python   --version  # Python 2.7.5（系统自带）
python3  --version  # Python 3.6.8（需要安装）

# 在 Ubuntu 22.04 上：
python   --version  # 可能报错 command not found
python3  --version  # Python 3.10.x

# 最佳实践：脚本中始终使用 python3
#!/usr/bin/env python3
```

### 坑点 3：权限问题

```bash
# 错误：使用 sudo pip（全局安装）
sudo pip3 install requests    # 不推荐！

# 正确：使用 --user 或虚拟环境
pip3 install --user requests  # 安装到用户目录
# 或者（推荐）
python3 -m venv .venv && source .venv/bin/activate
pip install requests
```

### 坑点 4：pyenv 编译失败

```bash
# 安装 Python 版本时编译失败，通常是缺少依赖
# CentOS 需要安装：
sudo yum install -y gcc make openssl-devel bzip2-devel \
    libffi-devel readline-devel sqlite-devel zlib-devel xz-devel

# Ubuntu 需要安装：
sudo apt install -y build-essential libssl-dev zlib1g-dev \
    libbz2-dev libreadline-dev libsqlite3-dev libffi-dev \
    liblzma-dev libncurses-dev
```

### 坑点 5：虚拟环境不可移动

```bash
# 虚拟环境中包含绝对路径，移动后会失效
mv /home/devops/project-a/.venv /home/devops/project-b/.venv  # 错误！

# 正确做法：在新位置重新创建
cd /home/devops/project-b
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 最佳实践总结

```bash
# 1. 使用 pyenv 管理 Python 版本
pyenv install 3.11.7
pyenv global 3.11.7

# 2. 每个项目使用独立的虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 3. 锁定依赖版本
pip freeze > requirements.txt

# 4. .gitignore 中排除虚拟环境
echo ".venv/" >> .gitignore
echo "__pycache__/" >> .gitignore

# 5. 使用国内镜像加速
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests

# 6. 脚本首行使用 env 方式
#!/usr/bin/env python3
```
