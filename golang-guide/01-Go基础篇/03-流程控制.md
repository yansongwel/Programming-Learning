## 导读

Go 的流程控制极其简洁：只有 `if`、`for`、`switch`、`select`（并发篇再讲）四种。没有 `while`、没有 `do-while`、没有 `try-catch`。`for` 一个关键字就覆盖了所有循环场景。本篇还会重点讲解 Go 独有的 `defer` 机制——SRE 在编写资源清理代码时的利器。

## 1. if/else 条件语句

### 1.1 基本用法

```go
package main

import "fmt"

func main() {
    temperature := 38.5

    // 基本 if-else
    // 注意：条件不需要括号，但大括号是必须的
    if temperature > 37.5 {
        fmt.Println("发烧了！")
    } else if temperature > 36.0 {
        fmt.Println("体温正常")
    } else {
        fmt.Println("体温偏低")
    }

    // 注意：Go 不支持三元运算符 a ? b : c
    // 只能用 if-else
    status := ""
    if temperature > 37.5 {
        status = "异常"
    } else {
        status = "正常"
    }
    fmt.Println("状态:", status)
}
```

> **与 Python 对比**：Python 有三元表达式 `x = "异常" if temp > 37.5 else "正常"`，Go 没有。Go 的设计哲学是只提供一种做事方式。

### 1.2 if 初始化语句（Go 特色）

```go
package main

import (
    "fmt"
    "os"
    "strconv"
)

func main() {
    // Go 的 if 可以包含一个初始化语句，用分号和条件分隔
    // 初始化的变量只在 if-else 块内可见

    // 经典模式：错误处理
    if port := os.Getenv("PORT"); port != "" {
        fmt.Println("从环境变量获取端口:", port)
    } else {
        fmt.Println("使用默认端口: 8080")
    }
    // fmt.Println(port) // 编译错误：port 在这里不可见

    // 最常见的用法：comma-ok 模式
    envPort := os.Getenv("PORT")
    if n, err := strconv.Atoi(envPort); err == nil {
        fmt.Printf("端口号: %d\n", n)
    } else {
        fmt.Printf("端口解析失败: %v, 使用默认值\n", err)
    }

    // 实际场景：检查 map 中的键
    headers := map[string]string{
        "Content-Type":  "application/json",
        "Authorization": "Bearer token123",
    }

    if auth, ok := headers["Authorization"]; ok {
        fmt.Println("认证头:", auth)
    } else {
        fmt.Println("未找到认证头")
    }

    // 实际场景：类型断言
    var value interface{} = "hello"
    if s, ok := value.(string); ok {
        fmt.Println("是字符串:", s)
    } else {
        fmt.Println("不是字符串")
    }
}
```

### 1.3 多条件判断

```go
package main

import (
    "fmt"
    "net/http"
)

// checkHTTPStatus 根据 HTTP 状态码返回描述
func checkHTTPStatus(code int) string {
    if code >= 200 && code < 300 {
        return "成功"
    } else if code >= 300 && code < 400 {
        return "重定向"
    } else if code >= 400 && code < 500 {
        return "客户端错误"
    } else if code >= 500 {
        return "服务器错误"
    }
    return "未知状态"
}

func main() {
    codes := []int{
        http.StatusOK,                  // 200
        http.StatusMovedPermanently,    // 301
        http.StatusNotFound,            // 404
        http.StatusInternalServerError, // 500
    }

    for _, code := range codes {
        fmt.Printf("HTTP %d: %s\n", code, checkHTTPStatus(code))
    }
}
```

## 2. for 循环

Go 只有 `for` 一个循环关键字，但它能表达三种循环模式。

### 2.1 经典三段式 for

```go
package main

import "fmt"

func main() {
    // 经典 C 风格 for 循环
    // for 初始化; 条件; 后置语句 { ... }
    fmt.Println("=== 经典 for ===")
    for i := 0; i < 5; i++ {
        fmt.Printf("i = %d\n", i)
    }

    // 多变量
    fmt.Println("\n=== 多变量 for ===")
    for i, j := 0, 10; i < j; i, j = i+1, j-1 {
        fmt.Printf("i=%d, j=%d\n", i, j)
    }

    // 倒序
    fmt.Println("\n=== 倒序 ===")
    for i := 5; i > 0; i-- {
        fmt.Printf("倒计时: %d\n", i)
    }
}
```

### 2.2 while 风格（省略初始化和后置语句）

```go
package main

import "fmt"

func main() {
    // Go 没有 while 关键字，用 for 替代
    // for 条件 { ... }
    fmt.Println("=== while 风格 ===")
    count := 0
    for count < 5 {
        fmt.Println("count:", count)
        count++
    }

    // 实际场景：重试机制
    fmt.Println("\n=== 重试机制 ===")
    maxRetries := 3
    retryCount := 0
    for retryCount < maxRetries {
        retryCount++
        fmt.Printf("第 %d 次尝试...\n", retryCount)

        // 模拟操作成功
        if retryCount == 2 {
            fmt.Println("成功！")
            break
        }
        fmt.Println("失败，重试...")
    }

    // 无限循环（等价于 while True）
    // for { ... }
    fmt.Println("\n=== 无限循环（带退出条件） ===")
    n := 0
    for {
        n++
        if n > 3 {
            break
        }
        fmt.Println("循环:", n)
    }
}
```

> **与 Python 对比**：
> - Python: `while count < 5:` => Go: `for count < 5 { }`
> - Python: `while True:` => Go: `for { }`
> - Python: `for i in range(5):` => Go: `for i := 0; i < 5; i++ { }`

### 2.3 for-range 遍历

```go
package main

import "fmt"

func main() {
    // ============ 遍历切片/数组 ============
    fruits := []string{"苹果", "香蕉", "橙子", "葡萄"}

    // 同时获取索引和值
    fmt.Println("=== 遍历切片（索引+值） ===")
    for i, fruit := range fruits {
        fmt.Printf("  索引 %d: %s\n", i, fruit)
    }

    // 只获取索引
    fmt.Println("\n=== 只获取索引 ===")
    for i := range fruits {
        fmt.Printf("  索引: %d\n", i)
    }

    // 只获取值（忽略索引）
    fmt.Println("\n=== 只获取值 ===")
    for _, fruit := range fruits {
        fmt.Printf("  值: %s\n", fruit)
    }

    // ============ 遍历字符串 ============
    fmt.Println("\n=== 遍历字符串（按 rune） ===")
    s := "Go语言"
    for i, ch := range s {
        fmt.Printf("  字节偏移 %d: %c (U+%04X)\n", i, ch, ch)
    }
    // 输出:
    // 字节偏移 0: G (U+0047)
    // 字节偏移 1: o (U+006F)
    // 字节偏移 2: 语 (U+8BED)
    // 字节偏移 5: 言 (U+8A00)
    // 注意：索引是字节偏移，不是字符偏移！

    // ============ 遍历 map ============
    fmt.Println("\n=== 遍历 map ===")
    servers := map[string]int{
        "web-01":  8080,
        "web-02":  8081,
        "api-01":  9090,
        "cache-01": 6379,
    }

    for host, port := range servers {
        fmt.Printf("  %s -> :%d\n", host, port)
    }
    // 注意：map 遍历顺序是随机的！

    // ============ 遍历 channel ============
    fmt.Println("\n=== 遍历 channel ===")
    ch := make(chan int, 5)
    // 往 channel 发送数据
    for i := 1; i <= 5; i++ {
        ch <- i * 10
    }
    close(ch) // 关闭 channel

    // range channel 会一直读取直到 channel 关闭
    for val := range ch {
        fmt.Printf("  收到: %d\n", val)
    }

    // ============ Go 1.22+ range over integers ============
    fmt.Println("\n=== Go 1.22+ 整数 range ===")
    for i := range 5 {
        fmt.Printf("  i = %d\n", i) // 0, 1, 2, 3, 4
    }
}
```

### 2.4 break 和 continue

```go
package main

import "fmt"

func main() {
    // ============ break：跳出循环 ============
    fmt.Println("=== break ===")
    for i := 0; i < 10; i++ {
        if i == 5 {
            break // 跳出整个循环
        }
        fmt.Println(i)
    }

    // ============ continue：跳过当前迭代 ============
    fmt.Println("\n=== continue（只打印偶数） ===")
    for i := 0; i < 10; i++ {
        if i%2 != 0 {
            continue // 跳过奇数
        }
        fmt.Println(i)
    }

    // ============ 带标签的 break（跳出多层循环） ============
    fmt.Println("\n=== 带标签的 break ===")
outer:
    for i := 0; i < 3; i++ {
        for j := 0; j < 3; j++ {
            if i == 1 && j == 1 {
                fmt.Println("在 i=1, j=1 处跳出外层循环")
                break outer // 跳出外层循环
            }
            fmt.Printf("  i=%d, j=%d\n", i, j)
        }
    }

    // ============ 带标签的 continue ============
    fmt.Println("\n=== 带标签的 continue ===")
outerLoop:
    for i := 0; i < 3; i++ {
        for j := 0; j < 3; j++ {
            if j == 1 {
                continue outerLoop // 跳到外层循环的下一次迭代
            }
            fmt.Printf("  i=%d, j=%d\n", i, j)
        }
    }

    // 实际场景：在嵌套循环中搜索
    fmt.Println("\n=== 实际场景：搜索矩阵 ===")
    matrix := [][]int{
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9},
    }
    target := 5
    found := false

search:
    for i, row := range matrix {
        for j, val := range row {
            if val == target {
                fmt.Printf("找到 %d 在位置 [%d][%d]\n", target, i, j)
                found = true
                break search
            }
        }
    }
    if !found {
        fmt.Printf("未找到 %d\n", target)
    }
}
```

## 3. switch 语句

### 3.1 基本用法（无需 break）

```go
package main

import (
    "fmt"
    "runtime"
)

func main() {
    // Go 的 switch 自动 break，不需要手动写
    // 这与 C/Java 不同！

    os := runtime.GOOS
    fmt.Printf("操作系统: %s\n", os)

    switch os {
    case "linux":
        fmt.Println("使用 systemd 管理服务")
    case "darwin":
        fmt.Println("使用 launchd 管理服务")
    case "windows":
        fmt.Println("使用 Windows Services")
    default:
        fmt.Println("未知操作系统")
    }
}
```

> **与 Python 对比**：Python 3.10+ 有 `match-case`（结构化模式匹配），但功能更强大也更复杂。Go 的 switch 更简洁直接。

### 3.2 多值 case

```go
package main

import "fmt"

func main() {
    day := "Saturday"

    switch day {
    case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday":
        fmt.Println("工作日，检查服务状态")
    case "Saturday", "Sunday":
        fmt.Println("周末，减少告警阈值")
    default:
        fmt.Println("无效的日期")
    }

    // 实际场景：HTTP 方法分发
    method := "POST"
    switch method {
    case "GET":
        fmt.Println("处理查询请求")
    case "POST", "PUT", "PATCH":
        fmt.Println("处理写入请求")
    case "DELETE":
        fmt.Println("处理删除请求")
    case "OPTIONS":
        fmt.Println("处理预检请求")
    default:
        fmt.Println("不支持的方法:", method)
    }
}
```

### 3.3 switch 初始化语句


### 3.5 类型 switch（type switch）

```go
package main

import "fmt"

// processValue 处理不同类型的值
func processValue(v interface{}) {
    // 类型 switch：根据接口值的实际类型分支
    switch val := v.(type) {
    case nil:
        fmt.Println("nil 值")
    case bool:
        fmt.Printf("布尔值: %v\n", val)
    case int:
        fmt.Printf("整数: %d\n", val)
    case int64:
        fmt.Printf("int64: %d\n", val)
    case float64:
        fmt.Printf("浮点数: %f\n", val)
    case string:
        fmt.Printf("字符串: %q (长度: %d)\n", val, len(val))
    case []byte:
        fmt.Printf("字节切片: %v\n", val)
    case []string:
        fmt.Printf("字符串切片: %v\n", val)
    case map[string]interface{}:
        fmt.Printf("map: %v\n", val)
    case error:
        fmt.Printf("错误: %v\n", val)
    default:
        fmt.Printf("未知类型: %T = %v\n", val, val)
    }
}

func main() {
    processValue(nil)
    processValue(true)
    processValue(42)
    processValue(3.14)
    processValue("hello")
    processValue([]byte{1, 2, 3})
    processValue([]string{"a", "b"})
    processValue(map[string]interface{}{"key": "value"})
    processValue(fmt.Errorf("出错了"))
    processValue(struct{ Name string }{"DevOps"})
}
```

### 3.6 fallthrough（显式贯穿）

```go
package main

import "fmt"

func main() {
    // Go 的 switch 默认不会 fallthrough
    // 如果需要贯穿到下一个 case，必须显式写 fallthrough

    n := 3
    fmt.Printf("n = %d: ", n)
    switch {
    case n > 0:
        fmt.Print("正数 ")
        fallthrough // 继续执行下一个 case（不检查条件！）
    case n > -5:
        fmt.Print("大于-5 ")
        fallthrough
    case n > -10:
        fmt.Print("大于-10 ")
    case n > -100:
        fmt.Print("大于-100 ") // 不会执行，上一个 case 没有 fallthrough
    }
    fmt.Println()
    // 输出: n = 3: 正数 大于-5 大于-10

    // 注意：fallthrough 无条件执行下一个 case 的代码
    // 它不会检查下一个 case 的条件
    // 实际项目中很少使用 fallthrough

    // 实际使用场景：版本兼容处理
    version := 3
    fmt.Printf("\n版本 %d 的特性:\n", version)
    switch version {
    case 3:
        fmt.Println("  - 新增: 热重载")
        fallthrough
    case 2:
        fmt.Println("  - 新增: 配置文件支持")
        fallthrough
    case 1:
        fmt.Println("  - 基础功能")
    }
}
```

## 4. defer 机制

`defer` 是 Go 最重要的特性之一，它延迟执行一个函数调用，直到包含它的函数返回。

### 4.1 基本用法

```go
package main

import "fmt"

func main() {
    fmt.Println("开始")
    defer fmt.Println("defer 1: 我最后执行")
    fmt.Println("中间")
    defer fmt.Println("defer 2: 我倒数第二执行")
    fmt.Println("结束")

    // 输出顺序:
    // 开始
    // 中间
    // 结束
    // defer 2: 我倒数第二执行
    // defer 1: 我最后执行
}
```

### 4.2 LIFO 执行顺序（后进先出）

```go
package main

import "fmt"

func main() {
    // defer 使用栈结构，后注册的先执行（LIFO）
    fmt.Println("=== LIFO 执行顺序 ===")
    for i := 0; i < 5; i++ {
        defer fmt.Printf("defer: %d\n", i)
    }
    fmt.Println("循环结束")

    // 输出:
    // 循环结束
    // defer: 4
    // defer: 3
    // defer: 2
    // defer: 1
    // defer: 0
}
```

### 4.4 defer 参数求值时机

```go
package main

import "fmt"

func main() {
    // 重要：defer 的参数在 defer 语句执行时就求值了
    // 而不是在 defer 函数执行时求值

    x := 10
    defer fmt.Println("defer 中的 x:", x) // x 在这里就被求值为 10
    x = 20
    fmt.Println("当前 x:", x)

    // 输出:
    // 当前 x: 20
    // defer 中的 x: 10   ← 不是 20！

    // 如果想让 defer 使用最新的值，用闭包
    y := 10
    defer func() {
        fmt.Println("闭包中的 y:", y) // y 通过闭包引用，取最新值
    }()
    y = 20
    fmt.Println("当前 y:", y)

    // 输出:
    // 当前 y: 20
    // 闭包中的 y: 20  ← 是 20！

    fmt.Println()

    // 用指针也可以
    z := 10
    zp := &z
    defer fmt.Println("指针中的 z:", *zp)
    // 注意：*zp 在 defer 时求值，所以还是 10
    z = 20
}
```

### 4.5 defer 与命名返回值

```go
package main

import "fmt"

// defer 可以修改命名返回值！
func doublePlus(n int) (result int) {
    defer func() {
        result++ // 修改命名返回值
    }()

    return n * 2 // 先设置 result = n*2，然后执行 defer，result++
}

// 实际应用：错误包装
func processFile(name string) (err error) {
    defer func() {
        if err != nil {
            err = fmt.Errorf("processFile(%s): %w", name, err)
        }
    }()

    // 如果这里返回错误，defer 会自动包装错误信息
    // f, err := os.Open(name)
    // ...
    return nil
}

func main() {
    result := doublePlus(5)
    fmt.Println("doublePlus(5) =", result) // 11（5*2=10，defer +1=11）
}
```

## 5. goto 语句

```go
package main

import "fmt"

func main() {
    // goto 可以无条件跳转到同函数内的标签
    // 不推荐使用，但某些场景下可以简化错误处理

    // ============ 基本用法 ============
    fmt.Println("开始")
    goto end // 跳转到 end 标签
    fmt.Println("这行不会执行")
end:
    fmt.Println("结束")

    // ============ 实际场景：多步骤资源清理（不推荐，用 defer 更好） ============
    // 这是 C 语言中常见的 goto 模式，Go 中应该用 defer 替代
    /*
    f1, err := openFile1()
    if err != nil {
        goto cleanup
    }
    f2, err := openFile2()
    if err != nil {
        goto cleanup1
    }
    // 使用文件...
    cleanup1:
        f2.Close()
    cleanup:
        f1.Close()
    */

    // ============ goto 的限制 ============
    // 1. 不能跳过变量声明
    // goto skip
    // x := 10    // 编译错误：goto 不能跳过变量声明
    // skip:

    // 2. 不能跳入内层作用域
    // goto inner
    // {
    //     inner:  // 编译错误
    // }

    // 3. 不能跨函数跳转

    // 总结：99% 的场景用 defer + if/for/break 就够了
    // goto 只在极少数性能敏感的底层代码中使用
}
```

## 6. for-range 的坑点

### 6.1 循环变量的问题（Go 1.22 之前的经典坑）

```go
package main

import (
    "fmt"
    "sync"
)

func main() {
    // ============ Go 1.22 之前的经典坑 ============
    // 在 Go 1.22 之前，for-range 的循环变量在每次迭代中被复用
    // 这导致闭包捕获的是同一个变量

    // Go 1.22+ 修复了这个问题：每次迭代创建新的变量
    // 但了解这个历史问题仍然重要

    values := []int{1, 2, 3, 4, 5}

    // 正确行为（Go 1.22+）：每次迭代 v 是新变量
    var wg sync.WaitGroup
    for _, v := range values {
        wg.Add(1)
        go func() {
            defer wg.Done()
            fmt.Printf("v = %d\n", v) // Go 1.22+: 输出 1,2,3,4,5（顺序随机）
        }()
    }
    wg.Wait()

    fmt.Println()

    // Go 1.21 及之前的版本中，上面的代码会输出：
    // v = 5
    // v = 5
    // v = 5
    // v = 5
    // v = 5
    // 因为所有 goroutine 共享同一个 v 变量

    // Go 1.21 及之前的修复方式：
    fmt.Println("=== Go 1.21 及之前的修复方式 ===")

    // 方式一：在循环内创建新变量
    for _, v := range values {
        v := v // 重新声明 v，创建新变量（shadow）
        wg.Add(1)
        go func() {
            defer wg.Done()
            fmt.Printf("v = %d\n", v)
        }()
    }
    wg.Wait()

    fmt.Println()

    // 方式二：通过参数传递
    for _, v := range values {
        wg.Add(1)
        go func(val int) {
            defer wg.Done()
            fmt.Printf("v = %d\n", val)
        }(v)
    }
    wg.Wait()
}
```

### 6.2 遍历时修改集合

```go
package main

import "fmt"

func main() {
    // ============ 遍历切片时修改 ============
    // 不要在 range 遍历中向切片 append！

    nums := []int{1, 2, 3, 4, 5}

    // 危险：可能导致无限循环或意外行为
    // for i, v := range nums {
    //     if v == 3 {
    //         nums = append(nums, 6) // 不要这样做！
    //     }
    // }

    // 安全方式：遍历的是原始切片的副本
    // range 在开始时就确定了长度，不会受后续 append 影响
    // 但修改已有元素是安全的
    for i := range nums {
        nums[i] *= 2
    }
    fmt.Println("修改后:", nums) // [2 4 6 8 10]

    // 安全的删除方式：倒序遍历
    items := []int{1, 2, 3, 4, 5, 6}
    fmt.Println("原始:", items)

    // 删除偶数
    result := items[:0] // 利用原切片的底层数组
    for _, v := range items {
        if v%2 != 0 {
            result = append(result, v)
        }
    }
    fmt.Println("删除偶数后:", result) // [1 3 5]

    // ============ 遍历 map 时修改 ============
    m := map[string]int{
        "a": 1,
        "b": 2,
        "c": 3,
    }

    // 遍历 map 时删除元素是安全的（Go 明确允许）
    for k, v := range m {
        if v > 1 {
            delete(m, k)
        }
    }
    fmt.Println("删除后:", m) // map[a:1]

    // 但遍历 map 时添加元素，新元素可能出现也可能不出现在遍历中
    // 行为是不确定的，不推荐
}
```

### 6.3 range 返回值是副本

```go
package main

import "fmt"

type Server struct {
    Name   string
    Status string
}

func main() {
    servers := []Server{
        {Name: "web-01", Status: "running"},
        {Name: "web-02", Status: "running"},
        {Name: "web-03", Status: "stopped"},
    }

    // 坑：range 返回的是元素的副本！
    // 修改 v 不会影响原切片
    for _, v := range servers {
        v.Status = "updated" // 修改的是副本，不影响原切片！
    }
    fmt.Println("range 值修改（不生效）:", servers)
    // [{web-01 running} {web-02 running} {web-03 stopped}]

    // 正确方式一：使用索引
    for i := range servers {
        servers[i].Status = "updated"
    }
    fmt.Println("索引修改（生效）:", servers)

    // 正确方式二：使用指针切片
    serverPtrs := []*Server{
        {Name: "api-01", Status: "running"},
        {Name: "api-02", Status: "stopped"},
    }
    for _, v := range serverPtrs {
        v.Status = "updated" // v 是指针，修改会影响原数据
    }
    for _, v := range serverPtrs {
        fmt.Printf("  %s: %s\n", v.Name, v.Status)
    }
}
```

## 实战场景：服务健康检查调度器

```go
package main

import (
    "fmt"
    "math/rand"
    "strings"
    "time"
)

// ServiceStatus 服务状态
type ServiceStatus int

const (
    StatusUnknown ServiceStatus = iota
    StatusHealthy
    StatusDegraded
    StatusDown
)

func (s ServiceStatus) String() string {
    switch s {
    case StatusHealthy:
        return "HEALTHY"
    case StatusDegraded:
        return "DEGRADED"
    case StatusDown:
        return "DOWN"
    default:
        return "UNKNOWN"
    }
}

// Service 服务信息
type Service struct {
    Name     string
    Endpoint string
    Status   ServiceStatus
    Latency  time.Duration
}

// checkService 模拟健康检查
func checkService(name, endpoint string) Service {
    // 模拟网络延迟
    latency := time.Duration(rand.Intn(500)) * time.Millisecond

    // 模拟不同状态
    var status ServiceStatus
    r := rand.Float64()
    switch {
    case r < 0.7:
        status = StatusHealthy
    case r < 0.9:
        status = StatusDegraded
    default:
        status = StatusDown
    }

    return Service{
        Name:     name,
        Endpoint: endpoint,
        Status:   status,
        Latency:  latency,
    }
}

func main() {
    // 服务列表
    services := map[string]string{
        "web-frontend": "http://frontend:3000/health",
        "api-gateway":  "http://gateway:8080/health",
        "user-service": "http://user-svc:9090/health",
        "db-primary":   "tcp://db-primary:5432",
        "cache-redis":  "tcp://redis:6379",
    }

    fmt.Println("=" + strings.Repeat("=", 70))
    fmt.Println(" 服务健康检查报告")
    fmt.Println("=" + strings.Repeat("=", 70))

    var (
        healthy  int
        degraded int
        down     int
    )

    // 检查所有服务
    for name, endpoint := range services {
        result := checkService(name, endpoint)

        // 根据状态设置标记
        var marker string
        switch result.Status {
        case StatusHealthy:
            marker = "[OK]"
            healthy++
        case StatusDegraded:
            marker = "[!!]"
            degraded++
        case StatusDown:
            marker = "[XX]"
            down++
        }

        fmt.Printf("  %s %-15s %-8s 延迟: %v\n",
            marker, result.Name, result.Status, result.Latency)
    }

    fmt.Println(strings.Repeat("-", 71))
    fmt.Printf("  总计: %d 个服务 | 健康: %d | 降级: %d | 宕机: %d\n",
        len(services), healthy, degraded, down)
    fmt.Println(strings.Repeat("=", 71))

    // 判断整体状态
    switch {
    case down > 0:
        fmt.Println("\n  总体状态: 紧急 - 有服务宕机，需要立即处理！")
    case degraded > 0:
        fmt.Println("\n  总体状态: 警告 - 有服务降级，请关注")
    default:
        fmt.Println("\n  总体状态: 正常 - 所有服务运行良好")
    }

    // 模拟重试逻辑
    fmt.Println("\n--- 重试宕机服务 ---")
    maxRetries := 3
    for name, endpoint := range services {
        result := checkService(name, endpoint)
        if result.Status != StatusDown {
            continue
        }

        fmt.Printf("服务 %s 宕机，开始重试...\n", name)
        recovered := false

        for retry := 1; retry <= maxRetries; retry++ {
            fmt.Printf("  第 %d/%d 次重试 %s...\n", retry, maxRetries, name)
            retryResult := checkService(name, endpoint)

            if retryResult.Status != StatusDown {
                fmt.Printf("  服务 %s 已恢复! 状态: %s\n", name, retryResult.Status)
                recovered = true
                break
            }
        }

        if !recovered {
            fmt.Printf("  服务 %s 重试 %d 次后仍然宕机，触发告警\n", name, maxRetries)
        }
    }
}
```

## 常见坑点总结

### 坑点 1：if 作用域中的变量遮蔽

```go
package main

import (
    "errors"
    "fmt"
)

func getData() (string, error) {
    return "", errors.New("获取失败")
}

func main() {
    var err error

    // 坑：这里的 data 和 err 是 if 块内的新变量
    // 外层的 err 没有被修改！
    if data, err := getData(); err != nil {
        fmt.Println("错误:", err)  // 在 if 内能看到 err
        _ = data
    }
    fmt.Println("外层 err:", err) // nil！外层 err 没变

    // 正确做法
    var data string
    data, err = getData()
    if err != nil {
        fmt.Println("错误:", err)
    }
    _ = data
}
```

### 坑点 2：defer 在循环中的资源泄漏

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    files := []string{"/etc/hostname", "/etc/os-release", "/etc/hosts"}

    // 坑：defer 是在函数返回时执行，不是在循环结束时
    // 如果循环次数很多，文件句柄会累积，导致资源泄漏
    /*
    for _, name := range files {
        f, err := os.Open(name)
        if err != nil {
            continue
        }
        defer f.Close() // 直到 main 函数返回才关闭！所有文件同时打开
        // 处理文件...
    }
    */

    // 正确做法一：提取到独立函数
    for _, name := range files {
        if err := processFile(name); err != nil {
            fmt.Println("处理失败:", err)
        }
    }

    // 正确做法二：使用闭包
    for _, name := range files {
        func() {
            f, err := os.Open(name)
            if err != nil {
                fmt.Println("打开失败:", err)
                return
            }
            defer f.Close() // 在闭包返回时就关闭
            // 处理文件...
            fmt.Printf("已处理: %s\n", name)
        }()
    }
}

func processFile(name string) error {
    f, err := os.Open(name)
    if err != nil {
        return err
    }
    defer f.Close() // 函数返回时关闭

    fmt.Printf("已处理: %s\n", name)
    return nil
}
```

## 速查表

### 控制结构对比

| 场景 | Go | Python | Shell |
|------|-----|--------|-------|
| 条件判断 | `if x > 0 { }` | `if x > 0:` | `if [ $x -gt 0 ]; then fi` |
| 三元表达式 | 不支持 | `a if cond else b` | `$((cond ? a : b))` |
| for 循环 | `for i := 0; i < n; i++ { }` | `for i in range(n):` | `for ((i=0;i<n;i++))` |
| while 循环 | `for cond { }` | `while cond:` | `while [ cond ]; do done` |
| 无限循环 | `for { }` | `while True:` | `while true; do done` |
| 遍历数组 | `for i, v := range arr { }` | `for i, v in enumerate(arr):` | `for v in "${arr[@]}"` |
| 遍历字典 | `for k, v := range m { }` | `for k, v in d.items():` | N/A |
| switch | `switch x { case 1: }` | `match x: case 1:` | `case $x in 1)` |
| 资源清理 | `defer f.Close()` | `with open() as f:` | `trap cleanup EXIT` |

### defer 规则速查

| 规则 | 说明 |
|------|------|
| 执行时机 | 包含它的函数返回前 |
| 多个 defer | LIFO（后进先出） |
| 参数求值 | defer 语句执行时立即求值 |
| 闭包变量 | 引用外部变量时取最新值 |
| 命名返回值 | defer 闭包可以修改 |
| 循环中 | 小心资源泄漏，提取到子函数 |
| panic 时 | defer 仍然执行 |

## 小结

1. **Go 的流程控制极简**：只有 `if`、`for`、`switch`（加上并发的 `select`），没有 `while`、`do-while`。
2. **if 初始化语句**是 Go 的特色，大量用于错误处理和 comma-ok 模式。
3. **for 是唯一的循环关键字**，三种形式覆盖所有场景。
4. **switch 自动 break**，`fallthrough` 需要显式声明（与 C/Java 相反）。
5. **defer 是资源管理神器**：文件关闭、锁释放、连接回收都靠它。但注意参数求值时机和循环中的使用。
6. **Go 1.22 修复了 for-range 循环变量的坑**，但理解这个历史问题对阅读旧代码很重要。
7. **不要用 goto**，用 defer + break/continue + 标签可以解决所有场景。

下一篇我们将学习 Go 的函数：多返回值、闭包、defer/panic/recover 等。
