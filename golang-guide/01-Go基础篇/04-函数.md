> **适用读者**：有 Python / Shell 经验的 SRE / DevOps 工程师
> **Go 版本**：Go 1.24

## 导读

函数是 Go 的一等公民（first-class citizen），可以赋值给变量、作为参数传递、作为返回值。Go 函数有几个独特之处：支持多返回值（天然适合错误处理）、有 `defer/panic/recover` 机制（替代 try-catch）、有 `init()` 特殊函数。本篇将全面覆盖这些内容，并通过 SRE 实战场景演示它们的用法。

## 1. 函数定义与调用

### 1.1 基本语法

```go
package main

import "fmt"

// 无参数无返回值
func sayHello() {
    fmt.Println("Hello, DevOps!")
}

// 有参数有返回值
func add(a int, b int) int {
    return a + b
}

// 相同类型的参数可以合并声明
func multiply(a, b int) int {
    return a * b
}

// 多种类型的参数
func formatServer(host string, port int, tls bool) string {
    protocol := "http"
    if tls {
        protocol = "https"
    }
    return fmt.Sprintf("%s://%s:%d", protocol, host, port)
}

func main() {
    sayHello()

    sum := add(3, 5)
    fmt.Println("3 + 5 =", sum)

    product := multiply(4, 6)
    fmt.Println("4 * 6 =", product)

    url := formatServer("api.example.com", 443, true)
    fmt.Println("URL:", url)
}
```

> **与 Python 对比**：
> - Python: `def add(a, b):` — 参数不需要类型
> - Go: `func add(a, b int) int` — 参数和返回值都需要类型
> - Go 的参数没有默认值，没有关键字参数（*args、**kwargs）

### 1.2 导出规则

```go
package mypackage

// 大写开头：导出函数（公开，其他包可以调用）
// 等同于 Python 中不以 _ 开头的函数
func PublicFunction() {
    // 其他包可以调用: mypackage.PublicFunction()
}

// 小写开头：未导出函数（私有，只能在本包内调用）
// 等同于 Python 中以 _ 开头的函数
func privateFunction() {
    // 只能在 mypackage 内部调用
}

// 这个规则适用于所有标识符：变量、常量、类型、结构体字段等
```

## 2. 多返回值

### 2.1 基本用法

```go
package main

import (
    "errors"
    "fmt"
    "net"
    "strconv"
    "strings"
)

// 返回两个值
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("除数不能为零")
    }
    return a / b, nil
}

// 返回三个值
func parseEndpoint(endpoint string) (string, int, error) {
    parts := strings.SplitN(endpoint, ":", 2)
    if len(parts) != 2 {
        return "", 0, fmt.Errorf("无效的端点格式: %q, 期望 host:port", endpoint)
    }

    host := parts[0]
    port, err := strconv.Atoi(parts[1])
    if err != nil {
        return "", 0, fmt.Errorf("无效的端口号: %q", parts[1])
    }

    if port < 1 || port > 65535 {
        return "", 0, fmt.Errorf("端口号超出范围: %d", port)
    }

    return host, port, nil
}

// 实际场景：解析网络地址
func resolveHost(hostname string) ([]string, error) {
    addrs, err := net.LookupHost(hostname)
    if err != nil {
        return nil, fmt.Errorf("DNS 解析失败: %w", err)
    }
    return addrs, nil
}

func main() {
    // 接收多返回值
    result, err := divide(10, 3)
    if err != nil {
        fmt.Println("错误:", err)
    } else {
        fmt.Printf("10 / 3 = %.4f\n", result)
    }

    // 错误情况
    _, err = divide(10, 0)
    if err != nil {
        fmt.Println("错误:", err)
    }

    // 解析端点
    host, port, err := parseEndpoint("localhost:8080")
    if err != nil {
        fmt.Println("解析失败:", err)
    } else {
        fmt.Printf("主机: %s, 端口: %d\n", host, port)
    }

    // 忽略不需要的返回值
    _, port2, _ := parseEndpoint("db-server:5432")
    fmt.Println("数据库端口:", port2)

    // DNS 解析
    addrs, err := resolveHost("localhost")
    if err != nil {
        fmt.Println("DNS 错误:", err)
    } else {
        fmt.Println("解析结果:", addrs)
    }
}
```

> **与 Python 对比**：Python 也支持多返回值（`return a, b`），但实际上返回的是元组。Go 的多返回值是语言原生支持的，不需要打包/解包。Go 的 `error` 返回值是核心惯例，替代了 Python 的异常机制。

### 2.2 命名返回值

```go
package main

import (
    "fmt"
    "math"
)

// 命名返回值：声明时给返回值命名
// 函数体内可以直接使用这些变量
// 裸 return 会返回命名返回值的当前值
func rectangleInfo(width, height float64) (area, perimeter float64) {
    area = width * height
    perimeter = 2 * (width + height)
    return // 裸 return，返回 area 和 perimeter 的当前值
}

// 命名返回值 + 错误处理
func safeSqrt(x float64) (result float64, err error) {
    if x < 0 {
        err = fmt.Errorf("不能对负数 %f 开方", x)
        return // 返回 result=0.0 (零值), err=...
    }
    result = math.Sqrt(x)
    return // 返回 result=..., err=nil
}

// 命名返回值的真正价值：配合 defer 修改返回值
func readConfig(path string) (config string, err error) {
    // defer 可以修改命名返回值
    defer func() {
        if err != nil {
            err = fmt.Errorf("readConfig(%s): %w", path, err)
        }
    }()

    // 模拟读取配置
    if path == "" {
        return "", fmt.Errorf("路径为空")
    }
    return "host=localhost\nport=8080", nil
}

func main() {
    area, peri := rectangleInfo(5, 3)
    fmt.Printf("矩形 5x3: 面积=%.1f, 周长=%.1f\n", area, peri)

    result, err := safeSqrt(16)
    fmt.Printf("sqrt(16) = %.1f, err = %v\n", result, err)

    result, err = safeSqrt(-1)
    fmt.Printf("sqrt(-1) = %.1f, err = %v\n", result, err)

    config, err := readConfig("")
    fmt.Printf("config = %q, err = %v\n", config, err)
    // 输出: config = "", err = readConfig(): 路径为空

    config, err = readConfig("/etc/app.conf")
    fmt.Printf("config = %q, err = %v\n", config, err)
}
```

## 3. 可变参数

```go
package main

import (
    "fmt"
    "strings"
)

// ...int 表示可变参数，类型为 []int
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

// 可变参数必须是最后一个参数
func logMessage(level string, parts ...string) {
    msg := strings.Join(parts, " ")
    fmt.Printf("[%s] %s\n", level, msg)
}

// 实际场景：灵活的日志函数
func logf(format string, args ...interface{}) {
    fmt.Printf("[LOG] "+format+"\n", args...)
}

// 实际场景：合并多个切片
func mergeSlices(slices ...[]string) []string {
    var result []string
    for _, s := range slices {
        result = append(result, s...)
    }
    return result
}

func main() {
    // 传递多个参数
    fmt.Println("sum(1,2,3) =", sum(1, 2, 3))
    fmt.Println("sum(1,2,3,4,5) =", sum(1, 2, 3, 4, 5))
    fmt.Println("sum() =", sum()) // 零个参数也可以

    // 传递切片（需要展开）
    nums := []int{10, 20, 30}
    fmt.Println("sum(nums...) =", sum(nums...)) // 用 ... 展开切片

    logMessage("INFO", "服务器", "启动", "成功")
    logMessage("ERROR", "连接", "超时")

    logf("服务 %s 在端口 %d 启动", "web", 8080)
    logf("CPU 使用率: %.1f%%", 85.7)

    // 合并切片
    servers := mergeSlices(
        []string{"web-01", "web-02"},
        []string{"api-01"},
        []string{"db-01", "db-02"},
    )
    fmt.Println("所有服务器:", servers)
}
```

> **与 Python 对比**：
> - Python: `def func(*args, **kwargs)` — 支持位置可变参数和关键字可变参数
> - Go: `func(args ...int)` — 只支持一种可变参数，且必须是最后一个参数
> - Go 没有 `**kwargs`，需要通过结构体或 map 实现类似功能

## 4. 函数作为参数/返回值（高阶函数）

### 4.1 函数类型

```go
package main

import (
    "fmt"
    "strings"
)

// 定义函数类型
type TransformFunc func(string) string
type FilterFunc func(string) bool

// 函数作为参数
func applyToAll(items []string, transform TransformFunc) []string {
    result := make([]string, len(items))
    for i, item := range items {
        result[i] = transform(item)
    }
    return result
}

// 过滤函数
func filter(items []string, predicate FilterFunc) []string {
    var result []string
    for _, item := range items {
        if predicate(item) {
            result = append(result, item)
        }
    }
    return result
}

// 函数作为返回值
func makePrefix(prefix string) TransformFunc {
    // 返回一个闭包
    return func(s string) string {
        return prefix + s
    }
}

// 管道模式：多个转换函数组合
func pipeline(input string, transforms ...TransformFunc) string {
    result := input
    for _, transform := range transforms {
        result = transform(result)
    }
    return result
}

func main() {
    servers := []string{"Web-01", "Api-Gateway", "Cache-Redis", "DB-Primary"}

    // 转换为小写
    lower := applyToAll(servers, strings.ToLower)
    fmt.Println("小写:", lower)

    // 转换为大写
    upper := applyToAll(servers, strings.ToUpper)
    fmt.Println("大写:", upper)

    // 添加前缀
    addEnv := makePrefix("prod-")
    withEnv := applyToAll(servers, addEnv)
    fmt.Println("添加前缀:", withEnv)

    // 过滤
    hasDB := filter(servers, func(s string) bool {
        return strings.Contains(strings.ToLower(s), "db")
    })
    fmt.Println("数据库服务:", hasDB)

    // 管道
    result := pipeline("  Hello World  ",
        strings.TrimSpace,
        strings.ToLower,
        makePrefix("log: "),
    )
    fmt.Println("管道结果:", result)
}
```

### 4.2 常见高阶函数模式

```go
package main

import (
    "fmt"
    "sort"
    "time"
)

// ============ 重试函数 ============
func retry(attempts int, delay time.Duration, fn func() error) error {
    for i := 0; i < attempts; i++ {
        err := fn()
        if err == nil {
            return nil
        }
        fmt.Printf("  第 %d 次尝试失败: %v\n", i+1, err)
        if i < attempts-1 {
            time.Sleep(delay)
        }
    }
    return fmt.Errorf("重试 %d 次后仍然失败", attempts)
}

// ============ 计时装饰器 ============
func withTiming(name string, fn func()) {
    start := time.Now()
    fn()
    elapsed := time.Since(start)
    fmt.Printf("[计时] %s 耗时: %v\n", name, elapsed)
}

// ============ Map 函数 ============
func mapInts(nums []int, fn func(int) int) []int {
    result := make([]int, len(nums))
    for i, n := range nums {
        result[i] = fn(n)
    }
    return result
}

// ============ Reduce 函数 ============
func reduceInts(nums []int, initial int, fn func(int, int) int) int {
    result := initial
    for _, n := range nums {
        result = fn(result, n)
    }
    return result
}

func main() {
    // 重试示例
    fmt.Println("=== 重试示例 ===")
    attempt := 0
    err := retry(3, 100*time.Millisecond, func() error {
        attempt++
        if attempt < 3 {
            return fmt.Errorf("连接拒绝")
        }
        fmt.Println("  连接成功！")
        return nil
    })
    if err != nil {
        fmt.Println("最终失败:", err)
    }

    // 计时示例
    fmt.Println("\n=== 计时示例 ===")
    withTiming("排序", func() {
        data := make([]int, 100000)
        for i := range data {
            data[i] = 100000 - i
        }
        sort.Ints(data)
    })

    // Map/Reduce 示例
    fmt.Println("\n=== Map/Reduce ===")
    nums := []int{1, 2, 3, 4, 5}

    doubled := mapInts(nums, func(n int) int { return n * 2 })
    fmt.Println("双倍:", doubled)

    sum := reduceInts(nums, 0, func(acc, n int) int { return acc + n })
    fmt.Println("总和:", sum)

    product := reduceInts(nums, 1, func(acc, n int) int { return acc * n })
    fmt.Println("乘积:", product)
}
```

## 5. 匿名函数

```go
package main

import (
    "fmt"
    "sort"
)

func main() {
    // ============ 匿名函数赋值给变量 ============
    greet := func(name string) string {
        return fmt.Sprintf("Hello, %s!", name)
    }
    fmt.Println(greet("DevOps"))

    // ============ 立即执行的匿名函数（IIFE） ============
    result := func(a, b int) int {
        return a + b
    }(3, 5)
    fmt.Println("IIFE 结果:", result)

    // ============ 作为回调使用 ============
    servers := []struct {
        Name   string
        CPU    float64
        Memory float64
    }{
        {"web-01", 85.5, 72.3},
        {"web-02", 45.2, 88.1},
        {"api-01", 92.0, 65.7},
        {"db-01", 30.1, 95.2},
    }

    // 按 CPU 使用率排序
    sort.Slice(servers, func(i, j int) bool {
        return servers[i].CPU > servers[j].CPU // 降序
    })
    fmt.Println("\n按 CPU 降序:")
    for _, s := range servers {
        fmt.Printf("  %s: CPU=%.1f%%, Mem=%.1f%%\n", s.Name, s.CPU, s.Memory)
    }

    // 按 Memory 使用率排序
    sort.Slice(servers, func(i, j int) bool {
        return servers[i].Memory > servers[j].Memory // 降序
    })
    fmt.Println("\n按内存降序:")
    for _, s := range servers {
        fmt.Printf("  %s: CPU=%.1f%%, Mem=%.1f%%\n", s.Name, s.CPU, s.Memory)
    }

    // ============ 匿名函数用于 goroutine ============
    done := make(chan bool)
    go func() {
        fmt.Println("\n来自 goroutine 的消息")
        done <- true
    }()
    <-done
}
```

## 6. 闭包及其陷阱

### 6.1 闭包基础

```go
package main

import "fmt"

// 闭包：函数 + 它引用的外部变量

// 计数器工厂
func makeCounter(start int) func() int {
    count := start
    return func() int {
        count++ // 引用外部变量 count
        return count
    }
}

// 累加器
func makeAccumulator() func(int) int {
    total := 0
    return func(n int) int {
        total += n
        return total
    }
}

// 限流器
func makeRateLimiter(maxCalls int) func() bool {
    calls := 0
    return func() bool {
        if calls >= maxCalls {
            return false // 超过限制
        }
        calls++
        return true
    }
}

// 缓存工厂（简单的 memoization）
func memoize(fn func(int) int) func(int) int {
    cache := make(map[int]int)
    return func(n int) int {
        if v, ok := cache[n]; ok {
            fmt.Printf("  缓存命中: fib(%d) = %d\n", n, v)
            return v
        }
        result := fn(n)
        cache[n] = result
        return result
    }
}

func main() {
    // 计数器
    fmt.Println("=== 计数器 ===")
    counter1 := makeCounter(0)
    counter2 := makeCounter(100)

    fmt.Println(counter1()) // 1
    fmt.Println(counter1()) // 2
    fmt.Println(counter1()) // 3
    fmt.Println(counter2()) // 101（独立的闭包，互不影响）
    fmt.Println(counter2()) // 102

    // 累加器
    fmt.Println("\n=== 累加器 ===")
    acc := makeAccumulator()
    fmt.Println(acc(10))  // 10
    fmt.Println(acc(20))  // 30
    fmt.Println(acc(30))  // 60

    // 限流器
    fmt.Println("\n=== 限流器 ===")
    limiter := makeRateLimiter(3)
    for i := 0; i < 5; i++ {
        fmt.Printf("  请求 %d: 允许=%v\n", i+1, limiter())
    }

    // 缓存
    fmt.Println("\n=== 缓存 ===")
    var fib func(int) int
    fib = func(n int) int {
        if n <= 1 {
            return n
        }
        return fib(n-1) + fib(n-2)
    }
    cachedFib := memoize(fib)
    fmt.Println("fib(10) =", cachedFib(10))
    fmt.Println("fib(10) =", cachedFib(10)) // 缓存命中
}
```

### 6.2 闭包陷阱

```go
package main

import (
    "fmt"
    "sync"
)

func main() {
    // ============ 陷阱一：循环中的闭包（Go 1.22 之前） ============
    fmt.Println("=== 循环闭包 ===")

    // Go 1.22+ 已修复此问题，每次迭代创建新变量
    funcs := make([]func(), 5)
    for i := 0; i < 5; i++ {
        funcs[i] = func() {
            fmt.Printf("  i = %d\n", i)
        }
    }
    // Go 1.22+: 输出 0,1,2,3,4
    // Go 1.21-: 输出 5,5,5,5,5
    for _, fn := range funcs {
        fn()
    }

    // 兼容旧版本的安全写法
    fmt.Println("\n=== 安全的闭包写法 ===")
    safeFuncs := make([]func(), 5)
    for i := 0; i < 5; i++ {
        i := i // 创建局部副本（Go 1.21 及之前需要这行）
        safeFuncs[i] = func() {
            fmt.Printf("  i = %d\n", i)
        }
    }
    for _, fn := range safeFuncs {
        fn()
    }

    // ============ 陷阱二：goroutine 中的闭包 ============
    fmt.Println("\n=== goroutine 闭包 ===")
    var wg sync.WaitGroup
    for i := 0; i < 5; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            fmt.Printf("  goroutine i = %d\n", i)
        }()
    }
    wg.Wait()

    // ============ 陷阱三：闭包修改外部变量 ============
    fmt.Println("\n=== 闭包修改外部变量 ===")
    x := 0
    inc := func() { x++ }
    dec := func() { x-- }

    inc()
    inc()
    inc()
    dec()
    fmt.Println("x =", x) // 2（闭包共享同一个 x）

    // 注意：多个 goroutine 共享闭包变量时需要加锁！
    // 否则会有数据竞争
}
```

## 7. init 函数

```go
package main

import "fmt"

// init 函数的特点：
// 1. 没有参数和返回值
// 2. 不能被手动调用
// 3. 在 main 函数之前自动执行
// 4. 同一个包内可以有多个 init 函数
// 5. 同一个文件内也可以有多个 init 函数
// 6. 执行顺序：依赖包的 init → 当前包的 init → main

// 包级变量先初始化
var config = loadDefaultConfig()

func loadDefaultConfig() map[string]string {
    fmt.Println("1. 包级变量初始化")
    return map[string]string{
        "host": "localhost",
        "port": "8080",
    }
}

// 第一个 init
func init() {
    fmt.Println("2. 第一个 init 函数执行")
    // 典型用途：验证配置
    if config["host"] == "" {
        panic("host 不能为空")
    }
}

// 第二个 init（同一个文件内可以有多个）
func init() {
    fmt.Println("3. 第二个 init 函数执行")
    // 典型用途：注册驱动、初始化连接池等
}

func main() {
    fmt.Println("4. main 函数执行")
    fmt.Println("配置:", config)
}

// 输出顺序：
// 1. 包级变量初始化
// 2. 第一个 init 函数执行
// 3. 第二个 init 函数执行
// 4. main 函数执行
```

### init 的常见使用场景

```go
// ============ 场景一：数据库驱动注册 ============
// 在导入包时自动注册驱动
// import _ "github.com/lib/pq" // 只执行 init，不使用包内其他内容
//
// pq 包的 init 函数内部做了:
// func init() {
//     sql.Register("postgres", &Driver{})
// }

// ============ 场景二：检查运行环境 ============
// func init() {
//     if runtime.NumCPU() < 2 {
//         log.Println("警告: CPU 核心数不足，性能可能受影响")
//     }
//     if runtime.GOOS != "linux" {
//         log.Println("警告: 非 Linux 环境，某些功能可能不可用")
//     }
// }

// ============ 场景三：初始化全局配置 ============
// var (
//     appConfig *Config
// )
//
// func init() {
//     var err error
//     appConfig, err = LoadConfig("config.yaml")
//     if err != nil {
//         log.Fatalf("加载配置失败: %v", err)
//     }
// }
```

> **与 Python 对比**：Python 中模块级代码在 import 时自动执行，`if __name__ == "__main__"` 控制主入口。Go 用 `init()` 和 `main()` 分别处理初始化和入口，更加明确。

## 8. defer/panic/recover 组合用法

### 8.1 panic — 触发崩溃

```go
package main

import "fmt"

func main() {
```

### 8.2 recover — 恢复 panic

```go
package main

import (
    "fmt"
    "runtime/debug"
)

// safeDiv 安全的除法，不会因为除零而崩溃
func safeDiv(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            // recover() 捕获到 panic
            err = fmt.Errorf("运算异常: %v", r)
        }
    }()

    return a / b, nil // 如果 b=0，这里会 panic
}

// safeExecute 安全执行任意函数
func safeExecute(name string, fn func()) (err error) {
    defer func() {
        if r := recover(); r != nil {
            // 获取调用栈信息
            stack := debug.Stack()
            err = fmt.Errorf("函数 %s panic: %v\n调用栈:\n%s", name, r, stack)
        }
    }()

    fn()
    return nil
}

// mustGetEnv 获取环境变量，不存在则 panic
// 适用于程序启动时必须有的配置
func mustGetEnv(key string) string {
    // value := os.Getenv(key)
    value := "" // 模拟
    if value == "" {
        panic(fmt.Sprintf("必需的环境变量 %s 未设置", key))
    }
    return value
}

func main() {
    // ============ 安全除法 ============
    fmt.Println("=== 安全除法 ===")
    result, err := safeDiv(10, 3)
    fmt.Printf("10 / 3 = %d, err = %v\n", result, err)

    result, err = safeDiv(10, 0)
    fmt.Printf("10 / 0 = %d, err = %v\n", result, err) // 不会崩溃

    // ============ 安全执行 ============
    fmt.Println("\n=== 安全执行 ===")
    err = safeExecute("正常函数", func() {
        fmt.Println("  正常执行")
    })
    fmt.Println("错误:", err)

    err = safeExecute("危险函数", func() {
        panic("出大事了！")
    })
    fmt.Println("错误:", err) // 不会崩溃，错误被捕获

    // ============ 程序继续运行 ============
    fmt.Println("\n程序正常退出")
}
```

### 8.3 recover 的最佳实践

```go
package main

import (
    "fmt"
    "log"
    "net/http"
)

// ============ 实践一：HTTP 服务器中间件 ============
func recoveryMiddleware(next http.HandlerFunc) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                log.Printf("PANIC 恢复: %v, URL: %s", err, r.URL.Path)
                http.Error(w, "Internal Server Error", http.StatusInternalServerError)
            }
        }()
        next(w, r)
    }
}

// ============ 实践二：Worker Pool 中的错误隔离 ============
type Task struct {
    Name string
    Fn   func() error
}

type TaskResult struct {
    Name  string
    Error error
}

func executeTask(task Task) TaskResult {
    result := TaskResult{Name: task.Name}

    defer func() {
        if r := recover(); r != nil {
            result.Error = fmt.Errorf("任务 panic: %v", r)
        }
    }()

    result.Error = task.Fn()
    return result
}

func main() {
    // 执行多个任务，一个任务 panic 不影响其他任务
    tasks := []Task{
        {
            Name: "正常任务",
            Fn:   func() error { fmt.Println("  正常任务完成"); return nil },
        },
        {
            Name: "Panic 任务",
            Fn:   func() error { panic("数组越界"); return nil },
        },
        {
            Name: "错误任务",
            Fn:   func() error { return fmt.Errorf("连接超时") },
        },
        {
            Name: "另一个正常任务",
            Fn:   func() error { fmt.Println("  另一个正常任务完成"); return nil },
        },
    }

    fmt.Println("=== 执行任务 ===")
    results := make([]TaskResult, len(tasks))
    for i, task := range tasks {
        results[i] = executeTask(task)
    }

    fmt.Println("\n=== 结果汇总 ===")
    for _, r := range results {
        if r.Error != nil {
            fmt.Printf("  [失败] %s: %v\n", r.Name, r.Error)
        } else {
            fmt.Printf("  [成功] %s\n", r.Name)
        }
    }
}
```

### 8.4 panic/recover 使用准则

```go
package main

import "fmt"

func main() {
    fmt.Println("=== panic/recover 使用准则 ===")
    fmt.Println()
    fmt.Println("什么时候应该 panic?")
    fmt.Println("  1. 程序启动时缺少必要配置")
    fmt.Println("  2. 不可恢复的编程错误（数组越界等）")
    fmt.Println("  3. init() 函数中的致命错误")
    fmt.Println()
    fmt.Println("什么时候应该 recover?")
    fmt.Println("  1. HTTP handler 中防止单个请求崩溃整个服务")
    fmt.Println("  2. goroutine 中防止一个 goroutine 崩溃整个程序")
    fmt.Println("  3. 第三方库可能 panic 的地方")
    fmt.Println()
    fmt.Println("什么时候不应该用 panic?")
    fmt.Println("  1. 普通的错误处理（用 error 返回值）")
    fmt.Println("  2. 文件不存在、网络超时等预期内的错误")
    fmt.Println("  3. 用户输入验证失败")
    fmt.Println()
    fmt.Println("Go 的哲学：错误是值，不是异常。")
    fmt.Println("用 error 返回值处理预期错误，用 panic 处理不可恢复的意外。")
}
```

> **与 Python 对比**：
> - Python: `try/except/finally` — 异常是主要的错误处理方式
> - Go: `error 返回值` + `defer/panic/recover` — 错误是返回值，panic 只在极端情况使用
> - Go 的方式更显式，每个错误都必须处理；Python 的异常可能被意外吞掉

## 实战场景：带重试和超时的 HTTP 客户端

```go
package main

import (
    "context"
    "fmt"
    "math/rand"
    "strings"
    "time"
)

// HTTPResponse 模拟 HTTP 响应
type HTTPResponse struct {
    StatusCode int
    Body       string
    Latency    time.Duration
}

// HTTPClient 带重试的 HTTP 客户端
type HTTPClient struct {
    MaxRetries int
    Timeout    time.Duration
    RetryDelay time.Duration
}

// NewHTTPClient 创建客户端
func NewHTTPClient(maxRetries int, timeout, retryDelay time.Duration) *HTTPClient {
    return &HTTPClient{
        MaxRetries: maxRetries,
        Timeout:    timeout,
        RetryDelay: retryDelay,
    }
}

// mockHTTPGet 模拟 HTTP GET 请求
func mockHTTPGet(url string) (HTTPResponse, error) {
    latency := time.Duration(rand.Intn(200)) * time.Millisecond

    // 模拟不同响应
    r := rand.Float64()
    switch {
    case r < 0.5:
        return HTTPResponse{200, `{"status":"ok"}`, latency}, nil
    case r < 0.7:
        return HTTPResponse{500, "Internal Server Error", latency}, nil
    case r < 0.85:
        return HTTPResponse{503, "Service Unavailable", latency}, nil
    default:
        return HTTPResponse{}, fmt.Errorf("连接超时: %s", url)
    }
}

// Get 执行 GET 请求（带重试）
func (c *HTTPClient) Get(ctx context.Context, url string) (HTTPResponse, error) {
    var lastErr error

    for attempt := 0; attempt <= c.MaxRetries; attempt++ {
        // 检查 context 是否已取消
        select {
        case <-ctx.Done():
            return HTTPResponse{}, fmt.Errorf("请求取消: %w", ctx.Err())
        default:
        }

        if attempt > 0 {
            delay := c.RetryDelay * time.Duration(attempt) // 指数退避
            fmt.Printf("    等待 %v 后重试...\n", delay)
            time.Sleep(delay)
        }

        fmt.Printf("  [尝试 %d/%d] GET %s\n", attempt+1, c.MaxRetries+1, url)

        resp, err := mockHTTPGet(url)
        if err != nil {
            lastErr = err
            fmt.Printf("    错误: %v\n", err)
            continue
        }

        // 5xx 错误重试
        if resp.StatusCode >= 500 {
            lastErr = fmt.Errorf("HTTP %d: %s", resp.StatusCode, resp.Body)
            fmt.Printf("    服务端错误: %v\n", lastErr)
            continue
        }

        // 4xx 错误不重试
        if resp.StatusCode >= 400 {
            return resp, fmt.Errorf("客户端错误 HTTP %d", resp.StatusCode)
        }

        // 成功
        fmt.Printf("    成功! 延迟: %v\n", resp.Latency)
        return resp, nil
    }

    return HTTPResponse{}, fmt.Errorf("重试 %d 次后失败: %w", c.MaxRetries+1, lastErr)
}

// BatchGet 批量请求
func (c *HTTPClient) BatchGet(ctx context.Context, urls []string) map[string]error {
    results := make(map[string]error)
    for _, url := range urls {
        _, err := c.Get(ctx, url)
        results[url] = err
    }
    return results
}

func main() {
    client := NewHTTPClient(2, 5*time.Second, 100*time.Millisecond)

    urls := []string{
        "http://api.example.com/health",
        "http://db.example.com/status",
        "http://cache.example.com/ping",
    }

    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()

    fmt.Println(strings.Repeat("=", 60))
    fmt.Println(" 批量健康检查")
    fmt.Println(strings.Repeat("=", 60))

    results := client.BatchGet(ctx, urls)

    fmt.Println("\n" + strings.Repeat("-", 60))
    fmt.Println(" 结果汇总")
    fmt.Println(strings.Repeat("-", 60))

    for url, err := range results {
        if err != nil {
            fmt.Printf("  [失败] %s: %v\n", url, err)
        } else {
            fmt.Printf("  [成功] %s\n", url)
        }
    }
}
```

## 常见坑点

### 坑点 1：defer 的参数在声明时求值

```go
package main

import "fmt"

func main() {
    x := 0
    defer fmt.Println("defer 值:", x) // x 在这里求值为 0
    x = 100
    fmt.Println("当前值:", x) // 100
    // 输出:
    // 当前值: 100
    // defer 值: 0    ← 不是 100！
}
```

### 坑点 2：recover 必须在 defer 函数中直接调用

```go
package main

import "fmt"

func main() {
    // 错误：recover 不在 defer 中
    // if r := recover(); r != nil {} // 不起作用

    // 错误：recover 在嵌套函数中
    defer func() {
        // 这样不行！
        func() {
            if r := recover(); r != nil {
                fmt.Println("嵌套中的 recover:", r) // 捕获不到
            }
        }()
    }()

    // 正确：recover 直接在 defer 函数中
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("正确的 recover:", r)
        }
    }()

    panic("测试")
}
```

### 坑点 3：闭包中的循环变量

```go
package main

import "fmt"

func main() {
    // 创建一组错误处理函数
    type handler struct {
        name string
        fn   func()
    }

    names := []string{"web", "api", "db"}
    handlers := make([]handler, 0)

    for _, name := range names {
        handlers = append(handlers, handler{
            name: name,
            fn: func() {
                // Go 1.22+: name 在每次迭代中是新变量
                fmt.Printf("处理 %s 服务\n", name)
            },
        })
    }

    for _, h := range handlers {
        h.fn()
    }
}
```

## 速查表

### 函数声明模式

| 模式 | 语法 | 说明 |
|------|------|------|
| 基本函数 | `func name(a int) int` | 最常见 |
| 多返回值 | `func name() (int, error)` | 错误处理标准模式 |
| 命名返回值 | `func name() (n int, err error)` | 配合 defer 使用 |
| 可变参数 | `func name(args ...int)` | 类似 Python 的 *args |
| 函数类型 | `type Fn func(int) int` | 定义函数签名 |
| 匿名函数 | `func(x int) int { return x }` | Lambda 表达式 |
| 闭包 | `func() func() int { ... }` | 捕获外部变量 |
| 方法 | `func (s *Server) Start()` | 下一篇讲 |

### defer/panic/recover 速查

| 操作 | 说明 |
|------|------|
| `defer fn()` | 延迟到函数返回前执行 |
| `defer func() { ... }()` | 延迟执行匿名函数 |
| `panic(v)` | 触发崩溃，v 可以是任何类型 |
| `recover()` | 捕获 panic，必须在 defer 中调用 |
| 执行顺序 | 多个 defer 按 LIFO（栈）顺序执行 |
| panic 时 | 当前函数的 defer 仍然执行 |
| recover 返回 | panic 的参数值，无 panic 时返回 nil |

### 与 Python 对比

| 特性 | Go | Python |
|------|-----|--------|
| 错误处理 | `err` 返回值 | `try/except` |
| 资源清理 | `defer` | `with` / `finally` |
| 崩溃恢复 | `defer + recover` | `except Exception` |
| 可变参数 | `...T` | `*args` |
| 关键字参数 | 不支持 | `**kwargs` |
| 默认参数 | 不支持 | `def f(x=1)` |
| Lambda | 匿名函数 | `lambda x: x` |
| 装饰器 | 高阶函数 | `@decorator` |

## 小结

1. **多返回值是 Go 错误处理的基石**：`result, err := fn()` 是你会写最多的模式。
2. **命名返回值 + defer** 可以优雅地包装错误信息。
3. **可变参数**（`...T`）不如 Python 灵活，但够用。复杂参数用 struct 或 functional options。
4. **闭包是强大的工具**：计数器、缓存、限流器、中间件都可以用闭包实现。
5. **init 函数谨慎使用**：适合驱动注册、环境检查，不适合复杂初始化逻辑。
6. **defer/panic/recover 不是 try-catch**：Go 推崇用 error 返回值处理预期错误，panic 只用于不可恢复的情况。
7. **recover 的典型场景**：HTTP handler 防崩溃、goroutine 错误隔离、第三方库兜底。
