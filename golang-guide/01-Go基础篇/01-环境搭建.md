## 导读

作为 SRE/DevOps 工程师，你可能已经习惯了 Python 的 `pip install` 和 Shell 的开箱即用。Go 的工具链同样简洁——单一二进制即可完成编译、测试、格式化、静态分析等全部工作。本篇将从零开始，帮你在 10 分钟内搭好一个生产级的 Go 开发环境。

## 1. Go 安装

### 1.1 Linux（推荐方式）

```bash
# 下载最新版本（以 Go 1.24.1 为例）
GO_VERSION="1.24.1"
wget "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz"

# 删除旧版本并解压到 /usr/local
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf "go${GO_VERSION}.linux-amd64.tar.gz"

# 配置环境变量（追加到 ~/.bashrc 或 ~/.zshrc）
cat >> ~/.bashrc << 'EOF'
# Go 环境变量
export GOROOT=/usr/local/go
export GOPATH=$HOME/go
export PATH=$PATH:$GOROOT/bin:$GOPATH/bin
EOF

# 使配置生效
source ~/.bashrc

# 验证安装
go version
# 输出: go version go1.24.1 linux/amd64
```

> **与 Python 对比**：Python 有 pyenv、conda、系统自带等多种安装方式，版本管理比较混乱。Go 官方推荐只保留一个版本，通过 Go Modules 管理依赖，非常干净。

### 1.2 macOS

```bash
# 方式一：Homebrew（推荐）
brew install go

# 方式二：官方安装包
# 下载 https://go.dev/dl/ 中的 .pkg 文件，双击安装
# 默认安装到 /usr/local/go

# 验证
go version
```

### 1.3 Windows

```powershell
# 方式一：Scoop（推荐，类似 Linux 包管理器）
scoop install go

# 方式二：官方安装包
# 下载 https://go.dev/dl/ 中的 .msi 文件，双击安装
# 默认安装到 C:\Go

# 验证
go version
```

### 1.4 使用 Docker（CI/CD 场景推荐）

```dockerfile
# Dockerfile
FROM golang:1.24-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o /app/myservice ./cmd/myservice

FROM alpine:3.19
COPY --from=builder /app/myservice /usr/local/bin/
ENTRYPOINT ["myservice"]
```

```bash
# 也可以直接用 Docker 运行 Go 命令
docker run --rm -v "$PWD":/app -w /app golang:1.24 go build -o myservice .
```

### 1.5 多版本管理

```bash

go1.23.6 download

# 使用指定版本
go1.23.6 version

```

## 2. GOPATH vs Go Modules 的历史演变

### 2.1 GOPATH 时代（Go 1.0 - Go 1.10）

```
# 早期 Go 的项目结构必须放在 GOPATH 下
$GOPATH/
├── src/             # 所有源码
│   ├── github.com/
│   │   └── user/
│   │       └── project/
│   │           └── main.go
│   └── golang.org/
│       └── x/
│           └── tools/
├── pkg/             # 编译缓存（.a 文件）
│   └── linux_amd64/
└── bin/             # 可执行文件
    └── project
```

**GOPATH 的痛点**：

```bash
# 1. 所有项目必须放在 $GOPATH/src 下
# 2. 没有版本管理，go get 永远拉最新代码
# 3. 无法在同一项目中使用同一个库的不同版本

# 当年的依赖管理 —— 没有版本号
go get github.com/gin-gonic/gin  # 永远拉 master 分支最新代码
```

### 2.2 过渡期（Go 1.11 - Go 1.15）

```bash
# Go 1.11 引入 Go Modules（实验性）
export GO111MODULE=on   # 手动开启

# Go 1.13 默认在 GOPATH 外的目录启用 Modules
# Go 1.16 默认全面启用 Go Modules
```

### 2.3 Go Modules 时代（Go 1.16+，当前）

```bash
# 现在：项目可以放在任何目录
mkdir -p ~/projects/myservice
cd ~/projects/myservice

# 初始化模块
go mod init github.com/myorg/myservice

# 添加依赖（自动管理版本）
go get github.com/gin-gonic/gin@v1.9.1

# 项目结构
myservice/
├── go.mod          # 模块定义 + 依赖版本（类似 requirements.txt）
├── go.sum          # 依赖哈希校验（类似 pip 的 hash 模式）
├── cmd/
│   └── myservice/
│       └── main.go
├── internal/       # 内部包，外部无法导入
│   └── handler/
└── pkg/            # 公共包，可被外部导入
    └── util/
```

> **与 Python 对比**：`go.mod` 类似 `requirements.txt` + `pyproject.toml`，`go.sum` 类似 `pip` 的 `--require-hashes`。但 Go Modules 是语言内置的，不需要额外工具。

## 3. go env 配置详解

```bash
# 查看所有环境变量
go env

# 查看指定变量
go env GOPATH GOROOT GOPROXY

# 设置环境变量（写入 $HOME/.config/go/env）
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOPRIVATE=*.corp.example.com
go env -w GONOSUMCHECK=*.corp.example.com

# 重置为默认值
go env -u GOPROXY
```

### 3.1 关键环境变量说明

```bash
# ==================== 路径相关 ====================

# GOROOT - Go 安装目录，一般不需要手动设置
go env GOROOT
# /usr/local/go

# GOPATH - 工作目录，存放下载的依赖和编译的二进制
go env GOPATH
# /home/devops/go

# GOBIN - go install 安装二进制的目录
# 默认为 $GOPATH/bin
go env -w GOBIN=$HOME/.local/bin  # 自定义

# GOMODCACHE - 模块缓存目录
go env GOMODCACHE
# /home/devops/go/pkg/mod

# GOCACHE - 编译缓存目录
go env GOCACHE
# /home/devops/.cache/go-build

# ==================== 编译相关 ====================

# GOOS - 目标操作系统
# 可选: linux, darwin, windows, freebsd 等
go env GOOS

# GOARCH - 目标架构
# 可选: amd64, arm64, 386, arm 等
go env GOARCH

# CGO_ENABLED - 是否启用 CGO
# 0: 禁用（纯静态编译，推荐容器场景）
# 1: 启用（需要 C 编译器）
go env CGO_ENABLED

# ==================== 模块/代理相关 ====================

# GOPROXY - 模块代理地址
go env GOPROXY
# https://proxy.golang.org,direct（默认）

# GOPRIVATE - 私有模块，跳过代理和校验
go env GOPRIVATE

# GONOSUMCHECK - 跳过 checksum 校验
go env GONOSUMCHECK

# GONOSUMDB - 跳过 sum 数据库查询
go env GONOSUMDB

# GONOPROXY - 跳过代理直接拉取
go env GONOPROXY
```

### 3.2 交叉编译实战

```bash
# 编译 Linux amd64 二进制
GOOS=linux GOARCH=amd64 go build -o myservice-linux-amd64 .

# 编译 macOS ARM（Apple Silicon）
GOOS=darwin GOARCH=arm64 go build -o myservice-darwin-arm64 .

# 编译 Windows
GOOS=windows GOARCH=amd64 go build -o myservice.exe .

# 查看 Go 支持的所有平台
go tool dist list
# 输出示例:
# aix/ppc64
# android/amd64
# darwin/amd64
# darwin/arm64
# linux/amd64
# linux/arm64
# windows/amd64
# ... 共 50+ 种组合
```

> **与 Python 对比**：Python 不支持交叉编译。Go 天生支持交叉编译，编译出的是静态链接的单一二进制，非常适合部署到容器或远程服务器。

## 6. go 命令详解

### 6.1 go run - 编译并运行

```bash
# 编译并运行单个文件
go run main.go

# 运行包含多个文件的 main 包
go run .
go run ./cmd/myservice

# 传递参数给程序
go run main.go -port 8080 -config config.yaml

# 等价于:
go build -o /tmp/main main.go && /tmp/main
```

> **与 Python 对比**：`go run main.go` 类似 `python main.py`，但 Go 会先编译再执行。

### 6.2 go build - 编译

```bash
# 编译当前目录的 main 包
go build
# 生成可执行文件: ./myservice（以模块名命名）

# 指定输出文件名
go build -o myservice-v2 .

# 编译指定包
go build ./cmd/myservice

# 带版本信息的编译（推荐 CI/CD 使用）
VERSION=$(git describe --tags --always --dirty)
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
GIT_COMMIT=$(git rev-parse --short HEAD)

go build -ldflags "-X main.Version=${VERSION} \
    -X main.BuildTime=${BUILD_TIME} \
    -X main.GitCommit=${GIT_COMMIT} \
    -s -w" \
    -o myservice ./cmd/myservice

# -s: 去掉符号表
# -w: 去掉 DWARF 调试信息
# 这两个参数可以减小二进制约 30%

# 查看编译过程（排查问题时有用）
go build -v ./...

# 静态编译（推荐容器部署）
CGO_ENABLED=0 go build -a -installsuffix cgo -o myservice .
```

### 6.3 go install - 编译并安装

```bash
# 安装到 $GOBIN 或 $GOPATH/bin
go install .

# 安装远程工具
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest

# 安装指定版本
go install golang.org/x/tools/gopls@v0.17.1
```

### 6.4 go get - 管理依赖

```bash
# 添加依赖
go get github.com/gin-gonic/gin@v1.9.1

# 更新依赖到最新版本
go get -u github.com/gin-gonic/gin

# 更新所有直接依赖
go get -u ./...

# 更新所有依赖（包括间接依赖）
go get -u all

# 降级/指定版本
go get github.com/gin-gonic/gin@v1.8.0

# 移除依赖（将版本设为 none）
go get github.com/gin-gonic/gin@none

# 下载依赖到本地缓存（不修改 go.mod）
go mod download
```

### 6.5 go mod - 模块管理

```bash
# 初始化模块
go mod init github.com/myorg/myservice

# 整理依赖（删除未使用的，添加缺失的）
go mod tidy

# 下载所有依赖到本地缓存
go mod download

# 将依赖复制到项目的 vendor 目录
go mod vendor

# 验证依赖的完整性
go mod verify

# 查看依赖图
go mod graph

# 查看某个依赖为什么被引入
go mod why github.com/some/package

# 编辑 go.mod（脚本化场景有用）
go mod edit -require github.com/gin-gonic/gin@v1.9.1
go mod edit -replace github.com/old/pkg=github.com/new/pkg@v1.0.0
go mod edit -replace github.com/some/pkg=../local-pkg  # 本地替换，开发调试用
```

### 6.6 go fmt / goimports - 格式化

```bash
# 格式化当前包
go fmt ./...

# goimports 格式化 + 自动整理 import（推荐）
goimports -w .

# gofumpt 更严格的格式化（推荐）
gofumpt -w .
```

> **与 Python 对比**：Go 有官方统一的代码风格（gofmt），相当于 Python 的 black，但是 Go 社区 100% 使用 gofmt，没有风格之争。

### 6.7 go vet - 静态分析

```bash
# 检查常见错误
go vet ./...

# 常见检查项:
# - Printf 格式字符串与参数不匹配
# - 未使用的变量
# - 不可达的代码
# - 锁的错误使用
# - 结构体复制时包含互斥锁
```

```go
// go vet 能发现的错误示例
package main

import "fmt"

func main() {
    name := "World"
    fmt.Printf("Hello %d\n", name) // go vet 报错: Printf 格式 %d 期望 int，实际是 string
}
```

### 6.8 go test - 测试

```bash
# 运行当前包的测试
go test

# 运行所有包的测试
go test ./...

# 详细输出
go test -v ./...

# 运行指定测试函数
go test -run TestMyFunction ./...

# 显示代码覆盖率
go test -cover ./...

# 生成覆盖率报告
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out -o coverage.html

# 运行基准测试
go test -bench=. -benchmem ./...

# 竞态检测（推荐 CI 中使用）
go test -race ./...

# 设置超时
go test -timeout 30s ./...

# 禁用缓存（总是重新运行）
go test -count=1 ./...
```

## 7. 第一个 Hello World 程序

### 7.1 最简版本

```go
// main.go
// Go 程序必须属于一个包，可执行程序的入口包必须是 main
package main

// 导入标准库的 fmt 包，用于格式化输入输出
import "fmt"

// main 函数是程序入口，不接受参数也不返回值
func main() {
    // Println 会自动换行
    fmt.Println("Hello, World!")
}
```

```bash
# 运行
go run main.go
# 输出: Hello, World!
```

### 7.2 SRE 实用版本 — 带命令行参数和版本信息

```go
// main.go
package main

import (
    "flag"
    "fmt"
    "os"
    "runtime"
)

// 这些变量会在编译时通过 -ldflags 注入
var (
    Version   = "dev"
    BuildTime = "unknown"
    GitCommit = "unknown"
)

func main() {
    // 定义命令行参数
    showVersion := flag.Bool("version", false, "显示版本信息")
    name := flag.String("name", "World", "要问候的名字")
    port := flag.Int("port", 8080, "监听端口")

    // 解析命令行参数
    flag.Parse()

    // 如果传入 -version 参数，打印版本信息并退出
    if *showVersion {
        fmt.Printf("Version:    %s\n", Version)
        fmt.Printf("Build Time: %s\n", BuildTime)
        fmt.Printf("Git Commit: %s\n", GitCommit)
        fmt.Printf("Go Version: %s\n", runtime.Version())
        fmt.Printf("OS/Arch:    %s/%s\n", runtime.GOOS, runtime.GOARCH)
        os.Exit(0)
    }

    fmt.Printf("Hello, %s! 服务将在端口 %d 启动。\n", *name, *port)
}
```

```bash
# 编译并注入版本信息
go build -ldflags "-X main.Version=v1.0.0 \
    -X main.BuildTime=$(date -u +%Y-%m-%dT%H:%M:%SZ) \
    -X main.GitCommit=$(git rev-parse --short HEAD 2>/dev/null || echo 'none')" \
    -o hello main.go

# 运行
./hello
# 输出: Hello, World! 服务将在端口 8080 启动。

./hello -name DevOps -port 9090
# 输出: Hello, DevOps! 服务将在端口 9090 启动。

./hello -version
# 输出:
# Version:    v1.0.0
# Build Time: 2025-03-15T10:30:00Z
# Git Commit: abc1234
# Go Version: go1.24.1
# OS/Arch:    linux/amd64
```

## 8. 项目初始化 go mod init

### 8.1 基本用法

```bash
# 创建项目目录
mkdir -p ~/projects/myservice
cd ~/projects/myservice

# 初始化模块（模块路径通常是代码仓库地址）
go mod init github.com/myorg/myservice

# 查看生成的 go.mod 文件
cat go.mod
```

```
module github.com/myorg/myservice

go 1.24
```

### 8.2 推荐的项目结构

```bash
# 标准 Go 项目结构（参考 golang-standards/project-layout）
myservice/
├── cmd/                    # 可执行文件入口
```

```go
// cmd/health-checker/main.go
package main

import (
    "fmt"
    "log"
    "net/http"
    "os"
    "time"

    "github.com/myorg/health-checker/internal/checker"
)

func main() {
    // 从环境变量或默认值获取配置
    port := getEnv("PORT", "8080")
    targets := []string{
        "https://www.google.com",
        "https://github.com",
    }

    // 创建健康检查器
    hc := checker.New(targets, 5*time.Second)

    // 注册 HTTP 处理器
    http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        results := hc.CheckAll()
        for _, result := range results {
            status := "UP"
            if !result.Healthy {
                status = "DOWN"
                w.WriteHeader(http.StatusServiceUnavailable)
            }
            fmt.Fprintf(w, "%s: %s (%v)\n", result.Target, status, result.Latency)
        }
    })

    log.Printf("健康检查服务启动在端口 %s\n", port)
    if err := http.ListenAndServe(":"+port, nil); err != nil {
        log.Fatal(err)
    }
}

// getEnv 获取环境变量，如果不存在则返回默认值
func getEnv(key, defaultValue string) string {
    if value := os.Getenv(key); value != "" {
        return value
    }
    return defaultValue
}
```

```go
// internal/checker/checker.go
package checker

import (
    "net/http"
    "sync"
    "time"
)

// Result 表示一次健康检查的结果
type Result struct {
    Target  string
    Healthy bool
    Latency time.Duration
    Error   string
}

// Checker 是健康检查器
type Checker struct {
    targets []string
    timeout time.Duration
    client  *http.Client
}

// New 创建一个新的健康检查器
func New(targets []string, timeout time.Duration) *Checker {
    return &Checker{
        targets: targets,
        timeout: timeout,
        client: &http.Client{
            Timeout: timeout,
        },
    }
}

// CheckAll 并发检查所有目标
func (c *Checker) CheckAll() []Result {
    results := make([]Result, len(c.targets))
    var wg sync.WaitGroup

    for i, target := range c.targets {
        wg.Add(1)
        go func(idx int, url string) {
            defer wg.Done()
            results[idx] = c.check(url)
        }(i, target)
    }

    wg.Wait()
    return results
}

// check 检查单个目标的健康状态
func (c *Checker) check(target string) Result {
    start := time.Now()

    resp, err := c.client.Get(target)
    latency := time.Since(start)

    if err != nil {
        return Result{
            Target:  target,
            Healthy: false,
            Latency: latency,
            Error:   err.Error(),
        }
    }
    defer resp.Body.Close()

    return Result{
        Target:  target,
        Healthy: resp.StatusCode < 400,
        Latency: latency,
    }
}
```

## 常见坑点

### 坑点 1：GOPATH 和 Go Modules 混淆

```bash
# 错误：在 GOPATH/src 下使用 go mod init
# 如果 GO111MODULE 没有明确设为 on，可能会导致混乱

# 正确做法：
# 1. 项目放在 GOPATH 外面
# 2. 或者确保 GO111MODULE=on（Go 1.16+ 默认）
go env -w GO111MODULE=on
```

### 坑点 2：忘记运行 go mod tidy

```bash
# 添加了新的 import 但没有运行 go mod tidy
# 会导致 go build 失败

# 养成习惯：每次修改依赖后运行
go mod tidy
```

### 坑点 3：GOPROXY 未配置导致下载超时

```bash
# 国内网络直接访问 proxy.golang.org 可能超时
# 报错: context deadline exceeded

# 解决：
go env -w GOPROXY=https://goproxy.cn,direct
```

### 坑点 4：VS Code 没有安装 Go 工具

```bash
# 如果 VS Code 的 Go 扩展提示缺少工具
# 按 Ctrl+Shift+P → Go: Install/Update Tools → 全选安装

# 或者手动安装
go install golang.org/x/tools/gopls@latest
go install github.com/go-delve/delve/cmd/dlv@latest
```

### 坑点 5：权限问题

```bash
# 错误: go install 报 permission denied
# 原因: $GOPATH/bin 没有写权限

# 解决:
mkdir -p "$GOPATH/bin"
chmod 755 "$GOPATH/bin"

# 或者自定义 GOBIN
go env -w GOBIN=$HOME/.local/bin
```

## 小结

1. **Go 的工具链是语言自带的**：不需要像 Python 那样安装一堆第三方工具（black、pylint、pytest 等），`go` 命令一站式搞定。
2. **Go Modules 是唯一的依赖管理方式**：忘掉 GOPATH，拥抱 `go.mod`。
3. **交叉编译是 Go 的杀手锏**：一条命令编译出任何平台的二进制，特别适合 SRE 的多平台部署场景。
4. **GOPROXY 是国内开发者的必配项**：配置好代理，省去无数下载超时的烦恼。
5. **Makefile 是 Go 项目的标配**：把常用命令封装好，团队协作更高效。
