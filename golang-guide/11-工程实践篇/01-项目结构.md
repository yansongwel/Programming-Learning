# Go 项目结构最佳实践

## 导读

良好的项目结构是大型 Go 项目成功的基础。它影响团队协作效率、代码可维护性、测试难度和部署复杂度。本篇将系统讲解 Go 项目的标准布局、分层架构、依赖注入和 Makefile 自动化，帮助 SRE/DevOps 工程师构建可维护的 Go 项目。

**本篇核心知识点：**
- Go 项目标准布局（cmd/internal/pkg/api 等）
- 各目录职责说明
- Clean Architecture 三层架构
- 依赖注入（手动注入 vs wire/fx）
- 项目初始化模板
- Makefile 编写
- 小项目 vs 大项目结构选择

### 1.2 各目录职责详解

```go
// directory_roles.go
// 演示各目录的职责和用法
package main

import "fmt"

func main() {
	directories := []struct {
		Path        string
		Role        string
		ImportRule  string
	}{
		{
			Path:       "cmd/",
			Role:       "主程序入口。每个子目录是一个独立的可执行文件。main.go 尽量精简，只做初始化和依赖组装。",
			ImportRule: "可以导入 internal/ 和 pkg/",
		},
		{
			Path:       "internal/",
			Role:       "私有代码。Go 编译器强制保证 internal/ 下的代码不能被项目外部导入。核心业务逻辑放在这里。",
			ImportRule: "只能被同项目的代码导入",
		},
		{
			Path:       "pkg/",
			Role:       "公共库。可以被外部项目导入的通用代码。只放真正通用的、稳定的、与业务无关的代码。",
			ImportRule: "可以被任何项目导入（慎重）",
		},
		{
			Path:       "api/",
			Role:       "API 定义文件。OpenAPI spec、Protobuf 定义、GraphQL schema 等。",
			ImportRule: "定义文件，非 Go 代码",
		},
		{
			Path:       "configs/",
			Role:       "配置文件模板。不包含敏感信息（密码、密钥等）。",
			ImportRule: "配置文件，非 Go 代码",
		},
		{
			Path:       "deployments/",
			Role:       "部署相关文件。Dockerfile、K8s manifests、docker-compose 等。",
			ImportRule: "部署配置，非 Go 代码",
		},
		{
			Path:       "scripts/",
			Role:       "自动化脚本。开发环境设置、数据库迁移、代码生成等。",
			ImportRule: "Shell/Python 脚本",
		},
		{
			Path:       "test/",
			Role:       "集成测试和测试数据。单元测试放在对应的包目录中（_test.go）。",
			ImportRule: "测试代码",
		},
	}

	fmt.Println("=== Go 项目目录职责 ===\n")
	for _, d := range directories {
		fmt.Printf("%-15s\n", d.Path)
		fmt.Printf("  职责: %s\n", d.Role)
		fmt.Printf("  导入规则: %s\n\n", d.ImportRule)
	}
}
```

```go
// cmd/server/main.go

// main.go 应该尽量精简，只负责：
// 1. 加载配置

// 4. 启动服务

package main

import (
	"context"
	"fmt"
	"log/slog"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"

// 版本信息（通过 -ldflags 注入）
var (
	version   = "dev"
	gitCommit = "unknown"
	buildDate = "unknown"
)

// Config 应用配置

		LogLevel:   getEnv("LOG_LEVEL", "info"),
	}
}

func getEnv(key, defaultValue string) string {
	if value := os.Getenv(key); value != "" {
		return value
	}
	return defaultValue
}

	// 1. 加载配置

		logLevel = slog.LevelDebug
	}
	logger := slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
		Level: logLevel,
	}))
	slog.SetDefault(logger)

	logger.Info("启动服务",
		"version", version,
		"commit", gitCommit,
		"build_date", buildDate,
	)

	// 3. 初始化依赖（数据库、缓存、外部服务等）
	// db, err := initDB(cfg)
	// cache, err := initRedis(cfg)
	// ...

	// 4. 创建服务层（依赖注入）
	// userRepo := repository.NewUserRepo(db)
	// userService := service.NewUserService(userRepo, cache)
	// userHandler := handler.NewUserHandler(userService)

	// 5. 设置路由
	mux := http.NewServeMux()
	mux.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Content-Type", "application/json")
		fmt.Fprintf(w, `{"status":"ok","version":"%s"}`, version)
	})
	mux.HandleFunc("/version", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintf(w, `{"version":"%s","commit":"%s","build":"%s"}`,
			version, gitCommit, buildDate)
	})
	// mux.Handle("/api/users", userHandler)

	// 6. 启动 HTTP 服务
	server := &http.Server{
		Addr:         cfg.ServerAddr,
		Handler:      mux,
		ReadTimeout:  15 * time.Second,
		WriteTimeout: 15 * time.Second,
		IdleTimeout:  60 * time.Second,
	}

	// 启动服务（非阻塞）
	go func() {
		logger.Info("HTTP 服务启动", "addr", cfg.ServerAddr)
		if err := server.ListenAndServe(); err != http.ErrServerClosed {
			logger.Error("HTTP 服务异常", "error", err)
			os.Exit(1)
		}
	}()

	// 7. 优雅关闭
	quit := make(chan os.Signal, 1)
	signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
	sig := <-quit
	logger.Info("收到关闭信号", "signal", sig.String())

	ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
	defer cancel()

	if err := server.Shutdown(ctx); err != nil {
		logger.Error("服务关闭失败", "error", err)
		return err
	}

	// 关闭其他资源
	// db.Close()
	// cache.Close()

	logger.Info("服务已关闭")
	return nil
}

func main() {
	if err := run(); err != nil {
		fmt.Fprintf(os.Stderr, "程序异常退出: %v\n", err)
		os.Exit(1)
	}
}
```

## 三、Clean Architecture 三层架构

### 3.1 架构概览

```
请求流程:
  HTTP Request → Handler → Service → Repository → Database
  HTTP Response ← Handler ← Service ← Repository ← Database

层次说明:
  ┌─────────────────────────────────────────┐
  │  Handler 层（表示层/控制层）               │
  │  - HTTP/gRPC 请求处理                    │
  │  - 参数校验、响应格式化                    │
  │  - 不包含业务逻辑                         │
  ├─────────────────────────────────────────┤
  │  Service 层（业务逻辑层）                  │
  │  - 核心业务规则                           │
  │  - 编排多个 Repository 的调用              │
  │  - 事务管理                              │
  ├─────────────────────────────────────────┤
  │  Repository 层（数据访问层）               │
  │  - 数据库 CRUD 操作                       │
  │  - 缓存操作                              │
  │  - 不包含业务逻辑                         │
  └─────────────────────────────────────────┘

依赖方向: Handler → Service → Repository
  上层依赖下层的接口（而非实现），便于测试和替换
```

### 3.3 Repository 层

```go
// internal/repository/user_repo.go
// 数据访问层：负责与数据库交互
package repository

import (
	"context"
	"database/sql"
	"fmt"
	"strings"
	"time"
)

// User 数据库模型（此处简化演示）
type User struct {
	ID        int64
	Username  string
	Email     string
	Password  string
	Role      string
	Active    bool
	CreatedAt time.Time
	UpdatedAt time.Time
}

// UserRepository 用户数据访问接口
// 通过接口解耦，便于 mock 测试
type UserRepository interface {
	Create(ctx context.Context, user *User) error
	GetByID(ctx context.Context, id int64) (*User, error)
	GetByUsername(ctx context.Context, username string) (*User, error)
	GetByEmail(ctx context.Context, email string) (*User, error)
	Update(ctx context.Context, user *User) error
	Delete(ctx context.Context, id int64) error
	List(ctx context.Context, offset, limit int, filters map[string]interface{}) ([]*User, int64, error)
}

// userRepo UserRepository 的 PostgreSQL 实现
type userRepo struct {
	db *sql.DB
}

// NewUserRepository 创建用户仓库实例
func NewUserRepository(db *sql.DB) UserRepository {
	return &userRepo{db: db}
}

func (r *userRepo) Create(ctx context.Context, user *User) error {
	query := `
		INSERT INTO users (username, email, password, role, active, created_at, updated_at)
		VALUES ($1, $2, $3, $4, $5, $6, $7)
		RETURNING id`

	now := time.Now()
	user.CreatedAt = now
	user.UpdatedAt = now

	return r.db.QueryRowContext(ctx, query,
		user.Username, user.Email, user.Password,
		user.Role, user.Active, user.CreatedAt, user.UpdatedAt,
	).Scan(&user.ID)
}

func (r *userRepo) GetByID(ctx context.Context, id int64) (*User, error) {
	query := `SELECT id, username, email, password, role, active, created_at, updated_at
		FROM users WHERE id = $1`

	user := &User{}
	err := r.db.QueryRowContext(ctx, query, id).Scan(
		&user.ID, &user.Username, &user.Email, &user.Password,
		&user.Role, &user.Active, &user.CreatedAt, &user.UpdatedAt,
	)
	if err == sql.ErrNoRows {
		return nil, nil // 返回 nil 表示未找到
	}
	return user, err
}

func (r *userRepo) GetByUsername(ctx context.Context, username string) (*User, error) {
	query := `SELECT id, username, email, password, role, active, created_at, updated_at
		FROM users WHERE username = $1`

	user := &User{}
	err := r.db.QueryRowContext(ctx, query, username).Scan(
		&user.ID, &user.Username, &user.Email, &user.Password,
		&user.Role, &user.Active, &user.CreatedAt, &user.UpdatedAt,
	)
	if err == sql.ErrNoRows {
		return nil, nil
	}
	return user, err
}

func (r *userRepo) GetByEmail(ctx context.Context, email string) (*User, error) {
	query := `SELECT id, username, email, password, role, active, created_at, updated_at
		FROM users WHERE email = $1`

	user := &User{}
	err := r.db.QueryRowContext(ctx, query, email).Scan(
		&user.ID, &user.Username, &user.Email, &user.Password,
		&user.Role, &user.Active, &user.CreatedAt, &user.UpdatedAt,
	)
	if err == sql.ErrNoRows {
		return nil, nil
	}
	return user, err
}

func (r *userRepo) Update(ctx context.Context, user *User) error {
	query := `UPDATE users SET username=$1, email=$2, role=$3, active=$4, updated_at=$5
		WHERE id=$6`
	user.UpdatedAt = time.Now()
	_, err := r.db.ExecContext(ctx, query,
		user.Username, user.Email, user.Role, user.Active, user.UpdatedAt, user.ID,
	)
	return err
}

func (r *userRepo) Delete(ctx context.Context, id int64) error {
	query := `DELETE FROM users WHERE id = $1`
	_, err := r.db.ExecContext(ctx, query, id)
	return err
}

func (r *userRepo) List(ctx context.Context, offset, limit int, filters map[string]interface{}) ([]*User, int64, error) {
	// 构建动态查询
	var conditions []string
	var args []interface{}
	argIdx := 1

	if role, ok := filters["role"].(string); ok && role != "" {
		conditions = append(conditions, fmt.Sprintf("role = $%d", argIdx))
		args = append(args, role)
		argIdx++
	}

	if search, ok := filters["search"].(string); ok && search != "" {
		conditions = append(conditions, fmt.Sprintf("(username ILIKE $%d OR email ILIKE $%d)", argIdx, argIdx))
		args = append(args, "%"+search+"%")
		argIdx++
	}

	where := ""
	if len(conditions) > 0 {
		where = "WHERE " + strings.Join(conditions, " AND ")
	}

	// 查询总数
	countQuery := fmt.Sprintf("SELECT COUNT(*) FROM users %s", where)
	var total int64
	if err := r.db.QueryRowContext(ctx, countQuery, args...).Scan(&total); err != nil {
		return nil, 0, err
	}

	// 查询列表
	listQuery := fmt.Sprintf(`SELECT id, username, email, password, role, active, created_at, updated_at
		FROM users %s ORDER BY id DESC LIMIT $%d OFFSET $%d`, where, argIdx, argIdx+1)
	args = append(args, limit, offset)

	rows, err := r.db.QueryContext(ctx, listQuery, args...)
	if err != nil {
		return nil, 0, err
	}
	defer rows.Close()

	var users []*User
	for rows.Next() {
		user := &User{}
		if err := rows.Scan(
			&user.ID, &user.Username, &user.Email, &user.Password,
			&user.Role, &user.Active, &user.CreatedAt, &user.UpdatedAt,
		); err != nil {
			return nil, 0, err
		}
		users = append(users, user)
	}

	return users, total, rows.Err()
}
```

```go
// userService UserService 的实现
type userService struct {
	repo   UserRepository
	cache  CacheService
	logger *slog.Logger
}

// NewUserService 创建用户服务
func NewUserService(repo UserRepository, cache CacheService, logger *slog.Logger) UserService {
	return &userService{
		repo:   repo,
		cache:  cache,
		logger: logger,
	}
}

func (s *userService) Create(ctx context.Context, username, email, password, role string) (*User, error) {
	// 业务规则 1：检查用户名是否已存在
	existing, err := s.repo.GetByUsername(ctx, username)
	if err != nil {
		return nil, fmt.Errorf("检查用户名失败: %w", err)
	}
	if existing != nil {
		return nil, errors.New("用户名已存在")
	}

	// 业务规则 2：检查邮箱是否已存在
	existing, err = s.repo.GetByEmail(ctx, email)
	if err != nil {
		return nil, fmt.Errorf("检查邮箱失败: %w", err)
	}
	if existing != nil {
		return nil, errors.New("邮箱已被注册")
	}

	// 业务规则 3：密码加密
	hashedPassword, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
	if err != nil {
		return nil, fmt.Errorf("密码加密失败: %w", err)
	}

	// 创建用户
	user := &User{
		Username: username,
		Email:    email,
		Password: string(hashedPassword),
		Role:     role,
		Active:   true,
	}

	if err := s.repo.Create(ctx, user); err != nil {
		return nil, fmt.Errorf("创建用户失败: %w", err)
	}

	s.logger.Info("用户创建成功",
		"user_id", user.ID,
		"username", user.Username,
	)

	return user, nil
}

func (s *userService) GetByID(ctx context.Context, id int64) (*User, error) {
	user, err := s.repo.GetByID(ctx, id)
	if err != nil {
		return nil, fmt.Errorf("查询用户失败: %w", err)
	}
	if user == nil {
		return nil, errors.New("用户不存在")
	}
	return user, nil
}

func (s *userService) Update(ctx context.Context, id int64, updates map[string]interface{}) (*User, error) {
	user, err := s.repo.GetByID(ctx, id)
	if err != nil {
		return nil, fmt.Errorf("查询用户失败: %w", err)
	}
	if user == nil {
		return nil, errors.New("用户不存在")
	}

	// 应用更新
	if username, ok := updates["username"].(string); ok {
		user.Username = username
	}
	if email, ok := updates["email"].(string); ok {
		user.Email = email
	}
	if role, ok := updates["role"].(string); ok {
		user.Role = role
	}
	if active, ok := updates["active"].(bool); ok {
		user.Active = active
	}

	if err := s.repo.Update(ctx, user); err != nil {
		return nil, fmt.Errorf("更新用户失败: %w", err)
	}

	// 清除缓存
	cacheKey := fmt.Sprintf("user:%d", id)
	if err := s.cache.Delete(ctx, cacheKey); err != nil {
		s.logger.Warn("清除缓存失败", "key", cacheKey, "error", err)
	}

	return user, nil
}

func (s *userService) Delete(ctx context.Context, id int64) error {
	user, err := s.repo.GetByID(ctx, id)
	if err != nil {
		return fmt.Errorf("查询用户失败: %w", err)
	}
	if user == nil {
		return errors.New("用户不存在")
	}

	if err := s.repo.Delete(ctx, id); err != nil {
		return fmt.Errorf("删除用户失败: %w", err)
	}

	return nil
}
```

## 七、常见坑点

```
坑点 1：过早过度设计目录结构
  新项目不需要复杂结构，从 flat 开始，代码多了再拆分。
  空目录 = 浪费认知负担。

坑点 2：cmd/main.go 包含业务逻辑
  main.go 只应做初始化、组装和启动。
  业务逻辑放 internal/。

坑点 3：internal/ 和 pkg/ 混淆
  internal/：项目私有，外部不可导入
  pkg/：公共库，任何人可导入
  99% 的代码应该在 internal/ 中。

坑点 4：循环依赖
  A 导入 B，B 又导入 A → 编译错误。
  解决：提取接口到独立的包，或合并包。

坑点 5：按类型分包（model/service/handler 分散在不同目录）
  导致高耦合低内聚。
  大项目应该按领域分包（user/order/notification）。

坑点 6：init() 函数滥用
  init() 隐式执行，难以测试和调试。
  优先使用显式的构造函数。

坑点 7：忘记 go mod tidy
  依赖未清理导致 go.sum 膨胀。
  CI 中加入 go mod tidy && git diff --exit-code go.sum。

坑点 8：Makefile 不支持跨平台
  某些 shell 命令在 macOS 和 Linux 上行为不同。
  注意 sed/date 等命令的差异。
```

## 八、速查表

### 目录选择速查

| 代码类型 | 放在哪里 | 原因 |
|---|---|---|
| 主程序入口 | `cmd/appname/main.go` | 标准约定 |
| 业务逻辑 | `internal/service/` | 不应暴露 |
| HTTP 处理 | `internal/handler/` | 不应暴露 |
| 数据访问 | `internal/repository/` | 不应暴露 |
| 数据模型 | `internal/model/` | 不应暴露 |
| 公共工具库 | `pkg/` | 可被外部使用 |
| API 定义 | `api/` | 方便查找 |
| 配置模板 | `configs/` | 不含敏感信息 |
| 部署配置 | `deployments/` | K8s/Docker |
| 数据库迁移 | `migrations/` | 版本化管理 |
| 集成测试 | `test/` | 与单元测试分离 |
| 自动化脚本 | `scripts/` | 开发运维 |

### 常用命令

```bash
# 初始化项目
go mod init github.com/org/myapp

# 创建标准目录
mkdir -p cmd/server internal/{config,handler,service,repository,model} configs deployments scripts

# 格式化
gofmt -s -w .
goimports -w .

# 检查依赖
go mod tidy
go mod verify
go mod graph

# 编译
CGO_ENABLED=0 go build -ldflags="-s -w" -o build/myapp cmd/server/main.go
```
