# Go 内存优化完全指南

## 导读

内存管理是 Go 性能优化中最重要的环节之一。Go 的垃圾回收器虽然方便，但不当的内存使用模式会导致 GC 压力增大、延迟抖动甚至 OOM。作为 SRE，理解 Go 的内存模型、GC 机制和优化技巧，是保障线上服务稳定运行的基础能力。

**本篇核心知识点：**
- Go 内存管理模型（mcache/mcentral/mheap）
- GC 垃圾回收机制（三色标记法、STW）
- GOGC 调优与 GOMEMLIMIT（Go 1.19+）
- 内存逃逸分析实践
- 减少堆分配的实用技巧
- sync.Pool 对象池使用
- 内存泄漏排查方法
- SRE 实战：优化高并发服务内存使用

## 一、Go 内存管理模型

### 1.1 整体架构

```go
// memory_model.go
// 演示 Go 内存管理的基本概念
package main

import (
	"fmt"
	"runtime"
	"unsafe"
)

/*
Go 内存管理三层架构:

┌─────────────────────────────────────────────┐
│                  应用层                       │
│    go 代码中的 make/new/字面量 等分配操作       │
├─────────────────────────────────────────────┤
│              mcache (每个 P 一个)              │
│   ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│   │  P0  │ │  P1  │ │  P2  │ │  P3  │       │
│   │mcache│ │mcache│ │mcache│ │mcache│       │
│   └──────┘ └──────┘ └──────┘ └──────┘       │
│   无锁分配小对象（<= 32KB）                    │
├─────────────────────────────────────────────┤
│               mcentral (全局)                 │
│   按大小分类的 span 缓存（67 种 size class）    │
│   mcache 耗尽时从这里补充（需要加锁）            │
├─────────────────────────────────────────────┤
│                mheap (全局)                    │
│   管理所有的 arena（64MB 块）                  │
│   mcentral 耗尽时从这里分配新的 span            │
│   大对象（> 32KB）直接在这里分配                │
├─────────────────────────────────────────────┤
│             操作系统 (mmap/munmap)             │
│   通过系统调用申请/归还物理内存                  │
└─────────────────────────────────────────────┘

关键概念:
- span: 连续的内存页，是内存管理的基本单位
- size class: 把对象按大小分成 67 个等级，减少碎片
- mcache: 每个 P（处理器）私有的缓存，分配小对象无需加锁
- mcentral: 全局的 span 缓存，按 size class 分类
- mheap: 最顶层的内存管理器，直接与 OS 交互
*/

// 不同大小的对象走不同的分配路径
func demonstrateAllocation() {
	// 微对象（<= 16B）: 使用 mcache 的 tiny allocator
	// 多个微对象可能共享同一个 16B 的内存块
	var a int8 = 42   // 1 字节
	var b int16 = 100  // 2 字节
	fmt.Printf("微对象 a: size=%d, addr=%p\n", unsafe.Sizeof(a), &a)
	fmt.Printf("微对象 b: size=%d, addr=%p\n", unsafe.Sizeof(b), &b)

	// 小对象（16B < size <= 32KB）: 从 mcache 的 mspan 分配
	// 根据 size class 找到合适大小的 span
	smallSlice := make([]byte, 1024) // 1KB
	fmt.Printf("小对象: size=%d, addr=%p\n", len(smallSlice), &smallSlice[0])

	// 大对象（> 32KB）: 直接从 mheap 分配
	// 大对象有专门的 span，不走 size class
	bigSlice := make([]byte, 64*1024) // 64KB
	fmt.Printf("大对象: size=%d, addr=%p\n", len(bigSlice), &bigSlice[0])
}

func main() {
	fmt.Println("=== Go 内存管理模型演示 ===\n")

	demonstrateAllocation()

	// 查看 GOMAXPROCS（决定了 mcache 的数量）
	fmt.Printf("\nGOMAXPROCS: %d（即有 %d 个 mcache）\n",
		runtime.GOMAXPROCS(0), runtime.GOMAXPROCS(0))

	// 查看内存统计
	var stats runtime.MemStats
	runtime.ReadMemStats(&stats)

	fmt.Println("\n=== 内存统计 ===")
	fmt.Printf("HeapAlloc:    %6d KB  (堆上已分配且使用中的内存)\n", stats.HeapAlloc/1024)
	fmt.Printf("HeapSys:      %6d KB  (从 OS 申请的堆内存总量)\n", stats.HeapSys/1024)
	fmt.Printf("HeapIdle:     %6d KB  (堆上空闲的 span)\n", stats.HeapIdle/1024)
	fmt.Printf("HeapInuse:    %6d KB  (堆上使用中的 span)\n", stats.HeapInuse/1024)
	fmt.Printf("HeapReleased: %6d KB  (归还给 OS 的内存)\n", stats.HeapReleased/1024)
	fmt.Printf("HeapObjects:  %6d     (堆上的对象数量)\n", stats.HeapObjects)
	fmt.Printf("StackInuse:   %6d KB  (栈使用的内存)\n", stats.StackInuse/1024)
	fmt.Printf("MSpanInuse:   %6d KB  (mspan 使用的内存)\n", stats.MSpanInuse/1024)
	fmt.Printf("MCacheInuse:  %6d KB  (mcache 使用的内存)\n", stats.MCacheInuse/1024)

	// Size Class 信息
	fmt.Println("\n=== Size Class 分布（部分） ===")
	fmt.Println("Class  Size  Objects  Bytes")
	for i, size := range stats.BySize {
		if size.Mallocs > 0 && i < 20 {
			fmt.Printf("  %2d  %5d  %7d  %7d KB\n",
				i, size.Size, size.Mallocs, size.Size*uint32(size.Mallocs)/1024)
		}
	}
}
```

### 1.2 栈 vs 堆

```go
// stack_vs_heap.go
// 演示栈分配和堆分配的区别
package main

import (
	"fmt"
	"runtime"
)

// stackAllocation 演示栈分配
// 编译器能确定变量不会逃逸到堆上时，会在栈上分配
func stackAllocation() int {
	// x 只在函数内部使用，编译器会在栈上分配
	x := 42
	y := x * 2
	return y
}

// heapAllocation 演示堆分配
// 返回局部变量的指针，变量必须在堆上分配
func heapAllocation() *int {
	// x 的指针被返回，编译器会把 x 分配到堆上
	x := 42
	return &x // x 逃逸到堆
}

// interfaceBoxing 接口装箱导致堆分配
func interfaceBoxing() interface{} {
	x := 42
	// 将值类型赋给 interface{} 会导致堆分配
	return x // x 被装箱（boxing），逃逸到堆
}

// sliceGrowth 切片增长导致堆分配
func sliceGrowth() []int {
	var s []int
	for i := 0; i < 100; i++ {
		s = append(s, i) // 当容量不足时，会在堆上分配新的底层数组
	}
	return s
}

// closureCapture 闭包捕获变量导致堆分配
func closureCapture() func() int {
	count := 0
	// count 被闭包捕获，必须在堆上分配
	return func() int {
		count++
		return count
	}
}

func main() {
	fmt.Println("=== 栈 vs 堆分配演示 ===\n")

	// 栈分配：快速，无 GC 压力
	result1 := stackAllocation()
	fmt.Printf("栈分配结果: %d\n", result1)

	// 堆分配：需要 GC 回收
	result2 := heapAllocation()
	fmt.Printf("堆分配结果: %d (地址: %p)\n", *result2, result2)

	// 接口装箱
	result3 := interfaceBoxing()
	fmt.Printf("接口装箱: %v\n", result3)

	// 切片增长
	result4 := sliceGrowth()
	fmt.Printf("切片长度: %d, 容量: %d\n", len(result4), cap(result4))

	// 闭包
	counter := closureCapture()
	fmt.Printf("闭包调用: %d, %d, %d\n", counter(), counter(), counter())

	// 打印内存统计
	var stats runtime.MemStats
	runtime.ReadMemStats(&stats)
	fmt.Printf("\n堆分配次数: %d\n", stats.Mallocs)
	fmt.Printf("堆释放次数: %d\n", stats.Frees)
	fmt.Printf("存活对象: %d\n", stats.Mallocs-stats.Frees)

	fmt.Println("\n使用逃逸分析查看分配位置:")
	fmt.Println("  go build -gcflags='-m -l' stack_vs_heap.go")
	fmt.Println("  -m: 打印逃逸分析结果")
	fmt.Println("  -l: 禁用内联（让分析结果更清晰）")
}
```

## 二、GC 垃圾回收机制

### 2.1 三色标记法

```go
// gc_tricolor.go
// 演示 Go GC 的三色标记法原理
package main

import (
	"fmt"
	"runtime"
	"runtime/debug"
	"time"
)

/*
Go GC 采用三色标记-清除算法（Tri-color Mark and Sweep）:

1. 初始状态: 所有对象为白色
2. 从根对象（栈、全局变量）开始，标记为灰色
3. 遍历灰色对象:
   - 将灰色对象标记为黑色
   - 将灰色对象引用的白色对象标记为灰色
4. 重复步骤 3，直到没有灰色对象
5. 回收所有白色对象

颜色含义:
- 白色: 可能是垃圾（待检查）
- 灰色: 存活对象，但其引用的对象还未检查
- 黑色: 确定存活，且引用的对象都已检查

写屏障（Write Barrier）:
- GC 与用户代码并发执行
- 如果用户代码在 GC 标记期间修改了引用关系，可能导致误回收
- 写屏障确保并发安全:
  - 混合写屏障（Go 1.8+）: 结合插入屏障和删除屏障
  - 在 GC 标记阶段，每次指针写操作都会触发写屏障

GC 触发条件:
1. 堆内存增长到 GOGC 指定的比例（默认 100%，即翻倍时触发）
2. 手动调用 runtime.GC()
3. 距上次 GC 超过 2 分钟（forcegc）

GC 阶段:
1. Mark Setup (STW): 开启写屏障，极短暂暂停 (< 1ms)
2. Marking (并发):   标记存活对象，与用户代码并发运行
3. Mark Termination (STW): 关闭写屏障，极短暂暂停
4. Sweeping (并发):  回收白色对象，与用户代码并发运行
*/

// Node 模拟一个对象引用图
type Node struct {
	ID       int
	Children []*Node
	Data     []byte
}

func buildObjectGraph() *Node {
	// 创建根节点
	root := &Node{
		ID:   0,
		Data: make([]byte, 1024),
	}

	// 创建子节点形成引用图
	for i := 1; i <= 10; i++ {
		child := &Node{
			ID:   i,
			Data: make([]byte, 1024),
		}
		root.Children = append(root.Children, child)

		// 子节点还有子节点
		for j := 1; j <= 5; j++ {
			grandchild := &Node{
				ID:   i*100 + j,
				Data: make([]byte, 512),
			}
			child.Children = append(child.Children, grandchild)
		}
	}

	return root
}

func main() {
	fmt.Println("=== Go GC 三色标记法演示 ===\n")

	// 开启 GC trace 日志
	// 等价于环境变量 GODEBUG=gctrace=1
	// 这里我们用代码方式演示
	debug.SetGCPercent(100) // GOGC=100（默认值）

	// 创建对象图
	root := buildObjectGraph()
	fmt.Printf("创建了对象图，根节点有 %d 个子节点\n", len(root.Children))

	// 手动触发 GC 并观察
	fmt.Println("\n--- 第一次 GC ---")
	start := time.Now()
	runtime.GC()
	fmt.Printf("GC 耗时: %v\n", time.Since(start))

	var stats runtime.MemStats
	runtime.ReadMemStats(&stats)
	fmt.Printf("GC 暂停次数: %d\n", stats.NumGC)
	fmt.Printf("最近一次 GC 暂停: %v\n", time.Duration(stats.PauseNs[(stats.NumGC+255)%256]))
	fmt.Printf("堆存活对象: %.2f KB\n", float64(stats.HeapAlloc)/1024)

	// 制造一些垃圾
	fmt.Println("\n--- 制造垃圾 ---")
	for i := 0; i < 10000; i++ {
		// 这些切片没有被引用，会成为垃圾
		_ = make([]byte, 1024)
	}

	fmt.Println("\n--- 第二次 GC ---")
	start = time.Now()
	runtime.GC()
	fmt.Printf("GC 耗时: %v\n", time.Since(start))

	runtime.ReadMemStats(&stats)
	fmt.Printf("GC 暂停次数: %d\n", stats.NumGC)
	fmt.Printf("最近一次 GC 暂停: %v\n", time.Duration(stats.PauseNs[(stats.NumGC+255)%256]))
	fmt.Printf("堆存活对象: %.2f KB\n", float64(stats.HeapAlloc)/1024)

	// 切断引用，让对象图成为垃圾
	fmt.Println("\n--- 切断根引用 ---")
	root = nil
	_ = root

	runtime.GC()
	runtime.ReadMemStats(&stats)
	fmt.Printf("GC 后堆存活对象: %.2f KB\n", float64(stats.HeapAlloc)/1024)

	// GC 统计
	fmt.Println("\n=== GC 统计摘要 ===")
	fmt.Printf("总 GC 暂停时间: %v\n", time.Duration(stats.PauseTotalNs))
	fmt.Printf("GC 总次数: %d\n", stats.NumGC)
	fmt.Printf("GC CPU 占比: %.4f%%\n", stats.GCCPUFraction*100)

	fmt.Println("\n提示: 使用 GODEBUG=gctrace=1 运行程序可以看到详细的 GC 日志")
	fmt.Println("  GODEBUG=gctrace=1 go run gc_tricolor.go")
	fmt.Println("\n日志格式说明:")
	fmt.Println("  gc # @#s #%: #+...+# ms clock, #+...+# ms cpu, #->#-># MB, # MB goal, # P")
	fmt.Println("  gc 次数 @运行时间 CPU占比%: 各阶段耗时 clock, CPU时间, 堆大小变化, 目标大小, P数量")
}
```

### 2.2 GOGC 调优与 GOMEMLIMIT

```go
// gogc_tuning.go
// 演示 GOGC 和 GOMEMLIMIT 调优
package main

import (
	"fmt"
	"math/rand"
	"runtime"
	"runtime/debug"
	"time"
)

/*
=== GOGC 参数说明 ===

GOGC 控制 GC 的触发频率:
- 默认值: 100
- 含义: 当新分配的内存达到上次 GC 后存活内存的 GOGC% 时，触发 GC
- GOGC=100: 堆内存翻倍时触发 GC
- GOGC=200: 堆内存增长到 3 倍时触发 GC（GC 频率降低，内存使用增加）
- GOGC=50:  堆内存增长到 1.5 倍时触发 GC（GC 频率增加，内存使用减少）
- GOGC=off: 关闭 GC（极少使用，可能 OOM）

计算公式:
  触发GC的堆大小 = 存活对象大小 × (1 + GOGC/100)

=== GOMEMLIMIT (Go 1.19+) ===

GOMEMLIMIT 设置 Go 运行时的内存使用上限:
- 软限制，不是硬限制
- 当内存接近限制时，GC 会更积极地运行
- 替代了之前使用 ballast（大块空闲内存）的 hack 方法
- 单位: 字节（支持后缀 B/KiB/MiB/GiB/TiB）
*/

// 模拟不同 GOGC 设置下的内存和 GC 行为
func benchmarkGOGC(gogcValue int, label string) {
	// 保存旧值
	old := debug.SetGCPercent(gogcValue)
	defer debug.SetGCPercent(old)

	fmt.Printf("\n=== %s (GOGC=%d) ===\n", label, gogcValue)

	var stats runtime.MemStats
	runtime.ReadMemStats(&stats)
	startGC := stats.NumGC

	start := time.Now()

	// 分配大量临时对象
	for i := 0; i < 100000; i++ {
		data := make([]byte, rand.Intn(1024)+64)
		_ = data
	}

	// 同时维持一些长期存活的对象
	var alive [][]byte
	for i := 0; i < 100; i++ {
		alive = append(alive, make([]byte, 10240))
	}

	runtime.ReadMemStats(&stats)
	duration := time.Since(start)

	fmt.Printf("  耗时: %v\n", duration)
	fmt.Printf("  GC 次数: %d\n", stats.NumGC-startGC)
	fmt.Printf("  当前堆大小: %.2f MB\n", float64(stats.HeapAlloc)/1024/1024)
	fmt.Printf("  堆系统申请: %.2f MB\n", float64(stats.HeapSys)/1024/1024)
	fmt.Printf("  GC 总暂停: %v\n", time.Duration(stats.PauseTotalNs))

	_ = alive // 保持引用
}

func main() {
	fmt.Println("=== GOGC 调优演示 ===")

	// 测试不同 GOGC 值
	benchmarkGOGC(50, "保守模式")   // 更频繁的 GC，更少的内存使用
	benchmarkGOGC(100, "默认模式")  // 默认行为
	benchmarkGOGC(200, "宽松模式")  // 更少的 GC，更多的内存使用

	fmt.Println("\n\n=== GOMEMLIMIT 演示 (Go 1.19+) ===")

	// 设置内存限制为 50MB
	// 当内存接近这个限制时，GC 会更积极地运行
	memLimit := int64(50 * 1024 * 1024) // 50 MiB
	previousLimit := debug.SetMemoryLimit(memLimit)
	fmt.Printf("设置 GOMEMLIMIT: %d MB (之前: %d)\n", memLimit/1024/1024, previousLimit)

	// GOGC=off + GOMEMLIMIT 是推荐的容器化部署配置
	// 这样 GC 只在内存接近限制时才触发
	fmt.Println("\n--- GOGC=off + GOMEMLIMIT=50MiB ---")
	debug.SetGCPercent(-1) // 关闭基于 GOGC 的 GC

	var stats runtime.MemStats
	runtime.ReadMemStats(&stats)
	startGC := stats.NumGC

	// 持续分配内存，直到接近限制
	var chunks [][]byte
	for i := 0; i < 100; i++ {
		chunk := make([]byte, 256*1024) // 256KB per chunk
		chunks = append(chunks, chunk)

		runtime.ReadMemStats(&stats)
		if i%10 == 0 {
			fmt.Printf("  分配 %d 块: 堆使用 %.2f MB, GC 次数 %d\n",
				i+1,
				float64(stats.HeapAlloc)/1024/1024,
				stats.NumGC-startGC,
			)
		}
	}

	runtime.ReadMemStats(&stats)
	fmt.Printf("\n最终: 堆使用 %.2f MB, GC 次数 %d\n",
		float64(stats.HeapAlloc)/1024/1024,
		stats.NumGC-startGC,
	)

	// 恢复默认设置
	debug.SetGCPercent(100)
	debug.SetMemoryLimit(-1) // 恢复默认（无限制）

	fmt.Println("\n=== 调优建议 ===")
	fmt.Println("1. 容器环境: GOMEMLIMIT=容器内存*0.8, GOGC=off")
	fmt.Println("   export GOMEMLIMIT=800MiB  # 1GB 容器")
	fmt.Println("   export GOGC=off")
	fmt.Println("2. 延迟敏感服务: GOGC=50（更频繁但更短的 GC）")
	fmt.Println("3. 吞吐优先服务: GOGC=200（更少的 GC，但每次暂停可能更长）")
	fmt.Println("4. 一般服务: 保持默认 GOGC=100")
}
```

## 三、内存逃逸分析

### 3.1 逃逸分析基础

```go
// escape_analysis.go
// 使用 go build -gcflags="-m -l" escape_analysis.go 查看逃逸分析结果
package main

import (
	"fmt"
	"io"
)

// === 不会逃逸的场景 ===

// noEscape1 局部变量不返回指针，在栈上分配
func noEscape1() int {
	x := 42 // 不逃逸
	return x
}

// noEscape2 切片在已知大小且不返回时，在栈上分配
func noEscape2() int {
	s := make([]int, 10) // 不逃逸（小且不返回）
	s[0] = 1
	return s[0]
}

// noEscape3 结构体值传递
func noEscape3() int {
	type Point struct{ X, Y int }
	p := Point{X: 1, Y: 2} // 不逃逸
	return p.X + p.Y
}

// === 会逃逸的场景 ===

// escape1 返回局部变量的指针
func escape1() *int {
	x := 42 // x 逃逸到堆
	return &x
}

// escape2 赋值给接口类型
func escape2() {
	x := 42
	var i interface{} = x // x 逃逸（接口装箱）
	_ = i
}

// escape3 发送到 channel
func escape3(ch chan *int) {
	x := 42  // x 逃逸
	ch <- &x // 通过 channel 传递指针
}

// escape4 被闭包引用
func escape4() func() int {
	x := 0 // x 逃逸（被闭包捕获）
	return func() int {
		x++
		return x
	}
}

// escape5 传递给 fmt.Println（参数是 ...interface{}）
func escape5() {
	x := 42
	fmt.Println(x) // x 逃逸（被装箱为 interface{}）
}

// escape6 切片 append 可能导致逃逸
func escape6() []int {
	s := make([]int, 0) // s 逃逸（返回了引用）
	s = append(s, 1, 2, 3)
	return s
}

// escape7 动态大小的切片
func escape7(n int) {
	s := make([]int, n) // s 逃逸（编译期无法确定大小）
	s[0] = 1
	_ = s
}

// escape8 map 的值
func escape8() {
	m := make(map[string]*int)
	x := 42
	m["key"] = &x // x 逃逸（map 的值在堆上）
	_ = m
}
```

```go
// 低效版本：每次拼接都产生新的字符串
func buildQuerySlow(table string, conditions map[string]string) string {
	query := "SELECT * FROM " + table + " WHERE "
	parts := make([]string, 0)
	for k, v := range conditions {
		parts = append(parts, fmt.Sprintf("%s = '%s'", k, v))
	}
	query += strings.Join(parts, " AND ")
	return query
}

// 高效版本：使用 strings.Builder，减少中间字符串分配
func buildQueryFast(table string, conditions map[string]string) string {
	var b strings.Builder
	b.Grow(128) // 预估大小，减少扩容
	b.WriteString("SELECT * FROM ")
	b.WriteString(table)
	b.WriteString(" WHERE ")

	first := true
	for k, v := range conditions {
		if !first {
			b.WriteString(" AND ")
		}
		b.WriteString(k)
		b.WriteString(" = '")

	}
```

```go
// sync_pool.go
// 演示 sync.Pool 的使用和注意事项
package main

import (
	"bytes"
	"encoding/json"
	"fmt"
	"runtime"
	"sync"
	"time"
)

// === 基本用法 ===

// Buffer Pool: 复用 bytes.Buffer
var bufferPool = sync.Pool{
	New: func() interface{} {
		// 当池中没有可用对象时，调用 New 创建
		return new(bytes.Buffer)
	},
}

// getBuffer 从池中获取 buffer
func getBuffer() *bytes.Buffer {
	return bufferPool.Get().(*bytes.Buffer)
}

// putBuffer 归还 buffer 到池中
func putBuffer(buf *bytes.Buffer) {
	buf.Reset() // 重要：归还前清空内容
	bufferPool.Put(buf)
}

// === 实际应用：JSON 序列化复用 buffer ===

type LogEntry struct {
	Timestamp string `json:"timestamp"`
	Level     string `json:"level"`
	Message   string `json:"message"`
	Fields    map[string]interface{} `json:"fields,omitempty"`
}

// 不使用对象池的版本
func serializeLogSlow(entry *LogEntry) ([]byte, error) {
	// 每次都分配新的 buffer
	return json.Marshal(entry)
}

// 使用对象池的版本
var encoderPool = sync.Pool{
	New: func() interface{} {
		return new(bytes.Buffer)
	},
}

func serializeLogFast(entry *LogEntry) ([]byte, error) {
	buf := encoderPool.Get().(*bytes.Buffer)
	defer func() {
		buf.Reset()
		encoderPool.Put(buf)
	}()

	encoder := json.NewEncoder(buf)
	if err := encoder.Encode(entry); err != nil {
		return nil, err
	}

	// 注意：必须复制数据，因为 buffer 会被归还复用
	result := make([]byte, buf.Len())
	copy(result, buf.Bytes())
	return result, nil
}

// === 对象池的正确使用模式 ===

// SlicePool 切片对象池（带容量管理）
type SlicePool struct {
	pool sync.Pool
	maxCap int
}

func NewSlicePool(initCap, maxCap int) *SlicePool {
	return &SlicePool{
		pool: sync.Pool{
			New: func() interface{} {
				s := make([]byte, 0, initCap)
				return &s
			},
		},
		maxCap: maxCap,
	}
}

func (sp *SlicePool) Get() *[]byte {
	return sp.pool.Get().(*[]byte)
}

func (sp *SlicePool) Put(s *[]byte) {
	// 重要：如果切片容量过大，不要归还（避免内存浪费）
	if cap(*s) > sp.maxCap {
		return // 丢弃过大的切片，让 GC 回收
	}
	*s = (*s)[:0] // 重置长度但保留容量
	sp.pool.Put(s)
}

func main() {
	fmt.Println("=== sync.Pool 对象池演示 ===\n")

	// 基本使用演示
	fmt.Println("--- 基本使用 ---")
	buf := getBuffer()
	buf.WriteString("hello, sync.Pool!")
	fmt.Printf("Buffer 内容: %s\n", buf.String())
	putBuffer(buf) // 归还到池中

	buf2 := getBuffer() // 可能获取到刚归还的 buffer
	fmt.Printf("复用的 Buffer 长度: %d (应该是 0)\n", buf2.Len())
	putBuffer(buf2)

	// 性能对比
	fmt.Println("\n--- 性能对比: JSON 序列化 ---")
	entry := &LogEntry{
		Timestamp: time.Now().Format(time.RFC3339),
		Level:     "INFO",
		Message:   "用户登录成功",
		Fields: map[string]interface{}{
			"user_id":  12345,
			"ip":       "192.168.1.100",
			"duration": "23ms",
		},
	}

	iterations := 100000

	// 不使用对象池
	runtime.GC()
	var stats1 runtime.MemStats
	runtime.ReadMemStats(&stats1)
	start1 := time.Now()
	for i := 0; i < iterations; i++ {
		_, _ = serializeLogSlow(entry)
	}
	duration1 := time.Since(start1)
	runtime.ReadMemStats(&stats1)

	// 使用对象池
	runtime.GC()
	var stats2Before runtime.MemStats
	runtime.ReadMemStats(&stats2Before)
	start2 := time.Now()
	for i := 0; i < iterations; i++ {
		_, _ = serializeLogFast(entry)
	}
	duration2 := time.Since(start2)
	var stats2After runtime.MemStats
	runtime.ReadMemStats(&stats2After)

	fmt.Printf("不使用 Pool: %v\n", duration1)
	fmt.Printf("使用 Pool:   %v\n", duration2)

	// 切片池演示
	fmt.Println("\n--- 切片池 ---")
	sp := NewSlicePool(1024, 64*1024)

	s := sp.Get()
	*s = append(*s, []byte("hello slice pool")...)
	fmt.Printf("切片内容: %s, cap: %d\n", string(*s), cap(*s))
	sp.Put(s)

	s2 := sp.Get()
	fmt.Printf("复用的切片长度: %d, cap: %d\n", len(*s2), cap(*s2))
	sp.Put(s2)

	// sync.Pool 注意事项
	fmt.Println("\n=== sync.Pool 注意事项 ===")
	fmt.Println("1. Pool 中的对象可能随时被 GC 回收（每次 GC 都会清空）")
	fmt.Println("2. 归还前必须重置对象状态（Reset/清零）")
	fmt.Println("3. 不要存储带有状态依赖的对象")
	fmt.Println("4. 对象太大时不要归还（设置最大容量限制）")
	fmt.Println("5. Get 返回的对象可能是新创建的，不要假设是复用的")
	fmt.Println("6. Pool 是并发安全的，内部使用了 P-local 缓存减少竞争")
}
```

```go
// 错误示例：goroutine 泄漏
func leakyGoroutines() {
	for i := 0; i < 100; i++ {
		ch := make(chan int)
		go func() {
			// 没人往 ch 里发数据，这个 goroutine 永远阻塞
			val := <-ch
			fmt.Println(val)
		}()
		// ch 没有被关闭，goroutine 泄漏
	}
}

// 正确示例：使用 context 控制 goroutine 生命周期
func safeGoroutines(ctx context.Context) {
	for i := 0; i < 100; i++ {
		ch := make(chan int)
		go func() {
			select {
			case val := <-ch:
				fmt.Println(val)
			case <-ctx.Done():
				return // context 取消时退出
			}
		}()
	}
}

// === 泄漏模式 2：time.After 泄漏 ===

// 错误示例：在循环中使用 time.After
func leakyTimer() {
	ch := make(chan int, 100)

	// 模拟生产者
	go func() {
		for i := 0; i < 1000; i++ {
			ch <- i
			time.Sleep(time.Millisecond)
		}
		close(ch)
	}()

	// 错误：每次循环都创建新的 timer，旧的不会被 GC 直到超时
	for {
		select {
		case val, ok := <-ch:
			if !ok {
				return
			}
			_ = val
		case <-time.After(5 * time.Second): // 每次循环创建新的 timer！
			fmt.Println("超时")
			return
		}
	}
}

// 正确示例：复用 timer
func safeTimer() {
	ch := make(chan int, 100)

	go func() {
		for i := 0; i < 1000; i++ {
			ch <- i
			time.Sleep(time.Millisecond)
		}
		close(ch)
	}()

	timer := time.NewTimer(5 * time.Second)
	defer timer.Stop()

	for {
		// 重置 timer
		if !timer.Stop() {
			select {
			case <-timer.C:
			default:
			}
		}
		timer.Reset(5 * time.Second)

		select {
		case val, ok := <-ch:
			if !ok {
				return
			}
			_ = val
		case <-timer.C:
			fmt.Println("超时")
			return
		}
	}
}

// 错误示例：小切片引用大数组
func leakySlice() []byte {

		bigData[i] = byte(i % 256)

	return bigData[:10]
}

	for i := range bigData {
		bigData[i] = byte(i % 256)

type CacheManager struct {
	mu    sync.RWMutex
	cache map[string][]byte
}

// 错误示例：只添加不删除
func (cm *CacheManager) SetBad(key string, value []byte) {

	defer cm.mu.Unlock()

	// 没有过期机制，map 会无限增长

// 正确示例：带 TTL 的缓存
type CacheEntry struct {
	Value     []byte
	ExpiresAt time.Time
}

type SafeCache struct {
	mu    sync.RWMutex
	cache map[string]*CacheEntry
}

func NewSafeCache() *SafeCache {
	sc := &SafeCache{
		cache: make(map[string]*CacheEntry),
	}
	// 启动清理 goroutine
	go sc.cleanup()
	return sc
}

func (sc *SafeCache) Set(key string, value []byte, ttl time.Duration) {
	sc.mu.Lock()
	defer sc.mu.Unlock()
	sc.cache[key] = &CacheEntry{
		Value:     value,
		ExpiresAt: time.Now().Add(ttl),
	}
}

func (sc *SafeCache) cleanup() {
	ticker := time.NewTicker(time.Minute)
	defer ticker.Stop()
	for range ticker.C {
		sc.mu.Lock()
		for key, entry := range sc.cache {
			if time.Now().After(entry.ExpiresAt) {
				delete(sc.cache, key)
			}
		}
		sc.mu.Unlock()
	}
}
```

## 六、SRE 实战：优化高并发服务内存使用

### 6.1 完整优化案例

```go
// sre_memory_optimization.go
// SRE 实战：一个有内存问题的服务及其优化版本
package main

import (
	"bytes"
	"context"
	"encoding/json"
	"fmt"
	"log/slog"
	"math/rand"
	"net/http"
	_ "net/http/pprof"
	"os"
	"os/signal"
	"runtime"
	"runtime/debug"
	"sync"
	"syscall"
	"time"
)

// === 优化后的请求处理 ===

// RequestProcessor 请求处理器（优化版）
type RequestProcessor struct {
	logger     *slog.Logger
	bufferPool sync.Pool
	resultPool sync.Pool
}

func NewRequestProcessor(logger *slog.Logger) *RequestProcessor {
	return &RequestProcessor{
		logger: logger,
		bufferPool: sync.Pool{
			New: func() interface{} {
				return bytes.NewBuffer(make([]byte, 0, 4096))
			},
		},
		resultPool: sync.Pool{
			New: func() interface{} {
				return &ProcessResult{
					Items: make([]ResultItem, 0, 64),
				}
			},
		},
	}
}

type ResultItem struct {
	ID    int     `json:"id"`
	Value float64 `json:"value"`
	Tag   string  `json:"tag"`
}

type ProcessResult struct {
	Items     []ResultItem `json:"items"`
	Total     int          `json:"total"`
	Timestamp string       `json:"timestamp"`
}

// Process 处理请求（优化版：复用对象）
func (rp *RequestProcessor) Process(n int) []byte {
	// 从池中获取 result 对象
	result := rp.resultPool.Get().(*ProcessResult)
	defer func() {
		// 清理并归还
		result.Items = result.Items[:0]
		result.Total = 0
		result.Timestamp = ""
		rp.resultPool.Put(result)
	}()

	// 填充数据（复用切片，避免 append 扩容）
	for i := 0; i < n && i < cap(result.Items); i++ {
		result.Items = append(result.Items, ResultItem{
			ID:    i,
			Value: rand.Float64() * 100,
			Tag:   "processed",
		})
	}
	result.Total = len(result.Items)
	result.Timestamp = time.Now().Format(time.RFC3339)

	// 从池中获取 buffer
	buf := rp.bufferPool.Get().(*bytes.Buffer)
	defer func() {
		buf.Reset()
		rp.bufferPool.Put(buf)
	}()

	// 序列化到复用的 buffer
	encoder := json.NewEncoder(buf)
	if err := encoder.Encode(result); err != nil {
		rp.logger.Error("序列化失败", "error", err)
		return nil
	}

	// 必须复制数据（因为 buffer 要归还池中）
	data := make([]byte, buf.Len())
	copy(data, buf.Bytes())
	return data
}

// === 带内存监控的服务 ===

type MemoryMonitor struct {
	logger      *slog.Logger
	interval    time.Duration
	memLimit    uint64
	alertSent   bool
}

func NewMemoryMonitor(logger *slog.Logger, memLimit uint64) *MemoryMonitor {
	return &MemoryMonitor{
		logger:   logger,
		interval: 10 * time.Second,
		memLimit: memLimit,
	}
}

func (mm *MemoryMonitor) Start(ctx context.Context) {
	ticker := time.NewTicker(mm.interval)
	defer ticker.Stop()

	for {
		select {
		case <-ctx.Done():
			return
		case <-ticker.C:
			var stats runtime.MemStats
			runtime.ReadMemStats(&stats)

			mm.logger.Info("内存状态",
				"heap_alloc_mb", fmt.Sprintf("%.2f", float64(stats.HeapAlloc)/1024/1024),
				"heap_sys_mb", fmt.Sprintf("%.2f", float64(stats.HeapSys)/1024/1024),
				"heap_objects", stats.HeapObjects,
				"goroutines", runtime.NumGoroutine(),
				"gc_count", stats.NumGC,
				"gc_pause_last", time.Duration(stats.PauseNs[(stats.NumGC+255)%256]).String(),
			)

			// 内存告警
			if stats.HeapAlloc > mm.memLimit && !mm.alertSent {
				mm.logger.Warn("内存使用超过阈值",
					"current_mb", float64(stats.HeapAlloc)/1024/1024,
					"limit_mb", float64(mm.memLimit)/1024/1024,
				)
				mm.alertSent = true
			} else if stats.HeapAlloc < mm.memLimit {
				mm.alertSent = false
			}
		}
	}
}

func main() {
	logger := slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
		Level: slog.LevelInfo,
	}))

	// 容器环境推荐设置
	// 假设容器限制 512MB，设置 GOMEMLIMIT 为 80%
	debug.SetMemoryLimit(400 * 1024 * 1024) // 400 MiB
	logger.Info("设置 GOMEMLIMIT", "limit_mb", 400)

	// 创建处理器
	processor := NewRequestProcessor(logger)

	// 启动内存监控
	ctx, cancel := context.WithCancel(context.Background())
	defer cancel()

	monitor := NewMemoryMonitor(logger, 200*1024*1024) // 200MB 告警
	go monitor.Start(ctx)

	// HTTP 路由
	mux := http.NewServeMux()

	mux.HandleFunc("/api/process", func(w http.ResponseWriter, r *http.Request) {
		data := processor.Process(50)
		if data == nil {
			http.Error(w, "处理失败", http.StatusInternalServerError)
			return
		}
		w.Header().Set("Content-Type", "application/json")
		w.Write(data)
	})

	mux.HandleFunc("/api/stats", func(w http.ResponseWriter, r *http.Request) {
		var stats runtime.MemStats
		runtime.ReadMemStats(&stats)

		response := map[string]interface{}{
			"memory": map[string]interface{}{
				"heap_alloc_mb":    float64(stats.HeapAlloc) / 1024 / 1024,
				"heap_sys_mb":      float64(stats.HeapSys) / 1024 / 1024,
				"heap_idle_mb":     float64(stats.HeapIdle) / 1024 / 1024,
				"heap_inuse_mb":    float64(stats.HeapInuse) / 1024 / 1024,
				"heap_released_mb": float64(stats.HeapReleased) / 1024 / 1024,
				"heap_objects":     stats.HeapObjects,
				"stack_inuse_mb":   float64(stats.StackInuse) / 1024 / 1024,
				"total_alloc_mb":   float64(stats.TotalAlloc) / 1024 / 1024,
			},
			"gc": map[string]interface{}{
				"count":            stats.NumGC,
				"pause_total":      time.Duration(stats.PauseTotalNs).String(),
				"pause_last":       time.Duration(stats.PauseNs[(stats.NumGC+255)%256]).String(),
				"cpu_fraction_pct": fmt.Sprintf("%.4f", stats.GCCPUFraction*100),
			},
			"runtime": map[string]interface{}{
				"goroutines": runtime.NumGoroutine(),
				"gomaxprocs": runtime.GOMAXPROCS(0),
				"version":    runtime.Version(),
			},
		}

		w.Header().Set("Content-Type", "application/json")
		json.NewEncoder(w).Encode(response)
	})

	server := &http.Server{
		Addr:    ":8080",
		Handler: mux,
	}

	go func() {
		logger.Info("服务启动", "addr", ":8080")
		if err := server.ListenAndServe(); err != http.ErrServerClosed {
			logger.Error("服务异常", "error", err)
		}
	}()

	quit := make(chan os.Signal, 1)
	signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
	<-quit

	cancel()
	shutdownCtx, shutdownCancel := context.WithTimeout(context.Background(), 10*time.Second)
	defer shutdownCancel()
	server.Shutdown(shutdownCtx)
	logger.Info("服务已关闭")
}
```

## 七、常见坑点

```
坑点 1：GOGC=off 不设置 GOMEMLIMIT
  如果关闭 GC 但不设置内存限制，程序会无限增长内存直到 OOM。
  正确做法：GOGC=off 必须配合 GOMEMLIMIT 使用。

坑点 2：sync.Pool 中存储带状态的对象
  sync.Pool 的对象在 GC 时会被清空，不能假设对象一定存在。
  也不能在归还后继续使用对象的引用。

坑点 3：误以为 HeapSys 是实际使用的内存
  HeapSys 是从 OS 申请的虚拟内存，不等于实际使用量。
  实际使用量看 HeapAlloc 或 HeapInuse。

坑点 4：切片缩容后底层数组不释放
  s = s[:0] 只重置长度，底层数组仍然占用内存。
  如果需要释放，要将变量设为 nil 或赋值新切片。

坑点 5：Map 删除元素后内存不释放
  Go 的 map 删除元素后不会缩容。
  如果 map 曾经很大，即使删空了，内存也不会归还。
  解决方案：创建新 map 替换旧 map。

坑点 6：逃逸分析被内联影响
  使用 -gcflags="-m" 看到的结果可能因为内联而不同。
  加 -l 禁用内联可以看到更准确的逃逸信息。

坑点 7：MemProfileRate 设置时机
  必须在程序启动时设置（最好在 init() 中）。
  运行中修改可能导致采样数据不准确。

坑点 8：容器环境内存统计不准
  runtime.MemStats 不考虑 cgroup 限制。
  容器中应参考 /sys/fs/cgroup/memory/ 下的指标。
```

|---|---|---|
| 减少堆分配 | 使用值类型、避免装箱 | 降低 GC 压力 |

| 小结构体 | 值传递 | 避免逃逸 |
