# Go 进阶篇 01 — 接口（Interface）

> **适用版本**：Go 1.24 | **面向读者**：有编程经验的 SRE / DevOps 工程师

---

## 导读

接口是 Go 语言最核心的抽象机制。与 Java/C# 不同，Go 的接口是**隐式实现**的——任何类型只要实现了接口中声明的全部方法，就自动满足该接口，无需显式声明 `implements`。这种设计被称为**鸭子类型**（Duck Typing）：如果它走起来像鸭子、叫起来像鸭子，那它就是鸭子。

对于 SRE/DevOps 工程师来说，理解接口至关重要：
- 编写可测试的代码（用接口做 mock）
- 理解标准库的设计哲学（`io.Reader`、`io.Writer` 等）
- 构建可扩展的工具链和插件系统

本篇将从接口定义开始，覆盖类型断言、接口组合、标准库常见接口，最终深入接口的 nil 陷阱和最佳实践。

## 1. 接口定义基础

### 1.1 什么是接口

接口是一组方法签名的集合。它定义了"行为契约"，但不关心具体实现。

```go
package main

import "fmt"

// Greeter 接口：任何能打招呼的东西
type Greeter interface {
	Greet() string
}

// Person 结构体
type Person struct {
	Name string
}

// Person 实现了 Greeter 接口（隐式实现，无需声明）
func (p Person) Greet() string {
	return fmt.Sprintf("你好，我是 %s", p.Name)
}

// Bot 结构体
type Bot struct {
	Model string
}

// Bot 也实现了 Greeter 接口
func (b Bot) Greet() string {
	return fmt.Sprintf("你好，我是 %s 型号的机器人", b.Model)
}

// sayHello 接受任何实现了 Greeter 接口的值
func sayHello(g Greeter) {
	fmt.Println(g.Greet())
}

func main() {
	p := Person{Name: "张三"}
	b := Bot{Model: "GPT-5"}

	sayHello(p) // 输出：你好，我是 张三
	sayHello(b) // 输出：你好，我是 GPT-5 型号的机器人
}
```

### 1.2 隐式实现（鸭子类型）

Go 不需要显式声明某个类型实现了某个接口。编译器会在赋值时自动检查。

```go
package main

import "fmt"

// Logger 接口
type Logger interface {
	Log(msg string)
}

// ConsoleLogger 没有任何 "implements Logger" 声明
type ConsoleLogger struct{}

func (c ConsoleLogger) Log(msg string) {
	fmt.Println("[控制台]", msg)
}

// FileLogger 同样隐式实现
type FileLogger struct {
	FilePath string
}

func (f FileLogger) Log(msg string) {
	fmt.Printf("[文件:%s] %s\n", f.FilePath, msg)
}

func main() {
	// 编译器自动检查 ConsoleLogger 是否满足 Logger 接口
	var logger Logger

	logger = ConsoleLogger{}
	logger.Log("服务启动成功")

	logger = FileLogger{FilePath: "/var/log/app.log"}
	logger.Log("开始处理请求")
}
```

### 1.3 编译期接口检查

有时我们想在编译期确认某个类型实现了特定接口，可以使用以下惯用写法：

```go
package main

import "fmt"

type Printer interface {
	Print()
}

type MyPrinter struct{}

func (m MyPrinter) Print() {
	fmt.Println("打印中...")
}

// 编译期检查：确保 MyPrinter 实现了 Printer 接口
// 如果 MyPrinter 没有实现 Printer，编译会报错
var _ Printer = (*MyPrinter)(nil)
var _ Printer = MyPrinter{} // 这种写法也可以

func main() {
	var p Printer = MyPrinter{}
	p.Print()
}
```

## 2. 空接口 interface{} 与 any

### 2.1 空接口的含义

空接口 `interface{}` 没有任何方法要求，因此**所有类型都实现了空接口**。从 Go 1.18 开始，`any` 是 `interface{}` 的类型别名。

```go
package main

import "fmt"

// printAnything 接受任何类型的值
func printAnything(v any) {
	fmt.Printf("值: %v, 类型: %T\n", v, v)
}

func main() {
	printAnything(42)           // 值: 42, 类型: int
	printAnything("hello")     // 值: hello, 类型: string
	printAnything(3.14)        // 值: 3.14, 类型: float64
	printAnything(true)        // 值: true, 类型: bool
	printAnything([]int{1, 2}) // 值: [1 2], 类型: []int
}
```

### 2.2 any 类型的使用场景

```go
package main

import (
	"encoding/json"
	"fmt"
	"log"
)

func main() {
	// 场景1：解析未知结构的 JSON
	jsonStr := `{"name": "k8s-node-01", "cpu": 4, "online": true}`
	var data map[string]any
	if err := json.Unmarshal([]byte(jsonStr), &data); err != nil {
		log.Fatal(err)
	}
	fmt.Println("节点名称:", data["name"]) // 节点名称: k8s-node-01
	fmt.Println("CPU 核数:", data["cpu"])  // CPU 核数: 4

	// 场景2：异构集合
	items := []any{1, "两", 3.0, true}
	for _, item := range items {
		fmt.Printf("%v (%T)\n", item, item)
	}
}
```

### 2.3 any 与 interface{} 完全等价

```go
package main

import "fmt"

// 以下两种写法完全等价（Go 1.18+）
func foo(v interface{}) {
	fmt.Println(v)
}

func bar(v any) {
	fmt.Println(v)
}

func main() {
	foo("使用 interface{}")
	bar("使用 any")
}
```

## 3. 类型断言

### 3.1 基本类型断言

类型断言用于从接口值中提取其底层具体类型。

```go
package main

import "fmt"

func main() {
	var i any = "hello, SRE"

	// 类型断言：将 any 转为 string
	s := i.(string)
	fmt.Println(s) // hello, SRE

	// 如果断言的类型不对，会 panic
	// n := i.(int) // panic: interface conversion: interface {} is string, not int
}
```

### 3.2 安全的类型断言（comma-ok 模式）

```go
package main

import "fmt"

func describe(i any) {
	// 安全断言：使用 value, ok 模式
	if s, ok := i.(string); ok {
		fmt.Printf("字符串: %q, 长度: %d\n", s, len(s))
		return
	}
	if n, ok := i.(int); ok {
		fmt.Printf("整数: %d\n", n)
		return
	}
	fmt.Printf("未知类型: %T\n", i)
}

func main() {
	describe("hello")   // 字符串: "hello", 长度: 5
	describe(42)         // 整数: 42
	describe(3.14)       // 未知类型: float64
}
```

### 3.3 SRE 实战：从配置中提取类型化的值

```go
package main

import (
	"encoding/json"
	"fmt"
	"log"
)

// Config 模拟从 JSON 配置文件读取的内容
type Config map[string]any

// GetString 安全地获取字符串类型的配置项
func (c Config) GetString(key string) (string, bool) {
	v, exists := c[key]
	if !exists {
		return "", false
	}
	s, ok := v.(string)
	return s, ok
}

// GetFloat64 安全地获取数字类型的配置项（JSON 数字默认解析为 float64）
func (c Config) GetFloat64(key string) (float64, bool) {
	v, exists := c[key]
	if !exists {
		return 0, false
	}
	f, ok := v.(float64)
	return f, ok
}

// GetBool 安全地获取布尔类型的配置项
func (c Config) GetBool(key string) (bool, bool) {
	v, exists := c[key]
	if !exists {
		return false, false
	}
	b, ok := v.(bool)
	return b, ok
}

func main() {
	raw := `{
		"host": "10.0.0.1",
		"port": 8080,
		"debug": true,
		"name": "api-gateway"
	}`

	var cfg Config
	if err := json.Unmarshal([]byte(raw), &cfg); err != nil {
		log.Fatal(err)
	}

	if host, ok := cfg.GetString("host"); ok {
		fmt.Println("主机:", host)
	}
	if port, ok := cfg.GetFloat64("port"); ok {
		fmt.Printf("端口: %d\n", int(port))
	}
	if debug, ok := cfg.GetBool("debug"); ok {
		fmt.Println("调试模式:", debug)
	}
}
```

## 4. 类型 switch

类型 switch 是对多个类型断言的优雅替代方案。

### 4.1 基本用法

```go
package main

import "fmt"

// classifyValue 根据值的类型进行分类
func classifyValue(v any) string {
	switch val := v.(type) {
	case nil:
		return "空值 (nil)"
	case int:
		return fmt.Sprintf("整数: %d", val)
	case float64:
		return fmt.Sprintf("浮点数: %.2f", val)
	case string:
		return fmt.Sprintf("字符串: %q (长度 %d)", val, len(val))
	case bool:
		if val {
			return "布尔值: 真"
		}
		return "布尔值: 假"
	case []int:
		return fmt.Sprintf("整数切片: %v (长度 %d)", val, len(val))
	default:
		return fmt.Sprintf("其他类型: %T", val)
	}
}

func main() {
	values := []any{
		nil, 42, 3.14, "hello", true, []int{1, 2, 3}, struct{}{},
	}
	for _, v := range values {
		fmt.Println(classifyValue(v))
	}
}
```

### 4.2 SRE 场景：处理监控告警事件

```go
package main

import "fmt"

// 告警事件类型定义
type CPUAlert struct {
	Node    string
	Usage   float64
}

type MemoryAlert struct {
	Node    string
	UsedGB  float64
	TotalGB float64
}

type DiskAlert struct {
	Node      string
	MountPath string
	UsedPct   float64
}

type NetworkAlert struct {
	Node    string
	Iface   string
	ErrorPkt int64
}

// processAlert 根据告警类型做不同处理
func processAlert(alert any) {
	switch a := alert.(type) {
	case CPUAlert:
		fmt.Printf("[CPU 告警] 节点 %s: CPU 使用率 %.1f%%\n", a.Node, a.Usage)
		if a.Usage > 90 {
			fmt.Println("  -> 紧急！触发自动扩容")
		}
	case MemoryAlert:
		usedPct := a.UsedGB / a.TotalGB * 100
		fmt.Printf("[内存告警] 节点 %s: 已用 %.1fGB / %.1fGB (%.1f%%)\n",
			a.Node, a.UsedGB, a.TotalGB, usedPct)
	case DiskAlert:
		fmt.Printf("[磁盘告警] 节点 %s: %s 使用率 %.1f%%\n",
			a.Node, a.MountPath, a.UsedPct)
		if a.UsedPct > 95 {
			fmt.Println("  -> 危险！触发日志清理")
		}
	case NetworkAlert:
		fmt.Printf("[网络告警] 节点 %s: 接口 %s 错误包 %d\n",
			a.Node, a.Iface, a.ErrorPkt)
	default:
		fmt.Printf("[未知告警] 类型: %T\n", a)
	}
}

func main() {
	alerts := []any{
		CPUAlert{Node: "node-01", Usage: 95.5},
		MemoryAlert{Node: "node-02", UsedGB: 14.2, TotalGB: 16},
		DiskAlert{Node: "node-03", MountPath: "/data", UsedPct: 97.3},
		NetworkAlert{Node: "node-04", Iface: "eth0", ErrorPkt: 1523},
	}

	for _, alert := range alerts {
		processAlert(alert)
		fmt.Println()
	}
}
```

## 5. 接口组合与嵌入

### 5.1 接口嵌入

Go 的接口支持组合——将小接口嵌入到大接口中，构成更大的行为契约。

```go
package main

import "fmt"

// Reader 接口
type Reader interface {
	Read(data []byte) (int, error)
}

// Writer 接口
type Writer interface {
	Write(data []byte) (int, error)
}

// Closer 接口
type Closer interface {
	Close() error
}

// ReadWriter 组合了 Reader 和 Writer
type ReadWriter interface {
	Reader
	Writer
}

// ReadWriteCloser 组合了三个接口
type ReadWriteCloser interface {
	Reader
	Writer
	Closer
}

// MyBuffer 实现了 ReadWriteCloser
type MyBuffer struct {
	data   []byte
	closed bool
}

func NewMyBuffer() *MyBuffer {
	return &MyBuffer{data: make([]byte, 0)}
}

func (b *MyBuffer) Read(p []byte) (int, error) {
	if b.closed {
		return 0, fmt.Errorf("缓冲区已关闭")
	}
	n := copy(p, b.data)
	b.data = b.data[n:]
	return n, nil
}

func (b *MyBuffer) Write(p []byte) (int, error) {
	if b.closed {
		return 0, fmt.Errorf("缓冲区已关闭")
	}
	b.data = append(b.data, p...)
	return len(p), nil
}

func (b *MyBuffer) Close() error {
	b.closed = true
	b.data = nil
	return nil
}

// 接受组合接口的函数
func copyData(rw ReadWriter) {
	buf := make([]byte, 1024)
	n, _ := rw.Read(buf)
	fmt.Printf("读取了 %d 字节: %s\n", n, buf[:n])
}

func main() {
	buf := NewMyBuffer()

	// *MyBuffer 同时满足 Reader、Writer、Closer、ReadWriter、ReadWriteCloser
	var rwc ReadWriteCloser = buf
	rwc.Write([]byte("Hello, SRE!"))

	data := make([]byte, 20)
	n, _ := rwc.Read(data)
	fmt.Printf("读取: %s\n", data[:n])

	rwc.Close()
	fmt.Println("缓冲区已关闭")
}
```

### 5.2 接口嵌入的扩展：添加额外方法

```go
package main

import "fmt"

type HealthChecker interface {
	HealthCheck() error
}

type MetricsReporter interface {
	ReportMetrics() map[string]float64
}

// Service 组合了两个接口并额外要求 Name 方法
type Service interface {
	HealthChecker
	MetricsReporter
	Name() string
}

// APIService 实现 Service 接口
type APIService struct {
	name   string
	health bool
}

func (a *APIService) Name() string {
	return a.name
}

func (a *APIService) HealthCheck() error {
	if !a.health {
		return fmt.Errorf("服务 %s 不健康", a.name)
	}
	return nil
}

func (a *APIService) ReportMetrics() map[string]float64 {
	return map[string]float64{
		"request_count":  1500,
		"error_rate":     0.02,
		"latency_p99_ms": 120.5,
	}
}

func monitorService(svc Service) {
	fmt.Printf("=== 监控服务: %s ===\n", svc.Name())

	if err := svc.HealthCheck(); err != nil {
		fmt.Printf("健康检查失败: %v\n", err)
	} else {
		fmt.Println("健康检查: 通过")
	}

	metrics := svc.ReportMetrics()
	for k, v := range metrics {
		fmt.Printf("  %s: %.2f\n", k, v)
	}
}

func main() {
	svc := &APIService{name: "user-api", health: true}
	monitorService(svc)
}
```

## 6. 常见标准库接口

### 6.1 io.Reader 和 io.Writer

这是 Go 中最重要的两个接口，几乎贯穿所有 I/O 操作。

```go
package main

import (
	"bytes"
	"fmt"
	"io"
	"os"
	"strings"
)

func main() {
	// strings.Reader 实现了 io.Reader
	reader := strings.NewReader("Hello from io.Reader!\n")

	// os.Stdout 实现了 io.Writer
	// io.Copy 在 Reader 和 Writer 之间传输数据
	bytesWritten, err := io.Copy(os.Stdout, reader)
	if err != nil {
		fmt.Fprintf(os.Stderr, "复制失败: %v\n", err)
		return
	}
	fmt.Printf("写入了 %d 字节\n", bytesWritten)

	// bytes.Buffer 同时实现了 Reader 和 Writer
	var buf bytes.Buffer
	buf.WriteString("第一行\n")
	buf.WriteString("第二行\n")
	fmt.Println("缓冲区内容:")
	io.Copy(os.Stdout, &buf)
}
```

### 6.2 自定义 io.Reader：模拟数据源

```go
package main

import (
	"fmt"
	"io"
)

// InfiniteZeroReader 无限返回零值字节——模拟 /dev/zero
type InfiniteZeroReader struct{}

func (r InfiniteZeroReader) Read(p []byte) (int, error) {
	for i := range p {
		p[i] = 0
	}
	return len(p), nil
}

// LimitedRepeatReader 重复返回指定字节，最多返回 N 字节
type LimitedRepeatReader struct {
	Char      byte
	Remaining int
}

func (r *LimitedRepeatReader) Read(p []byte) (int, error) {
	if r.Remaining <= 0 {
		return 0, io.EOF // 告知调用方：没有更多数据了
	}
	n := len(p)
	if n > r.Remaining {
		n = r.Remaining
	}
	for i := 0; i < n; i++ {
		p[i] = r.Char
	}
	r.Remaining -= n
	return n, nil
}

func main() {
	// 从 InfiniteZeroReader 读 10 个字节
	zeroReader := InfiniteZeroReader{}
	buf := make([]byte, 10)
	n, _ := zeroReader.Read(buf)
	fmt.Printf("从 ZeroReader 读取 %d 字节: %v\n", n, buf)

	// 从 LimitedRepeatReader 读取
	repeatReader := &LimitedRepeatReader{Char: 'A', Remaining: 5}
	result, _ := io.ReadAll(repeatReader)
	fmt.Printf("从 RepeatReader 读取: %q\n", string(result))
}
```

### 6.3 fmt.Stringer 接口

```go
package main

import "fmt"

// fmt.Stringer 接口只有一个方法：String() string
// 实现它可以自定义类型的打印格式

type IPAddress struct {
	A, B, C, D byte
}

// 实现 fmt.Stringer
func (ip IPAddress) String() string {
	return fmt.Sprintf("%d.%d.%d.%d", ip.A, ip.B, ip.C, ip.D)
}

type Server struct {
	Name string
	IP   IPAddress
	Port int
}

func (s Server) String() string {
	return fmt.Sprintf("%s (%s:%d)", s.Name, s.IP, s.Port)
}

func main() {
	ip := IPAddress{192, 168, 1, 100}
	fmt.Println(ip) // 192.168.1.100

	srv := Server{Name: "web-01", IP: ip, Port: 443}
	fmt.Println(srv) // web-01 (192.168.1.100:443)

	// 在格式化字符串中自动调用 String()
	fmt.Printf("服务器信息: %v\n", srv)
	fmt.Printf("服务器信息(详细): %+v\n", srv)
}
```

### 6.4 error 接口

```go
package main

import "fmt"

// error 是 Go 的内置接口：
// type error interface {
//     Error() string
// }

// DeployError 自定义错误类型
type DeployError struct {
	Service string
	Version string
	Reason  string
}

func (e *DeployError) Error() string {
	return fmt.Sprintf("部署失败: 服务=%s, 版本=%s, 原因=%s",
		e.Service, e.Version, e.Reason)
}

func deploy(service, version string) error {
	if version == "" {
		return &DeployError{
			Service: service,
			Version: version,
			Reason:  "版本号不能为空",
		}
	}
	fmt.Printf("成功部署 %s (v%s)\n", service, version)
	return nil
}

func main() {
	if err := deploy("api-server", "1.2.3"); err != nil {
		fmt.Println(err)
	}

	if err := deploy("api-server", ""); err != nil {
		fmt.Println(err) // 部署失败: 服务=api-server, 版本=, 原因=版本号不能为空
	}
}
```

### 6.5 sort.Interface

```go
package main

import (
	"fmt"
	"sort"
)

// sort.Interface 要求实现三个方法：Len()、Less()、Swap()

type Pod struct {
	Name     string
	CPU      float64 // CPU 使用量（核）
	Memory   int     // 内存使用量（MB）
	Restarts int     // 重启次数
}

// ByRestarts 按重启次数排序
type ByRestarts []Pod

func (p ByRestarts) Len() int           { return len(p) }
func (p ByRestarts) Less(i, j int) bool { return p[i].Restarts > p[j].Restarts } // 降序
func (p ByRestarts) Swap(i, j int)      { p[i], p[j] = p[j], p[i] }

// ByCPU 按 CPU 使用量排序
type ByCPU []Pod

func (p ByCPU) Len() int           { return len(p) }
func (p ByCPU) Less(i, j int) bool { return p[i].CPU > p[j].CPU } // 降序
func (p ByCPU) Swap(i, j int)      { p[i], p[j] = p[j], p[i] }

func main() {
	pods := []Pod{
		{"nginx-abc12", 0.5, 128, 0},
		{"redis-def34", 1.2, 512, 3},
		{"app-ghi56", 2.1, 256, 12},
		{"worker-jkl78", 0.8, 1024, 7},
		{"db-mno90", 3.5, 2048, 1},
	}

	// 按重启次数排序
	sort.Sort(ByRestarts(pods))
	fmt.Println("=== 按重启次数排序（降序）===")
	for _, p := range pods {
		fmt.Printf("  %-15s 重启=%d\n", p.Name, p.Restarts)
	}

	// 按 CPU 使用量排序
	sort.Sort(ByCPU(pods))
	fmt.Println("\n=== 按 CPU 使用量排序（降序）===")
	for _, p := range pods {
		fmt.Printf("  %-15s CPU=%.1f 核\n", p.Name, p.CPU)
	}

	// Go 1.8+ 更简洁的写法：sort.Slice
	sort.Slice(pods, func(i, j int) bool {
		return pods[i].Memory > pods[j].Memory
	})
	fmt.Println("\n=== 按内存使用量排序（降序）===")
	for _, p := range pods {
		fmt.Printf("  %-15s 内存=%dMB\n", p.Name, p.Memory)
	}
}
```

## 7. 接口最佳实践

### 7.1 小接口原则

Go 社区推崇**小而精**的接口。标准库中大量接口只有 1~2 个方法。

```go
package main

import "fmt"

// 好的设计：小接口
type Pinger interface {
	Ping() error
}

type Shutdowner interface {
	Shutdown() error
}

type Starter interface {
	Start() error
}

// 可以按需组合
type Lifecycle interface {
	Starter
	Shutdowner
}

// 不好的设计（反面例子）：大而全的接口
// type GodService interface {
//     Start() error
//     Stop() error
//     Restart() error
//     Status() string
//     HealthCheck() error
//     Ping() error
//     GetConfig() map[string]any
//     SetConfig(key, value string) error
//     GetLogs(n int) []string
//     ...
// }

// RedisClient 演示小接口的实现
type RedisClient struct {
	Addr string
}

func (r *RedisClient) Ping() error {
	fmt.Printf("PING %s -> PONG\n", r.Addr)
	return nil
}

func (r *RedisClient) Start() error {
	fmt.Printf("连接到 Redis: %s\n", r.Addr)
	return nil
}

func (r *RedisClient) Shutdown() error {
	fmt.Printf("断开 Redis 连接: %s\n", r.Addr)
	return nil
}

// 函数只依赖它需要的最小接口
func checkHealth(p Pinger) error {
	return p.Ping()
}

func manageLifecycle(svc Lifecycle) {
	svc.Start()
	defer svc.Shutdown()
	fmt.Println("服务运行中...")
}

func main() {
	client := &RedisClient{Addr: "localhost:6379"}

	checkHealth(client)    // 只需要 Pinger
	manageLifecycle(client) // 需要 Starter + Shutdowner
}
```

### 7.2 接受接口，返回结构体

这是 Go 中非常重要的设计原则。

```go
package main

import (
	"fmt"
	"io"
	"strings"
)

// 不好：返回接口（隐藏了具体类型，调用方无法获取额外功能）
// func NewReader() io.Reader { ... }

// 好：返回具体结构体，函数参数接受接口
type LogProcessor struct {
	Prefix string
	Count  int
}

// Process 的参数接受接口（灵活），返回具体类型（明确）
func NewLogProcessor(prefix string) *LogProcessor {
	return &LogProcessor{Prefix: prefix}
}

func (lp *LogProcessor) Process(r io.Reader) ([]string, error) {
	data, err := io.ReadAll(r)
	if err != nil {
		return nil, fmt.Errorf("读取日志失败: %w", err)
	}

	lines := strings.Split(string(data), "\n")
	var result []string
	for _, line := range lines {
		if line == "" {
			continue
		}
		result = append(result, fmt.Sprintf("[%s] %s", lp.Prefix, line))
		lp.Count++
	}
	return result, nil
}

func main() {
	// 可以传入任何 io.Reader：文件、HTTP 响应、字符串...
	logData := "2024-01-01 启动服务\n2024-01-01 收到请求\n2024-01-01 处理完成\n"
	reader := strings.NewReader(logData)

	processor := NewLogProcessor("INFO")
	lines, err := processor.Process(reader)
	if err != nil {
		fmt.Println("错误:", err)
		return
	}

	for _, line := range lines {
		fmt.Println(line)
	}
	fmt.Printf("共处理 %d 行日志\n", processor.Count)
}
```

### 7.3 在消费者端定义接口

Go 的惯用做法是：**接口由使用方（消费者）定义，而非提供方定义**。

```go
package main

import "fmt"

// ========== 提供方代码 ==========

// RealDatabase 是具体实现，不定义接口
type RealDatabase struct {
	DSN string
}

func (db *RealDatabase) Query(sql string) ([]map[string]any, error) {
	// 实际实现：执行 SQL 查询
	fmt.Printf("执行查询: %s (DSN: %s)\n", sql, db.DSN)
	return []map[string]any{{"id": 1, "name": "test"}}, nil
}

func (db *RealDatabase) Exec(sql string) error {
	fmt.Printf("执行操作: %s\n", sql)
	return nil
}

// ========== 消费方代码 ==========

// 消费方只定义自己需要的接口（而非引用提供方的接口）
type Querier interface {
	Query(sql string) ([]map[string]any, error)
}

// UserService 依赖 Querier 接口，而不是 RealDatabase
type UserService struct {
	DB Querier
}

func (s *UserService) GetUser(id int) {
	results, err := s.DB.Query(fmt.Sprintf("SELECT * FROM users WHERE id = %d", id))
	if err != nil {
		fmt.Println("查询失败:", err)
		return
	}
	fmt.Println("查询结果:", results)
}

func main() {
	db := &RealDatabase{DSN: "postgres://localhost/mydb"}
	svc := &UserService{DB: db}
	svc.GetUser(1)
}
```

## 8. 接口 nil 陷阱

这是 Go 中最容易踩的坑之一。

### 8.1 nil 接口 vs 持有 nil 值的接口

```go
package main

import "fmt"

type MyError struct {
	Code    int
	Message string
}

func (e *MyError) Error() string {
	return fmt.Sprintf("错误 %d: %s", e.Code, e.Message)
}

// 这个函数有 bug！
func doSomethingBuggy(fail bool) error {
	var err *MyError // 值为 nil 的 *MyError 指针

	if fail {
		err = &MyError{Code: 500, Message: "内部错误"}
	}

	// 即使 err 是 nil，返回的 error 接口不是 nil！
	// 因为接口包含了类型信息 (*MyError) + 值 (nil)
	return err
}

// 正确的写法
func doSomethingFixed(fail bool) error {
	if fail {
		return &MyError{Code: 500, Message: "内部错误"}
	}
	return nil // 直接返回 nil，不经过中间变量
}

func main() {
	// 演示 nil 陷阱
	fmt.Println("=== Bug 版本 ===")
	err1 := doSomethingBuggy(false)
	if err1 != nil {
		fmt.Println("出错了！但其实逻辑上没出错:", err1) // 会走到这里！
	} else {
		fmt.Println("成功")
	}
	fmt.Printf("err1 == nil? %v, 类型: %T, 值: %v\n", err1 == nil, err1, err1)

	// 正确版本
	fmt.Println("\n=== 修复版本 ===")
	err2 := doSomethingFixed(false)
	if err2 != nil {
		fmt.Println("出错了！:", err2)
	} else {
		fmt.Println("成功") // 正确走到这里
	}
	fmt.Printf("err2 == nil? %v, 类型: %T, 值: %v\n", err2 == nil, err2, err2)
}
```

### 8.2 深入理解接口的内部结构

```go
package main

import (
	"fmt"
	"reflect"
)

type Animal interface {
	Speak() string
}

type Dog struct {
	Name string
}

func (d *Dog) Speak() string {
	return "汪汪!"
}

func inspectInterface(name string, i any) {
	v := reflect.ValueOf(i)
	t := reflect.TypeOf(i)

	if t == nil {
		fmt.Printf("%s: 完全的 nil 接口 (类型=nil, 值=nil)\n", name)
		return
	}

	fmt.Printf("%s: 类型=%v, 值=%v, 是否为 nil=%v\n",
		name, t, v, i == nil)

	// 如果底层是指针，检查指针是否为 nil
	if v.Kind() == reflect.Ptr {
		fmt.Printf("  指针是否为 nil: %v\n", v.IsNil())
	}
}

func main() {
	// 情况1：完全的 nil 接口
	var a1 Animal
	fmt.Printf("a1 == nil? %v\n", a1 == nil) // true
	inspectInterface("a1", a1)

	// 情况2：接口持有 nil 指针
	var dog *Dog // nil 指针
	var a2 Animal = dog
	fmt.Printf("\na2 == nil? %v\n", a2 == nil) // false!（这就是陷阱！）
	inspectInterface("a2", a2)

	// 情况3：接口持有有效值
	a3 := Animal(&Dog{Name: "旺财"})
	fmt.Printf("\na3 == nil? %v\n", a3 == nil) // false
	inspectInterface("a3", a3)
}
```

### 8.3 如何安全地检查接口值

```go
package main

import (
	"fmt"
	"reflect"
)

// isNilInterface 安全地检查接口值是否"实质上"为 nil
func isNilInterface(v any) bool {
	if v == nil {
		return true
	}
	rv := reflect.ValueOf(v)
	kind := rv.Kind()
	// 只有这些 Kind 可以调用 IsNil()
	switch kind {
	case reflect.Ptr, reflect.Map, reflect.Slice, reflect.Chan, reflect.Func, reflect.Interface:
		return rv.IsNil()
	}
	return false
}

func main() {
	var p *int
	var m map[string]int
	var s []int

	tests := []struct {
		name string
		val  any
	}{
		{"nil 本身", nil},
		{"nil *int", p},
		{"nil map", m},
		{"nil slice", s},
		{"有效 int", 42},
		{"有效 string", "hello"},
	}

	for _, tt := range tests {
		fmt.Printf("%-15s -> == nil: %-5v, isNilInterface: %v\n",
			tt.name, tt.val == nil, isNilInterface(tt.val))
	}
}
```

## 9. 实战场景：基于接口的插件系统

```go
package main

import (
	"fmt"
	"strings"
)

// ========== 插件接口定义 ==========

// Notifier 通知器接口
type Notifier interface {
	Name() string
	Send(title, message string) error
}

// ========== 各种通知器实现 ==========

// SlackNotifier Slack 通知
type SlackNotifier struct {
	WebhookURL string
	Channel    string
}

func (s *SlackNotifier) Name() string { return "Slack" }

func (s *SlackNotifier) Send(title, message string) error {
	fmt.Printf("[Slack -> %s] %s: %s\n", s.Channel, title, message)
	// 实际场景：发送 HTTP POST 到 WebhookURL
	return nil
}

// EmailNotifier 邮件通知
type EmailNotifier struct {
	SMTPServer string
	To         []string
}

func (e *EmailNotifier) Name() string { return "Email" }

func (e *EmailNotifier) Send(title, message string) error {
	fmt.Printf("[Email -> %s] %s: %s\n",
		strings.Join(e.To, ","), title, message)
	return nil
}

// PagerDutyNotifier PagerDuty 通知
type PagerDutyNotifier struct {
	ServiceKey string
}

func (p *PagerDutyNotifier) Name() string { return "PagerDuty" }

func (p *PagerDutyNotifier) Send(title, message string) error {
	fmt.Printf("[PagerDuty -> %s] %s: %s\n",
		p.ServiceKey[:8]+"...", title, message)
	return nil
}

// ========== 告警管理器 ==========

type AlertManager struct {
	notifiers []Notifier
}

func NewAlertManager() *AlertManager {
	return &AlertManager{}
}

func (am *AlertManager) Register(n Notifier) {
	am.notifiers = append(am.notifiers, n)
	fmt.Printf("注册通知器: %s\n", n.Name())
}

func (am *AlertManager) Alert(title, message string) {
	fmt.Printf("\n=== 发送告警: %s ===\n", title)
	for _, n := range am.notifiers {
		if err := n.Send(title, message); err != nil {
			fmt.Printf("通知器 %s 发送失败: %v\n", n.Name(), err)
		}
	}
}

func main() {
	am := NewAlertManager()

	// 注册多种通知器（接口的威力：统一处理不同实现）
	am.Register(&SlackNotifier{
		WebhookURL: "https://hooks.slack.com/xxx",
		Channel:    "#alerts",
	})
	am.Register(&EmailNotifier{
		SMTPServer: "smtp.company.com",
		To:         []string{"sre@company.com", "oncall@company.com"},
	})
	am.Register(&PagerDutyNotifier{
		ServiceKey: "abcdef1234567890",
	})

	// 发送告警——所有注册的通知器都会收到
	am.Alert("CPU 使用率过高", "节点 node-01 CPU 使用率达到 95%")
	am.Alert("磁盘空间不足", "节点 node-03 /data 分区使用率 98%")
}
```

## 10. 常见坑点总结

### 坑点1：指针接收者 vs 值接收者

```go
package main

import "fmt"

type Speaker interface {
	Speak() string
}

type Cat struct{ Name string }

// 值接收者——Cat 和 *Cat 都能实现 Speaker
func (c Cat) Speak() string { return c.Name + ": 喵~" }

type Fish struct{ Name string }

// 指针接收者——只有 *Fish 能实现 Speaker
func (f *Fish) Speak() string { return f.Name + ": 咕噜咕噜" }

func main() {
	var s Speaker

	s = Cat{Name: "橘猫"}   // OK：值接收者，值和指针都行
	fmt.Println(s.Speak())

	s = &Cat{Name: "黑猫"} // OK
	fmt.Println(s.Speak())

	// s = Fish{Name: "金鱼"} // 编译错误！Fish 没有实现 Speaker，只有 *Fish 实现了
	s = &Fish{Name: "金鱼"} // OK
	fmt.Println(s.Speak())
}
```

### 坑点2：空切片 vs nil 切片作为接口值

```go
package main

import "fmt"

func main() {
	var s1 []int        // nil 切片
	s2 := []int{}       // 空切片（非 nil）
	s3 := make([]int, 0) // 空切片（非 nil）

	fmt.Println(s1 == nil) // true
	fmt.Println(s2 == nil) // false
	fmt.Println(s3 == nil) // false

	// 当赋值给接口时
	var i1 any = s1
	var i2 any = s2
	fmt.Println(i1 == nil) // false! s1 是 nil 切片但接口值不是 nil
	fmt.Println(i2 == nil) // false
}
```

## 速查表 / 小结

| 概念 | 语法/说明 |
|------|-----------|
| 定义接口 | `type Name interface { Method() ReturnType }` |
| 隐式实现 | 实现所有方法即满足接口，无需 `implements` |
| 空接口 | `interface{}` 或 `any`（Go 1.18+），接受任何类型 |
| 类型断言 | `val, ok := iface.(ConcreteType)` |
| 类型 switch | `switch v := iface.(type) { case T: ... }` |
| 接口嵌入 | `type RW interface { Reader; Writer }` |
| 编译期检查 | `var _ Interface = (*Type)(nil)` |
| nil 陷阱 | 持有 nil 指针的接口值 != nil |
| 小接口原则 | 接口方法越少越好（1~3 个方法） |
| 消费者定义 | 接口在使用方定义，不在提供方 |
| 接受接口返回结构体 | 函数参数用接口，返回值用具体类型 |

### 关键标准库接口速查

| 接口 | 包 | 方法 | 用途 |
|------|-----|------|------|
| `io.Reader` | `io` | `Read(p []byte) (n int, err error)` | 数据读取 |
| `io.Writer` | `io` | `Write(p []byte) (n int, err error)` | 数据写入 |
| `io.Closer` | `io` | `Close() error` | 资源关闭 |
| `fmt.Stringer` | `fmt` | `String() string` | 自定义打印 |
| `error` | 内置 | `Error() string` | 错误表示 |
| `sort.Interface` | `sort` | `Len()`, `Less()`, `Swap()` | 自定义排序 |
| `json.Marshaler` | `encoding/json` | `MarshalJSON() ([]byte, error)` | 自定义 JSON 序列化 |
| `json.Unmarshaler` | `encoding/json` | `UnmarshalJSON([]byte) error` | 自定义 JSON 反序列化 |
| `io.ReadWriter` | `io` | `Read()` + `Write()` | 读写组合 |
| `io.ReadCloser` | `io` | `Read()` + `Close()` | HTTP 响应体等 |

### 决策指南

```
需要抽象吗？
├── 只有一个实现 → 不需要接口
├── 需要可测试性 → 定义接口，方便 mock
├── 多种实现 → 定义接口
└── 接口方法 > 5个？ → 考虑拆分为小接口
```
