# Go 进阶篇 02 — 错误处理（Error Handling）

> **适用版本**：Go 1.24 | **面向读者**：有编程经验的 SRE / DevOps 工程师

---

## 导读

Go 的错误处理是该语言最具争议性的设计之一。Go 没有 try/catch/finally，而是通过**返回值**来传递错误。这种设计强迫开发者在每一步都显式处理错误，看似冗长，却能写出更健壮、更易调试的代码。

对于 SRE/DevOps 工程师来说，理解 Go 的错误处理尤其重要：
- 运维工具中的错误必须被正确处理和上报
- 错误信息需要包含足够的上下文以便排查
- 需要区分可恢复错误和不可恢复错误
- 需要理解 panic/recover 的正确使用场景

本篇将全面覆盖 Go 错误处理的各个方面，从基础到高级，包括 Go 1.13+ 引入的错误包装/解包机制。

## 1. error 接口的本质

### 1.1 error 是一个内置接口

```go
package main

import "fmt"

// error 接口在 Go 内置定义如下（不需要导入任何包）：
// type error interface {
//     Error() string
// }

// 最简单的错误使用
func divide(a, b float64) (float64, error) {
	if b == 0 {
		return 0, fmt.Errorf("除数不能为零")
	}
	return a / b, nil
}

func main() {
	// 正常情况
	result, err := divide(10, 3)
	if err != nil {
		fmt.Println("错误:", err)
		return
	}
	fmt.Printf("10 / 3 = %.2f\n", result)

	// 错误情况
	result, err = divide(10, 0)
	if err != nil {
		fmt.Println("错误:", err) // 错误: 除数不能为零
		return
	}
	fmt.Printf("10 / 0 = %.2f\n", result)
}
```

### 1.2 error 就是一个值

Rob Pike（Go 的创始人之一）提出了著名观点："Errors are values"。错误不是异常，它是程序正常逻辑的一部分。

```go
package main

import (
	"fmt"
	"strings"
)

// error 可以被存储、传递、比较
func main() {
	// 错误可以存储在变量中
	var lastErr error

	operations := []string{"read", "write", "delete", "???"}
	for _, op := range operations {
		if err := performOperation(op); err != nil {
			lastErr = err
			fmt.Printf("操作 %q 失败: %v\n", op, err)
		} else {
			fmt.Printf("操作 %q 成功\n", op)
		}
	}

	if lastErr != nil {
		fmt.Printf("\n最后一个错误: %v\n", lastErr)
	}
}

func performOperation(op string) error {
	valid := []string{"read", "write", "delete"}
	for _, v := range valid {
		if op == v {
			return nil
		}
	}
	return fmt.Errorf("不支持的操作: %s (支持: %s)",
		op, strings.Join(valid, ", "))
}
```

## 2. 创建错误

### 2.1 errors.New — 简单静态错误

```go
package main

import (
	"errors"
	"fmt"
)

// 使用 errors.New 创建简单错误
var (
	ErrNotFound     = errors.New("资源未找到")
	ErrUnauthorized = errors.New("未授权")
	ErrTimeout      = errors.New("请求超时")
)

func getResource(id string) (string, error) {
	switch id {
	case "abc":
		return "数据 ABC", nil
	case "":
		return "", ErrNotFound
	default:
		return "", fmt.Errorf("资源 %s: %w", id, ErrNotFound)
	}
}

func main() {
	// 正常获取
	data, err := getResource("abc")
	if err != nil {
		fmt.Println("错误:", err)
	} else {
		fmt.Println("数据:", data)
	}

	// 资源不存在
	_, err = getResource("")
	if err != nil {
		fmt.Println("错误:", err)
	}

	// errors.New 创建的错误可以用 == 比较
	fmt.Println(err == ErrNotFound) // true
}
```

### 2.2 fmt.Errorf — 带格式化的错误

```go
package main

import "fmt"

func connectDB(host string, port int) error {
	// fmt.Errorf 可以创建包含动态信息的错误
	return fmt.Errorf("连接数据库失败: host=%s, port=%d, 原因: 连接被拒绝",
		host, port)
}

func readConfig(path string) error {
	return fmt.Errorf("读取配置文件 %q 失败: 文件不存在", path)
}

func main() {
	if err := connectDB("10.0.0.5", 5432); err != nil {
		fmt.Println(err)
	}

	if err := readConfig("/etc/app/config.yaml"); err != nil {
		fmt.Println(err)
	}
}
```

## 3. 自定义错误类型

### 3.1 实现 Error() 方法

```go
package main

import "fmt"

// HTTPError 自定义 HTTP 错误类型
type HTTPError struct {
	StatusCode int
	Status     string
	URL        string
	Method     string
}

// 实现 error 接口
func (e *HTTPError) Error() string {
	return fmt.Sprintf("HTTP %d %s: %s %s", e.StatusCode, e.Status, e.Method, e.URL)
}

// IsClientError 判断是否为客户端错误
func (e *HTTPError) IsClientError() bool {
	return e.StatusCode >= 400 && e.StatusCode < 500
}

// IsServerError 判断是否为服务端错误
func (e *HTTPError) IsServerError() bool {
	return e.StatusCode >= 500
}

func callAPI(url string) error {
	// 模拟 API 调用失败
	return &HTTPError{
		StatusCode: 503,
		Status:     "Service Unavailable",
		URL:        url,
		Method:     "GET",
	}
}

func main() {
	err := callAPI("https://api.example.com/health")
	if err != nil {
		fmt.Println("请求失败:", err)

		// 类型断言获取详细信息
		if httpErr, ok := err.(*HTTPError); ok {
			fmt.Printf("状态码: %d\n", httpErr.StatusCode)
			fmt.Printf("是服务端错误: %v\n", httpErr.IsServerError())
			fmt.Printf("是客户端错误: %v\n", httpErr.IsClientError())
		}
	}
}
```

### 3.2 带多种信息的错误类型

```go
package main

import (
	"fmt"
	"time"
)

// DeployError 部署错误，包含详细的上下文信息
type DeployError struct {
	Service   string
	Version   string
	Namespace string
	Phase     string    // build, push, rollout
	Timestamp time.Time
	Err       error     // 底层错误
}

func (e *DeployError) Error() string {
	return fmt.Sprintf("[%s] 部署失败 %s/%s (v%s) 在阶段 %q: %v",
		e.Timestamp.Format("2006-01-02 15:04:05"),
		e.Namespace, e.Service, e.Version, e.Phase, e.Err)
}

// Unwrap 允许使用 errors.Is 和 errors.As 解包底层错误
func (e *DeployError) Unwrap() error {
	return e.Err
}

// ValidationError 配置校验错误
type ValidationError struct {
	Field   string
	Value   any
	Message string
}

func (e *ValidationError) Error() string {
	return fmt.Sprintf("校验失败: 字段 %q (值=%v): %s", e.Field, e.Value, e.Message)
}

func deploy(service, version, namespace string) error {
	// 模拟：镜像构建成功，但推送失败
	return &DeployError{
		Service:   service,
		Version:   version,
		Namespace: namespace,
		Phase:     "push",
		Timestamp: time.Now(),
		Err:       fmt.Errorf("镜像仓库返回 401 Unauthorized"),
	}
}

func validateConfig(replicas int) error {
	if replicas <= 0 || replicas > 100 {
		return &ValidationError{
			Field:   "replicas",
			Value:   replicas,
			Message: "副本数必须在 1-100 之间",
		}
	}
	return nil
}

func main() {
	// 部署错误
	if err := deploy("user-api", "2.1.0", "production"); err != nil {
		fmt.Println(err)
	}

	// 校验错误
	if err := validateConfig(0); err != nil {
		fmt.Println(err)
	}
	if err := validateConfig(200); err != nil {
		fmt.Println(err)
	}
}
```

## 4. 错误包装与解包（Go 1.13+）

### 4.1 使用 %w 包装错误

```go
package main

import (
	"errors"
	"fmt"
	"os"
)

// 底层函数
func readFile(path string) ([]byte, error) {
	data, err := os.ReadFile(path)
	if err != nil {
		// 使用 %w 包装原始错误，保留错误链
		return nil, fmt.Errorf("读取文件 %s: %w", path, err)
	}
	return data, nil
}

// 中间层函数
func loadConfig(path string) (string, error) {
	data, err := readFile(path)
	if err != nil {
		// 继续包装，添加更多上下文
		return "", fmt.Errorf("加载配置: %w", err)
	}
	return string(data), nil
}

// 顶层函数
func initApp() error {
	_, err := loadConfig("/etc/myapp/config.yaml")
	if err != nil {
		return fmt.Errorf("应用初始化: %w", err)
	}
	return nil
}

func main() {
	err := initApp()
	if err != nil {
		// 完整的错误链信息
		fmt.Println("错误:", err)
		// 输出类似：应用初始化: 加载配置: 读取文件 /etc/myapp/config.yaml: open ...: no such file or directory

		// 使用 errors.Is 检查底层错误类型
		if errors.Is(err, os.ErrNotExist) {
			fmt.Println("原因: 文件不存在")
		}

		// 使用 errors.Unwrap 逐层解包
		fmt.Println("\n错误链:")
		current := err
		for current != nil {
			fmt.Printf("  -> %v\n", current)
			current = errors.Unwrap(current)
		}
	}
}
```

### 4.2 errors.Is — 检查错误链中是否包含特定错误

```go
package main

import (
	"errors"
	"fmt"
	"io"
	"os"
)

// 定义哨兵错误
var (
	ErrDBConnectionFailed = errors.New("数据库连接失败")
	ErrQueryTimeout       = errors.New("查询超时")
	ErrNoRows             = errors.New("未找到记录")
)

func queryUser(id int) error {
	// 模拟：底层数据库连接失败
	baseErr := ErrDBConnectionFailed
	// 逐层包装
	return fmt.Errorf("查询用户 %d: %w",
		id, fmt.Errorf("执行 SQL: %w", baseErr))
}

func main() {
	err := queryUser(42)

	// errors.Is 会遍历整个错误链
	fmt.Println("是数据库连接失败?", errors.Is(err, ErrDBConnectionFailed)) // true
	fmt.Println("是查询超时?", errors.Is(err, ErrQueryTimeout))            // false
	fmt.Println("是未找到记录?", errors.Is(err, ErrNoRows))                 // false

	// 常见的标准库哨兵错误
	_, openErr := os.Open("/不存在的文件")
	wrappedErr := fmt.Errorf("操作失败: %w", openErr)

	fmt.Println("\n是文件不存在?", errors.Is(wrappedErr, os.ErrNotExist))   // true
	fmt.Println("是权限拒绝?", errors.Is(wrappedErr, os.ErrPermission))    // false
	fmt.Println("是 EOF?", errors.Is(wrappedErr, io.EOF))                   // false
}
```

### 4.3 errors.As — 提取错误链中的特定类型

```go
package main

import (
	"errors"
	"fmt"
	"net"
)

// APIError 自定义 API 错误
type APIError struct {
	Code    int
	Message string
}

func (e *APIError) Error() string {
	return fmt.Sprintf("API 错误 %d: %s", e.Code, e.Message)
}

// RetryableError 可重试的错误
type RetryableError struct {
	Err        error
	RetryAfter int // 秒
}

func (e *RetryableError) Error() string {
	return fmt.Sprintf("可重试错误 (在 %d 秒后重试): %v", e.RetryAfter, e.Err)
}

func (e *RetryableError) Unwrap() error {
	return e.Err
}

func callService() error {
	apiErr := &APIError{Code: 429, Message: "请求过于频繁"}
	retryErr := &RetryableError{Err: apiErr, RetryAfter: 30}
	return fmt.Errorf("调用用户服务: %w", retryErr)
}

func main() {
	err := callService()
	fmt.Println("原始错误:", err)

	// errors.As 在错误链中查找特定类型
	var retryErr *RetryableError
	if errors.As(err, &retryErr) {
		fmt.Printf("这是可重试错误，%d 秒后重试\n", retryErr.RetryAfter)
	}

	var apiErr *APIError
	if errors.As(err, &apiErr) {
		fmt.Printf("API 错误码: %d, 消息: %s\n", apiErr.Code, apiErr.Message)
	}

	// 检查网络错误（标准库示例）
	_, netErr := net.Dial("tcp", "192.0.2.1:80") // 不可达的 IP
	if netErr != nil {
		var opErr *net.OpError
		if errors.As(netErr, &opErr) {
			fmt.Printf("\n网络操作错误: Op=%s, Net=%s, Addr=%v\n",
				opErr.Op, opErr.Net, opErr.Addr)
		}
	}
}
```

## 5. 哨兵错误（Sentinel Error）

### 5.1 什么是哨兵错误

哨兵错误是预定义的、全局的错误值，用于表示特定的错误条件。

```go
package main

import (
	"errors"
	"fmt"
)

// 定义哨兵错误（包级别变量，通常以 Err 开头）
var (
	ErrNotFound      = errors.New("未找到")
	ErrAlreadyExists = errors.New("已存在")
	ErrInvalidInput  = errors.New("无效输入")
	ErrPermission    = errors.New("权限不足")
	ErrInternal      = errors.New("内部错误")
)

// 模拟用户存储
var users = map[string]string{
	"admin": "管理员",
	"sre":   "SRE 工程师",
}

func getUser(name string) (string, error) {
	if name == "" {
		return "", fmt.Errorf("获取用户: %w", ErrInvalidInput)
	}
	user, ok := users[name]
	if !ok {
		return "", fmt.Errorf("获取用户 %q: %w", name, ErrNotFound)
	}
	return user, nil
}

func createUser(name, role string) error {
	if name == "" {
		return fmt.Errorf("创建用户: %w", ErrInvalidInput)
	}
	if _, ok := users[name]; ok {
		return fmt.Errorf("创建用户 %q: %w", name, ErrAlreadyExists)
	}
	users[name] = role
	return nil
}

func main() {
	// 测试各种情况
	testCases := []struct {
		action string
		fn     func() error
	}{
		{"获取已存在用户", func() error { _, err := getUser("admin"); return err }},
		{"获取不存在用户", func() error { _, err := getUser("ghost"); return err }},
		{"获取空名称", func() error { _, err := getUser(""); return err }},
		{"创建新用户", func() error { return createUser("devops", "DevOps 工程师") }},
		{"创建重复用户", func() error { return createUser("admin", "管理员") }},
	}

	for _, tc := range testCases {
		err := tc.fn()
		if err == nil {
			fmt.Printf("[成功] %s\n", tc.action)
			continue
		}

		fmt.Printf("[失败] %s: %v\n", tc.action, err)

		// 使用 errors.Is 判断错误类型并做出相应处理
		switch {
		case errors.Is(err, ErrNotFound):
			fmt.Println("  -> 处理方式: 返回 404")
		case errors.Is(err, ErrAlreadyExists):
			fmt.Println("  -> 处理方式: 返回 409 Conflict")
		case errors.Is(err, ErrInvalidInput):
			fmt.Println("  -> 处理方式: 返回 400 Bad Request")
		default:
			fmt.Println("  -> 处理方式: 返回 500")
		}
	}
}
```

### 5.2 标准库常见哨兵错误

```go
package main

import (
	"errors"
	"fmt"
	"io"
	"os"
	"strings"
)

func main() {
	// io.EOF — 最常见的哨兵错误之一
	reader := strings.NewReader("hello")
	buf := make([]byte, 10)

	for {
		n, err := reader.Read(buf)
		if errors.Is(err, io.EOF) {
			fmt.Println("到达文件末尾，读取结束")
			break
		}
		if err != nil {
			fmt.Println("读取错误:", err)
			break
		}
		fmt.Printf("读取 %d 字节: %s\n", n, buf[:n])
	}

	// os 包的哨兵错误
	_, err := os.Open("/不存在")
	fmt.Println("\nos.ErrNotExist:", errors.Is(err, os.ErrNotExist))

	// 打印常见的标准库哨兵错误
	fmt.Println("\n常见标准库哨兵错误:")
	fmt.Println("  io.EOF:", io.EOF)
	fmt.Println("  io.ErrUnexpectedEOF:", io.ErrUnexpectedEOF)
	fmt.Println("  io.ErrClosedPipe:", io.ErrClosedPipe)
	fmt.Println("  os.ErrNotExist:", os.ErrNotExist)
	fmt.Println("  os.ErrExist:", os.ErrExist)
	fmt.Println("  os.ErrPermission:", os.ErrPermission)
}
```

## 6. panic 和 recover

### 6.1 panic — 不可恢复的致命错误

```go
package main

import "fmt"

// panic 会导致程序崩溃（除非被 recover 捕获）
// 使用场景：程序遇到不应该发生的情况（编程错误、不可恢复的状态）

func mustParsePort(s string) int {
	var port int
	_, err := fmt.Sscanf(s, "%d", &port)
	if err != nil {
		panic(fmt.Sprintf("无效的端口号: %q: %v", s, err))
	}
	if port < 1 || port > 65535 {
		panic(fmt.Sprintf("端口号超出范围: %d", port))
	}
	return port
}

func main() {
	// 正常情况
	port := mustParsePort("8080")
	fmt.Println("端口:", port)

	// 会 panic（取消注释可以看到效果）
	// port = mustParsePort("abc")
	// port = mustParsePort("99999")

	fmt.Println("程序继续运行...")
}
```

### 6.2 recover — 捕获 panic

```go
package main

import "fmt"

// safeDiv 演示 recover 的使用
func safeDiv(a, b int) (result int, err error) {
	// defer + recover 组合捕获 panic
	defer func() {
		if r := recover(); r != nil {
			err = fmt.Errorf("捕获到 panic: %v", r)
		}
	}()

	// 整数除以零会 panic
	return a / b, nil
}

func main() {
	// 正常除法
	result, err := safeDiv(10, 3)
	if err != nil {
		fmt.Println("错误:", err)
	} else {
		fmt.Println("10 / 3 =", result)
	}

	// 除以零——会被 recover 捕获
	result, err = safeDiv(10, 0)
	if err != nil {
		fmt.Println("错误:", err) // 错误: 捕获到 panic: runtime error: integer divide by zero
	} else {
		fmt.Println("10 / 0 =", result)
	}

	fmt.Println("程序没有崩溃，继续运行...")
}
```

### 6.3 SRE 场景：HTTP 中间件的 recover

```go
package main

import (
	"fmt"
	"log"
	"net/http"
	"runtime/debug"
)

// recoveryMiddleware 防止单个请求的 panic 导致整个服务崩溃
func recoveryMiddleware(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		defer func() {
			if rec := recover(); rec != nil {
				// 记录 panic 信息和调用栈
				log.Printf("[PANIC] %s %s: %v\n%s",
					r.Method, r.URL.Path, rec, debug.Stack())

				// 返回 500 错误给客户端
				http.Error(w, "Internal Server Error", http.StatusInternalServerError)
			}
		}()

		next.ServeHTTP(w, r)
	})
}

func main() {
	mux := http.NewServeMux()

	// 正常处理器
	mux.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "OK")
	})

	// 会 panic 的处理器
	mux.HandleFunc("/panic", func(w http.ResponseWriter, r *http.Request) {
		panic("模拟 panic 场景！")
	})

	// 使用 recovery 中间件包装
	handler := recoveryMiddleware(mux)

	fmt.Println("服务器启动在 :8080")
	fmt.Println("  GET /health — 正常")
	fmt.Println("  GET /panic  — 会触发 panic 但不会崩溃")
	// 取消下行注释可以启动服务器
	// log.Fatal(http.ListenAndServe(":8080", handler))
	_ = handler
}
```

### 6.4 panic 的使用原则

```go
package main

import "fmt"

/*
何时使用 panic：
1. 初始化阶段遇到不可恢复的错误（如必要的配置文件缺失）
2. 编程错误（如数组越界、nil 指针——Go runtime 自动 panic）
3. Must 函数（如 regexp.MustCompile、template.Must）

何时不使用 panic：
1. 普通业务逻辑错误 → 使用 error 返回值
2. 网络/IO 错误 → 使用 error 返回值
3. 用户输入验证错误 → 使用 error 返回值
4. 任何"预期内"的错误 → 使用 error 返回值
*/

// MustInitDB 初始化数据库，失败则 panic（适合在 main/init 中调用）
func MustInitDB(dsn string) {
	if dsn == "" {
		panic("数据库 DSN 不能为空")
	}
	fmt.Println("数据库初始化成功:", dsn)
}

// connectDB 连接数据库，失败返回 error（适合在业务代码中调用）
func connectDB(dsn string) error {
	if dsn == "" {
		return fmt.Errorf("数据库 DSN 不能为空")
	}
	fmt.Println("数据库连接成功:", dsn)
	return nil
}

func main() {
	// 在 main 中使用 Must 系列函数——程序启动阶段
	MustInitDB("postgres://localhost:5432/mydb")

	// 在业务逻辑中使用 error 返回值
	if err := connectDB(""); err != nil {
		fmt.Println("连接失败:", err)
		// 可以选择重试、降级等
	}
}
```

### 7.2 错误信息要有上下文

```go
package main

import (
	"fmt"
	"os"

// 坏的做法：无上下文的错误
func readConfigBad(path string) ([]byte, error) {

func initService(configPath string) error {
```

### 7.3 错误只处理一次

```go
package main

import (
	"fmt"
	"log"
)

// 坏的做法：同一个错误被处理了两次（又记日志又返回）
func processRequestBad(id int) error {
	err := doWork(id)
	if err != nil {
		log.Printf("处理请求 %d 失败: %v", id, err) // 处理了一次
		return err                                     // 又返回给调用方处理
	}
	return nil
}

// 好的做法：要么处理（记日志），要么返回，不要两者都做
func processRequestGood(id int) error {
	err := doWork(id)
	if err != nil {
		// 只添加上下文并返回，让顶层调用方统一处理
		return fmt.Errorf("处理请求 %d: %w", id, err)
	}
	return nil
}

func doWork(id int) error {
	if id < 0 {
		return fmt.Errorf("无效的请求 ID: %d", id)
	}
	return nil
}

func main() {
	// 在顶层统一处理错误
	if err := processRequestGood(-1); err != nil {
		log.Printf("错误: %v", err)
		// 只在一个地方记录日志
	}

	_ = processRequestBad // 避免未使用的警告
}
```

### 7.4 使用 defer 清理资源

```go
package main

import (
	"fmt"
	"os"
)

// 好的做法：使用 defer 确保资源释放
func processFile(path string) error {
	f, err := os.Open(path)
	if err != nil {
		return fmt.Errorf("打开文件: %w", err)
	}
	defer f.Close() // 确保函数退出时关闭文件

	// 处理文件内容...
	info, err := f.Stat()
	if err != nil {
		return fmt.Errorf("获取文件信息: %w", err)
	}
	fmt.Printf("文件 %s, 大小 %d 字节\n", info.Name(), info.Size())

	return nil
}

// 处理 defer Close 的错误
func writeFile(path string, data []byte) (retErr error) {
	f, err := os.Create(path)
	if err != nil {
		return fmt.Errorf("创建文件: %w", err)
	}

	// 如果写入出错，Close 的错误可能也很重要
	defer func() {
		closeErr := f.Close()
		if retErr == nil {
			retErr = closeErr
		}
	}()

	_, err = f.Write(data)
	if err != nil {
		return fmt.Errorf("写入文件: %w", err)
	}

	return nil
}

func main() {
	if err := processFile("/etc/hostname"); err != nil {
		fmt.Println("错误:", err)
	}

	if err := writeFile("/tmp/test-output.txt", []byte("Hello, SRE!\n")); err != nil {
		fmt.Println("错误:", err)
	} else {
		fmt.Println("文件写入成功")
	}
}
```

```go
	"strings"

	if err := json.Unmarshal([]byte(raw), &cfg); err == nil {
		if cfg.Name != "" {
			if cfg.Host != "" {
				if cfg.Port > 0 && cfg.Port < 65536 {
					if cfg.MaxConns > 0 {
						return &cfg, nil
					} else {
						return nil, fmt.Errorf("max_conns 必须大于 0")
					}
				} else {
					return nil, fmt.Errorf("端口号无效")
				}
			} else {
				return nil, fmt.Errorf("host 不能为空")
			}
		} else {
			return nil, fmt.Errorf("name 不能为空")
		}
	} else {
		return nil, fmt.Errorf("JSON 解析失败: %w", err)

// 好的做法：提前返回（Guard Clauses）
func parseConfigGood(raw string) (*ServerConfig, error) {

	if err := json.Unmarshal([]byte(raw), &cfg); err != nil {
		return nil, fmt.Errorf("JSON 解析失败: %w", err)
	}

	if strings.TrimSpace(cfg.Name) == "" {
		return nil, fmt.Errorf("name 不能为空")

		return nil, fmt.Errorf("端口号无效: %d (范围 1-65535)", cfg.Port)
	}

	if cfg.MaxConns <= 0 {
		return nil, fmt.Errorf("max_conns 必须大于 0, 当前值: %d", cfg.MaxConns)
	}

}

	}

	_, err = parseConfigGood(bad)

		fmt.Println("错误:", err)
```

## 9. 实战场景：构建健壮的重试机制

```go
package main

import (
	"errors"
	"fmt"
	"math/rand"
	"time"
)

// RetryConfig 重试配置
type RetryConfig struct {
	MaxAttempts int
	BaseDelay   time.Duration
	MaxDelay    time.Duration
}

// TemporaryError 临时性错误接口
type TemporaryError interface {
	error
	IsTemporary() bool
}

// NetworkError 网络错误
type NetworkError struct {
	Op        string
	Addr      string
	Temporary bool
}

func (e *NetworkError) Error() string {
	status := "永久"
	if e.Temporary {
		status = "临时"
	}
	return fmt.Sprintf("网络%s错误: %s -> %s", status, e.Op, e.Addr)
}

func (e *NetworkError) IsTemporary() bool {
	return e.Temporary
}

// isTemporary 判断错误是否为临时性错误
func isTemporary(err error) bool {
	var tempErr TemporaryError
	if errors.As(err, &tempErr) {
		return tempErr.IsTemporary()
	}
	return false
}

// retry 通用重试函数
func retry(cfg RetryConfig, operation func() error) error {
	var lastErr error

	for attempt := 1; attempt <= cfg.MaxAttempts; attempt++ {
		err := operation()
		if err == nil {
			if attempt > 1 {
				fmt.Printf("  第 %d 次尝试成功\n", attempt)
			}
			return nil
		}

		lastErr = err
		fmt.Printf("  第 %d 次尝试失败: %v\n", attempt, err)

		// 如果不是临时性错误，不再重试
		if !isTemporary(err) {
			return fmt.Errorf("不可重试的错误 (尝试 %d 次后): %w", attempt, err)
		}

		// 计算退避延迟
		if attempt < cfg.MaxAttempts {
			delay := cfg.BaseDelay * time.Duration(1<<(attempt-1))
			if delay > cfg.MaxDelay {
				delay = cfg.MaxDelay
			}
			fmt.Printf("  等待 %v 后重试...\n", delay)
			// 实际使用中这里会 time.Sleep(delay)
		}
	}

	return fmt.Errorf("所有 %d 次尝试均失败: %w", cfg.MaxAttempts, lastErr)
}

// 模拟一个不稳定的服务调用
func unstableServiceCall() error {
	n := rand.Intn(10)
	switch {
	case n < 3:
		return nil // 30% 概率成功
	case n < 8:
		return &NetworkError{Op: "connect", Addr: "api:8080", Temporary: true} // 50% 概率临时错误
	default:
		return &NetworkError{Op: "connect", Addr: "api:8080", Temporary: false} // 20% 概率永久错误
	}
}

func main() {
	rand.New(rand.NewSource(time.Now().UnixNano()))

	cfg := RetryConfig{
		MaxAttempts: 5,
		BaseDelay:   100 * time.Millisecond,
		MaxDelay:    2 * time.Second,
	}

	fmt.Println("=== 调用不稳定服务 ===")
	err := retry(cfg, unstableServiceCall)
	if err != nil {
		fmt.Println("最终失败:", err)

		var netErr *NetworkError
		if errors.As(err, &netErr) {
			fmt.Printf("网络错误详情: Op=%s, Addr=%s, Temporary=%v\n",
				netErr.Op, netErr.Addr, netErr.Temporary)
		}
	} else {
		fmt.Println("最终成功!")
	}
}
```

## 10. 多错误处理（Go 1.20+ errors.Join）

```go
	"fmt"

// validateDeployment 校验部署配置，收集所有错误
func validateDeployment(name, image string, replicas, port int) error {
	var errs []error

	if name == "" {
		errs = append(errs, fmt.Errorf("部署名称不能为空"))

		errs = append(errs, fmt.Errorf("镜像地址不能为空"))

	return errors.Join(errs...)
```

## 常见坑点总结

| 坑点 | 说明 | 解决方案 |
|------|------|----------|
| 忽略错误 | `val, _ := fn()` 丢弃错误 | 始终检查并处理错误 |
| 错误消息无上下文 | 只返回 `return err` | 使用 `fmt.Errorf("上下文: %w", err)` |
| 重复处理错误 | 又记日志又返回 | 要么记日志要么返回，不要两者都做 |
| 错误字符串比较 | `err.Error() == "某错误"` | 使用 `errors.Is` 或 `errors.As` |
| panic 滥用 | 在业务逻辑中 panic | panic 仅用于不可恢复的错误 |
| 丢失错误链 | 使用 `%v` 而非 `%w` 包装 | 使用 `%w` 保留错误链 |
| nil error 接口陷阱 | 返回持有 nil 值的接口 | 直接返回 nil，不经过中间变量 |

## 速查表 / 小结

| 操作 | 语法 |
|------|------|
| 创建简单错误 | `errors.New("错误消息")` |
| 格式化错误 | `fmt.Errorf("上下文: %s", detail)` |
| 包装错误 | `fmt.Errorf("上下文: %w", err)` |
| 解包错误 | `errors.Unwrap(err)` |
| 检查错误链 | `errors.Is(err, target)` |
| 提取错误类型 | `errors.As(err, &target)` |
| 合并多个错误 | `errors.Join(err1, err2, ...)` (Go 1.20+) |
| 自定义错误类型 | 实现 `Error() string` 方法 |
| 可解包的错误 | 实现 `Unwrap() error` 方法 |
| panic | `panic("不可恢复的错误")` |
| recover | `defer func() { if r := recover(); r != nil { ... } }()` |

### 错误处理决策树

```
发生了错误？
├── 是预期内的错误 → 使用 error 返回值
│   ├── 需要区分错误类型？
│   │   ├── 简单区分 → 哨兵错误 + errors.Is
│   │   └── 需要额外信息 → 自定义错误类型 + errors.As
│   ├── 需要添加上下文？ → fmt.Errorf("%w", err)
│   └── 多个错误？ → errors.Join (Go 1.20+)
├── 是编程错误 → panic
│   └── 初始化阶段 → Must 函数
└── 需要防止崩溃 → recover（仅用于 goroutine 边界/HTTP 中间件）
```
