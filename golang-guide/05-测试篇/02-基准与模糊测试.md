# 02-基准与模糊测试

## 导读

当你确认代码功能正确（单元测试通过）之后，下一个问题是：**它够快吗？它够健壮吗？** 作为 SRE/DevOps 工程师，你编写的网关、代理、监控采集器都对性能有极高要求；而面对用户输入或外部数据时，程序的健壮性至关重要。

Go 在 `testing` 包中内置了两种强大的测试机制来回答这两个问题：

- **基准测试（Benchmark）**：精确测量代码性能，发现性能瓶颈
- **模糊测试（Fuzz Testing，Go 1.18+）**：自动生成随机输入，发现边界条件 bug

本章将系统讲解：

- Benchmark 基准测试：`BenchmarkXxx`、`b.N`、计时控制
- 基准测试结果分析与 `benchstat` 工具
- 内存分配报告 `b.ReportAllocs()`
- 子基准测试
- Fuzz Testing 模糊测试：`FuzzXxx`、`f.Fuzz`/`f.Add`、语料库
- 模糊测试发现 bug 的实际案例

> 基于 Go 1.24，所有代码完整可运行。

## 一、Benchmark 基准测试基础

### 1.1 基准测试函数签名

```go
// 基准测试函数必须以 Benchmark 开头，参数为 *testing.B
func BenchmarkXxx(b *testing.B) {
	// b.N 由 testing 框架动态调整，确保测试运行足够长的时间以获得稳定结果
	for i := 0; i < b.N; i++ {
		// 被测代码
	}
}
```

### 1.2 第一个基准测试

```go
// 文件: stringutil/concat.go
package stringutil

import (
	"bytes"
	"fmt"
	"strings"
)

// ConcatPlus 使用 + 拼接字符串
func ConcatPlus(strs []string) string {
	result := ""
	for _, s := range strs {
		result += s
	}
	return result
}

// ConcatSprintf 使用 fmt.Sprintf 拼接字符串
func ConcatSprintf(strs []string) string {
	result := ""
	for _, s := range strs {
		result = fmt.Sprintf("%s%s", result, s)
	}
	return result
}

// ConcatJoin 使用 strings.Join 拼接字符串
func ConcatJoin(strs []string) string {
	return strings.Join(strs, "")
}

// ConcatBuilder 使用 strings.Builder 拼接字符串
func ConcatBuilder(strs []string) string {
	var builder strings.Builder
	for _, s := range strs {
		builder.WriteString(s)
	}
	return builder.String()
}

// ConcatBuilderPrealloc 使用 strings.Builder + 预分配拼接字符串
func ConcatBuilderPrealloc(strs []string) string {
	var builder strings.Builder
	// 预估总长度并预分配
	totalLen := 0
	for _, s := range strs {
		totalLen += len(s)
	}
	builder.Grow(totalLen)
	for _, s := range strs {
		builder.WriteString(s)
	}
	return builder.String()
}

// ConcatBuffer 使用 bytes.Buffer 拼接字符串
func ConcatBuffer(strs []string) string {
	var buf bytes.Buffer
	for _, s := range strs {
		buf.WriteString(s)
	}
	return buf.String()
}
```

### 1.3 运行基准测试

```bash
# 运行所有基准测试（必须用 -bench 标志）
# . 表示匹配所有基准测试函数
go test -bench=. ./stringutil/

# 运行特定的基准测试
go test -bench=BenchmarkConcatBuilder ./stringutil/

# 同时运行单元测试和基准测试
go test -v -bench=. ./stringutil/

# 只运行基准测试，跳过单元测试
go test -bench=. -run=^$ ./stringutil/

# 指定运行时间（默认 1 秒）
go test -bench=. -benchtime=5s ./stringutil/

# 指定运行次数
go test -bench=. -benchtime=1000x ./stringutil/

# 多次运行以获得稳定结果（配合 benchstat 使用）
go test -bench=. -count=10 ./stringutil/ > bench.txt
```

## 二、计时控制：ResetTimer / StopTimer / StartTimer

### 2.1 精确控制计时范围

```go
package benchmark

import (
	"math/rand"
	"sort"
	"testing"
)

// BenchmarkSort_WithSetup 演示计时控制
func BenchmarkSort_WithSetup(b *testing.B) {
	for i := 0; i < b.N; i++ {
		// 停止计时——准备数据不应计入基准
		b.StopTimer()
		data := make([]int, 10000)
		for j := 0; j < len(data); j++ {
			data[j] = rand.Intn(100000)
		}
		// 开始计时——只测量排序本身的性能
		b.StartTimer()

		sort.Ints(data)
	}
}

// BenchmarkSort_WithResetTimer 使用 ResetTimer
func BenchmarkSort_WithResetTimer(b *testing.B) {
	// 一次性准备数据（适用于数据可以复用的场景）
	data := make([]int, 10000)
	for j := 0; j < len(data); j++ {
		data[j] = rand.Intn(100000)
	}
	// 重置计时器——排除上面准备数据的时间
	b.ResetTimer()

	for i := 0; i < b.N; i++ {
		// 注意：sort.Ints 会修改原切片，第二次迭代排序的是已排序的数据
		// 这可能导致测试不准确，此处仅为演示 ResetTimer
		dataCopy := make([]int, len(data))
		copy(dataCopy, data)
		sort.Ints(dataCopy)
	}
}

// BenchmarkExpensiveSetup 复杂初始化场景
func BenchmarkExpensiveSetup(b *testing.B) {
	// 模拟：创建一个大型数据结构
	type Record struct {
		ID    int
		Name  string
		Score float64
	}

	// 准备阶段（不计入基准）
	records := make([]Record, 10000)
	for i := range records {
		records[i] = Record{
			ID:    i,
			Name:  "user",
			Score: rand.Float64() * 100,
		}
	}
	b.ResetTimer()

	// 被测量的代码
	for i := 0; i < b.N; i++ {
		// 按 Score 排序
		sort.Slice(records, func(i, j int) bool {
			return records[i].Score < records[j].Score
		})
	}
}
```

## 四、子基准测试

子基准测试允许在一个基准测试函数中测试不同参数组合。

### 4.1 不同数据规模的子基准测试

```go
package subbench

import (
	"sort"
	"math/rand"
	"testing"
)

// BenchmarkSort_Sizes 测试不同数据规模下的排序性能
func BenchmarkSort_Sizes(b *testing.B) {
	sizes := []int{10, 100, 1000, 10000, 100000}

	for _, size := range sizes {
		b.Run(fmt.Sprintf("size=%d", size), func(b *testing.B) {
			// 准备数据
			data := make([]int, size)
			for i := range data {
				data[i] = rand.Intn(size * 10)
			}
			b.ResetTimer()
			b.ReportAllocs()

			for i := 0; i < b.N; i++ {
				b.StopTimer()
				// 每次迭代都需要一份新的未排序数据
				dataCopy := make([]int, len(data))
				copy(dataCopy, data)
				b.StartTimer()

				sort.Ints(dataCopy)
			}
		})
	}
}
```

```go
package subbench

import (
	"crypto/md5"
	"crypto/sha1"
	"crypto/sha256"
	"crypto/sha512"
	"fmt"
	"hash"
	"testing"
)

// BenchmarkHash 对比不同哈希算法的性能
func BenchmarkHash(b *testing.B) {
	data := []byte("The quick brown fox jumps over the lazy dog")

	// 定义要测试的哈希算法
	algorithms := []struct {
		name string
		hash hash.Hash
	}{
		{"MD5", md5.New()},
		{"SHA1", sha1.New()},
		{"SHA256", sha256.New()},
		{"SHA512", sha512.New()},
	}

	// 不同数据大小
	sizes := []int{64, 256, 1024, 4096, 65536}

	for _, algo := range algorithms {
		for _, size := range sizes {
			name := fmt.Sprintf("%s/size=%d", algo.name, size)
			b.Run(name, func(b *testing.B) {
				// 生成指定大小的数据
				testData := make([]byte, size)
				for i := range testData {
					testData[i] = data[i%len(data)]
				}
				b.ResetTimer()
				b.ReportAllocs()
				b.SetBytes(int64(size)) // 报告吞吐量（bytes/sec）

				for i := 0; i < b.N; i++ {
					algo.hash.Reset()
					algo.hash.Write(testData)
					algo.hash.Sum(nil)
				}
			})
		}
	}
}
```

```bash
# 运行子基准测试
go test -bench=BenchmarkHash -benchmem ./subbench/

# 运行特定子基准测试
go test -bench=BenchmarkHash/SHA256 ./subbench/

# 运行特定大小
go test -bench="BenchmarkHash/.*/size=1024" ./subbench/
```

### 4.2 b.SetBytes() 报告吞吐量

```go
package throughput

import (
	"compress/gzip"
	"bytes"
	"io"
	"testing"
)

// BenchmarkGzipCompress 测试 gzip 压缩吞吐量
func BenchmarkGzipCompress(b *testing.B) {
	levels := []struct {
		name  string
		level int
	}{
		{"BestSpeed", gzip.BestSpeed},
		{"Default", gzip.DefaultCompression},
		{"BestCompression", gzip.BestCompression},
	}

	// 准备测试数据
	data := bytes.Repeat([]byte("Hello World! This is test data for compression benchmark. "), 1000)

	for _, lv := range levels {
		b.Run(lv.name, func(b *testing.B) {
			b.SetBytes(int64(len(data))) // 设置处理的字节数，输出会包含 MB/s
			b.ReportAllocs()
			b.ResetTimer()

			for i := 0; i < b.N; i++ {
				var buf bytes.Buffer
				w, _ := gzip.NewWriterLevel(&buf, lv.level)
				w.Write(data)
				w.Close()
			}
		})
	}
}

// BenchmarkGzipDecompress 测试 gzip 解压吞吐量
func BenchmarkGzipDecompress(b *testing.B) {
	// 准备压缩数据
	data := bytes.Repeat([]byte("Hello World! This is test data for decompression benchmark. "), 1000)
	var compressed bytes.Buffer
	w := gzip.NewWriter(&compressed)
	w.Write(data)
	w.Close()
	compressedBytes := compressed.Bytes()

	b.SetBytes(int64(len(data)))
	b.ReportAllocs()
	b.ResetTimer()

	for i := 0; i < b.N; i++ {
		reader, _ := gzip.NewReader(bytes.NewReader(compressedBytes))
		io.ReadAll(reader)
		reader.Close()
	}
}
```

## 五、benchstat 结果分析工具

`benchstat` 是官方提供的基准测试结果分析工具，用于比较两次基准测试的结果差异。

### 5.1 安装与使用

```bash
# 安装 benchstat
go install golang.org/x/perf/cmd/benchstat@latest

# 运行基准测试并保存结果
go test -bench=. -count=10 ./stringutil/ > old.txt

# 修改代码后再次运行
go test -bench=. -count=10 ./stringutil/ > new.txt

# 比较两次结果
benchstat old.txt new.txt
```

### 5.2 benchstat 输出解读

```
name                    old time/op    new time/op    delta
ConcatPlus-8            112μs ± 3%     108μs ± 2%    -3.57%  (p=0.000 n=10+10)
ConcatBuilder-8         680ns ± 2%     450ns ± 1%   -33.82%  (p=0.000 n=10+10)
ConcatBuilderPrealloc-8 310ns ± 1%     295ns ± 2%    -4.84%  (p=0.001 n=10+10)

name                    old alloc/op   new alloc/op   delta
ConcatPlus-8            530kB ± 0%     530kB ± 0%      ~     (p=1.000 n=10+10)
ConcatBuilder-8         2.08kB ± 0%    1.02kB ± 0%  -50.96%  (p=0.000 n=10+10)

name                    old allocs/op  new allocs/op  delta
ConcatPlus-8            99.0 ± 0%      99.0 ± 0%       ~     (all equal)

```

各列含义：

| 列 | 说明 |
|----|------|
| `old time/op` | 旧版本每次操作耗时 |
| `new time/op` | 新版本每次操作耗时 |
| `delta` | 变化百分比（负数表示改善） |
| `± X%` | 标准差百分比 |
| `p=0.000` | p 值（<0.05 表示差异具有统计显著性） |
| `n=10+10` | 新旧各 10 个样本 |
| `~` | 无显著差异 |

### 5.3 基准测试工作流

```bash
#!/bin/bash
# bench-compare.sh - 对比优化前后的性能

set -e

PACKAGE="./..."
COUNT=10

echo "=== 切换到优化前的代码 ==="
git stash  # 或者 git checkout <old-commit>

echo "=== 运行基准测试（优化前）==="
go test -bench=. -benchmem -count=${COUNT} ${PACKAGE} > /tmp/bench_old.txt

echo "=== 切换到优化后的代码 ==="
git stash pop  # 或者 git checkout <new-commit>

echo "=== 运行基准测试（优化后）==="
go test -bench=. -benchmem -count=${COUNT} ${PACKAGE} > /tmp/bench_new.txt

echo "=== 对比结果 ==="
benchstat /tmp/bench_old.txt /tmp/bench_new.txt
```

## 六、实战：完整的性能测试案例

### 6.1 JSON 序列化性能对比

```go
// 文件: jsonbench/json_bench.go
package jsonbench

import (
	"encoding/json"
)

// Event 事件结构体（模拟日志采集场景）
type Event struct {
	Timestamp  string            `json:"timestamp"`
	Level      string            `json:"level"`
	Message    string            `json:"message"`
	Service    string            `json:"service"`
	Host       string            `json:"host"`
	Tags       map[string]string `json:"tags"`
	Fields     map[string]interface{} `json:"fields"`
}

// NewSampleEvent 创建样例事件
func NewSampleEvent() *Event {
	return &Event{
		Timestamp: "2024-01-15T10:30:00Z",
		Level:     "info",
		Message:   "Request processed successfully",
		Service:   "api-gateway",
		Host:      "prod-server-01",
		Tags: map[string]string{
			"env":     "production",
			"region":  "us-east-1",
			"version": "v2.1.0",
		},
		Fields: map[string]interface{}{
			"request_id":    "req-abc123",
			"status_code":   200,
			"response_time": 45.2,
			"bytes_sent":    1024,
		},
	}
}

// MarshalJSON 标准库序列化
func MarshalJSON(e *Event) ([]byte, error) {
	return json.Marshal(e)
}

// UnmarshalJSON 标准库反序列化
func UnmarshalJSON(data []byte) (*Event, error) {
```

## 七、Fuzz Testing 模糊测试（Go 1.18+）

模糊测试是 Go 1.18 引入的原生特性，通过自动生成随机输入来发现代码中的边界条件 bug 和异常。

### 7.1 模糊测试基础

```go
// 文件: fuzzexample/reverse.go
package fuzzexample

import (
	"errors"
	"unicode/utf8"
)

// Reverse 反转字符串（可能有 bug 的版本）
func Reverse(s string) (string, error) {
	if !utf8.ValidString(s) {
		return "", errors.New("输入不是有效的 UTF-8 字符串")
	}
	runes := []rune(s)
	for i, j := 0, len(runes)-1; i < j; i, j = i+1, j-1 {
		runes[i], runes[j] = runes[j], runes[i]
	}
	return string(runes), nil
}
```

```go
// 文件: fuzzexample/reverse_test.go
package fuzzexample

import (
	"testing"
	"unicode/utf8"
)

// FuzzReverse 模糊测试反转函数
func FuzzReverse(f *testing.F) {
	// f.Add() 添加种子语料库（seed corpus）
	// 这些值作为模糊测试的起始输入
	f.Add("hello")
	f.Add("世界")
	f.Add("")
	f.Add("a")
	f.Add("abc")
	f.Add("Hello, 世界! 🌍")
	f.Add("12345")

	// f.Fuzz() 定义模糊测试逻辑
	// 参数类型必须与 f.Add() 中的类型一致
	f.Fuzz(func(t *testing.T, orig string) {
		// 反转两次应该得到原始字符串
		rev, err1 := Reverse(orig)
		if err1 != nil {
			// 如果第一次反转失败，直接返回（这不是 bug）
			return
		}

		doubleRev, err2 := Reverse(rev)
		if err2 != nil {
			t.Fatalf("二次反转失败: %v", err2)
		}

		// 属性1：反转两次等于原始值
		if orig != doubleRev {
			t.Errorf("反转两次不等于原始值:\n原始: %q\n一次: %q\n二次: %q",
				orig, rev, doubleRev)
		}

		// 属性2：反转后仍是有效的 UTF-8
		if utf8.ValidString(orig) && !utf8.ValidString(rev) {
			t.Errorf("反转后不是有效的 UTF-8: %q -> %q", orig, rev)
		}

		// 属性3：长度不变
		if len([]rune(orig)) != len([]rune(rev)) {
			t.Errorf("反转后 rune 长度变化: %d -> %d", len([]rune(orig)), len([]rune(rev)))
		}
	})
}
```

### 8.1 案例：URL 解析器的边界 bug

```go
// 文件: urlparser/parser.go
package urlparser

import (
	"fmt"
	"strings"
)

// URLInfo URL 解析结果
type URLInfo struct {
	Scheme string
	Host   string
	Port   string
	Path   string
	Query  string
}

// ParseURL 自定义 URL 解析器（故意包含 bug 用于演示）
func ParseURL(rawURL string) (*URLInfo, error) {
	if rawURL == "" {
		return nil, fmt.Errorf("URL 不能为空")
	}

	info := &URLInfo{}

	// 解析 scheme
	if idx := strings.Index(rawURL, "://"); idx > 0 {
		info.Scheme = rawURL[:idx]
		rawURL = rawURL[idx+3:]
	}

	// 解析 query
	if idx := strings.Index(rawURL, "?"); idx >= 0 {
		info.Query = rawURL[idx+1:]
		rawURL = rawURL[:idx]
	}

	// 解析 path
	if idx := strings.Index(rawURL, "/"); idx >= 0 {
		info.Path = rawURL[idx:]
		rawURL = rawURL[:idx]
	}

	// 解析 host 和 port
	// BUG: 如果 host 包含 IPv6 地址（如 [::1]:8080），解析会出错
	if idx := strings.LastIndex(rawURL, ":"); idx >= 0 {
		info.Host = rawURL[:idx]
		info.Port = rawURL[idx+1:]
	} else {
		info.Host = rawURL
	}

	return info, nil
}
```

### 8.2 案例：数字格式化器的 panic

```go
// 文件: formatter/format.go
package formatter

import (
	"fmt"
	"strings"
)

// FormatNumber 将数字格式化为带千分位分隔符的字符串
// 例如：1234567 -> "1,234,567"
func FormatNumber(n int64) string {
	s := fmt.Sprintf("%d", n)
	negative := false
	if s[0] == '-' {
		negative = true
		s = s[1:]
	}

	// BUG: 当 n = 0 时，下面的逻辑可能不会正确处理
	// BUG: 没有处理 s 长度 <= 3 的简单情况
	parts := make([]string, 0)
	for len(s) > 3 {
		parts = append([]string{s[len(s)-3:]}, parts...)
		s = s[:len(s)-3]
	}
	if len(s) > 0 {
		parts = append([]string{s}, parts...)
	}

	result := strings.Join(parts, ",")
	if negative {
		result = "-" + result
	}
	return result
}

// FormatBytes 将字节数格式化为人类可读的形式
// 例如：1536 -> "1.50 KB"
func FormatBytes(bytes int64) string {
	if bytes < 0 {
		return fmt.Sprintf("-%s", FormatBytes(-bytes))
	}

	units := []string{"B", "KB", "MB", "GB", "TB", "PB"}
	size := float64(bytes)
	unitIndex := 0

	for size >= 1024 && unitIndex < len(units)-1 {
		size /= 1024
		unitIndex++
	}

	if unitIndex == 0 {
		return fmt.Sprintf("%d B", bytes)
	}
	return fmt.Sprintf("%.2f %s", size, units[unitIndex])
}
```

```go
		// 属性2：去掉逗号和负号后应该等于原始数字的绝对值字符串

			if len(parts[0]) == 0 || len(parts[0]) > 3 {

			}
			// 后续部分必须是 3 位
			for i := 1; i < len(parts); i++ {
				if len(parts[i]) != 3 {
					t.Errorf("FormatNumber(%d) = %q; 第 %d 部分长度 %d != 3",
						n, result, i, len(parts[i]))
				}
			}
		}

		// 属性4：负数结果应以 - 开头
		if n < 0 && !strings.HasPrefix(result, "-") {
			t.Errorf("FormatNumber(%d) = %q; 负数应以 - 开头", n, result)
		}
```

## 九、常见坑点

### 坑点 1：b.N 循环写错位置

```go
// 错误：被测代码不在 b.N 循环中
func BenchmarkWrong(b *testing.B) {
	// 这个函数只被调用了一次！
	expensiveOperation()
	for i := 0; i < b.N; i++ {
		// 什么都不做
	}
}

// 正确：被测代码在 b.N 循环中
func BenchmarkCorrect(b *testing.B) {
	for i := 0; i < b.N; i++ {
		expensiveOperation()
	}
}
```

### 坑点 2：编译器优化消除了被测代码

```go
// 错误：编译器可能优化掉无副作用的计算
func BenchmarkOptimizedAway(b *testing.B) {
	for i := 0; i < b.N; i++ {
		fibonacci(20) // 返回值被丢弃，编译器可能优化掉
	}
}

// 正确：使用全局变量防止优化
var benchResult int

func BenchmarkPreventOptimization(b *testing.B) {
	var r int
	for i := 0; i < b.N; i++ {
		r = fibonacci(20)
	}
	benchResult = r // 赋值给全局变量，防止编译器优化
}
```

### 坑点 3：StopTimer/StartTimer 开销

```go
// 注意：StopTimer/StartTimer 本身有一定开销
// 如果被测操作非常快（几纳秒），Stop/Start 的开销会显著影响结果
// 这种情况下应该批量处理

// 不推荐：每次迭代都 Stop/Start
func BenchmarkWithStopStart(b *testing.B) {
	for i := 0; i < b.N; i++ {
		b.StopTimer()
		data := prepareSmallData() // 非常快的准备
		b.StartTimer()
		processSmallData(data)     // 非常快的处理
	}
}

// 推荐：只在准备阶段使用一次 ResetTimer
func BenchmarkWithResetTimer(b *testing.B) {
	data := prepareSmallData()
	b.ResetTimer()
	for i := 0; i < b.N; i++ {
		processSmallData(data)
	}
}
```

### 坑点 4：模糊测试的函数签名限制

```go
// 错误：参数类型不支持
func FuzzWrongType(f *testing.F) {
	// f.Add([]int{1, 2, 3}) // 编译错误：不支持 []int 类型
	// f.Add(time.Now())      // 编译错误：不支持 time.Time 类型
}

// 正确：只使用支持的类型
func FuzzCorrectType(f *testing.F) {
	f.Add("hello", int64(42), true, []byte("data"))
	f.Fuzz(func(t *testing.T, s string, n int64, b bool, data []byte) {
		// 如果需要复杂类型，可以从基本类型构造
	})
}
```

### 坑点 5：基准测试结果不稳定

```bash
# 不稳定的原因：
# 1. 系统负载波动
# 2. CPU 温度降频
# 3. 后台进程干扰

# 解决方案：
# 1. 多次运行
go test -bench=. -count=10 > result.txt

# 2. 使用 benchstat 统计分析
benchstat result.txt

# 3. 固定 CPU 频率（Linux）
# sudo cpupower frequency-set -g performance

# 4. 关闭不必要的后台程序

# 5. 使用 taskset 绑定 CPU 核心
# taskset -c 0 go test -bench=.
```

## 十、速查表

### 基准测试命令

```bash
go test -bench=.                    # 运行所有基准测试
go test -bench=BenchmarkXxx         # 运行匹配的基准测试
go test -bench=. -run=^$            # 只运行基准测试
go test -bench=. -benchtime=5s      # 设置运行时间
go test -bench=. -benchtime=1000x   # 设置运行次数
go test -bench=. -benchmem          # 显示内存分配
go test -bench=. -count=10          # 运行 10 次
go test -bench=. -cpu=1,2,4,8       # 测试不同 CPU 数
go test -bench=. -cpuprofile=p.out  # 生成 CPU profile
go test -bench=. -memprofile=m.out  # 生成内存 profile
```

### 模糊测试命令

```bash
go test -fuzz=FuzzXxx               # 运行模糊测试
go test -fuzz=FuzzXxx -fuzztime=30s # 限制时间
go test -fuzz=FuzzXxx -fuzztime=1000x # 限制次数
go test -run=FuzzXxx                # 只用种子语料库运行
go clean -fuzzcache                 # 清理模糊测试缓存
```

### *testing.B 方法速查

| 方法 | 说明 |
|------|------|
| `b.N` | 迭代次数（框架自动调整） |
| `b.ResetTimer()` | 重置计时器 |
| `b.StopTimer()` | 暂停计时器 |
| `b.StartTimer()` | 恢复计时器 |
| `b.ReportAllocs()` | 报告内存分配 |
| `b.SetBytes(n)` | 设置处理字节数（报告吞吐量） |
| `b.RunParallel(body)` | 并行基准测试 |
| `b.Run(name, f)` | 子基准测试 |
| `b.Log/Logf` | 记录日志 |
| `b.Error/Fatal` | 报告错误 |
| `b.Skip` | 跳过基准测试 |

### *testing.F 方法速查

| 方法 | 说明 |
|------|------|
| `f.Add(args...)` | 添加种子语料库 |
| `f.Fuzz(func(t, args))` | 定义模糊测试函数 |
| `f.Log/Logf` | 记录日志 |
| `f.Error/Fatal` | 报告错误 |
| `f.Skip` | 跳过模糊测试 |

### 性能优化决策流程

```
1. 先写正确的代码
2. 写基准测试
3. 运行基准测试获取基线数据: go test -bench=. -benchmem -count=10 > old.txt
4. 分析性能瓶颈: go test -bench=. -cpuprofile=cpu.out && go tool pprof cpu.out
5. 实施优化
6. 再次运行基准测试: go test -bench=. -benchmem -count=10 > new.txt
7. 对比结果: benchstat old.txt new.txt
8. 确认优化有效且没有引入 bug
```
